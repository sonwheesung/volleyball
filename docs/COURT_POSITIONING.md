# COURT_POSITIONING — 수비/리시브 포지셔닝 모델 (보드 연출 단일 기준)

> **상태: ✅ 구현 반영(`receiveFormation`/`serveFormation`/`switchedSpots`에 좌표 박힘 — 룰 43·45·47·49 등).
> 단 "magic 숫자 제거"(역할유도 `positionOf` 자동화)는 미달 — 여전히 하드코딩 좌표.** 사용자 교습(2026-06-21) + 5-1 서브리시브 리서치 기반. 추측 금지 —
> 이 문서가 `receiveFormation`·`serveFormation`·`switchedSpots`·`courtDirector`의 **단일 기준**이다.
> 위치 변경은 코드보다 **이 문서를 먼저** 고친다([[verify-domain-definitions]]·DOC_DISCIPLINE).

---

## 0. 왜 이 문서가 필요한가 (자동화가 안 되던 이유)

지금까지 보드 포지션은 **자리마다 magic 숫자**(예: 전위 패서 y=0.645→0.695→0.75→0.665)를 박았다.
그래서 로테이션마다 어긋나고 같은 곳을 반복 수정했다(BOARD_RULES 룰 32→43→49 오락가락).

> **핵심 진단**: 위치는 magic 숫자가 아니라 **역할에서 유도(derive)** 되어야 자동화된다.
> "이 선수는 왜 여기 있나"가 역할로 설명되면, 어떤 로테이션이든 자리가 자동으로 나온다.
> 이 문서는 그 **역할 → 위치 규칙**을 정의한다.

---

## 1. 지배 원칙 (모든 자리의 상위 제약)

1. **오버랩(서브 컨택 *순간만*)**: 같은 행 좌<중<우(전위 4·3·2 / 후위 5·6·1), 같은 열 전위<후위
   (전위가 네트에 더 가까움: 4·5 / 3·6 / 2·1). 이 순서만 지키면 나머지 자리는 자유.
   - **인접 쌍만 따진다(FIVB 7.4, 출처 확인 2026-06-24)**: 전위 선수는 **같은 열의 후위 짝**보다만 네트
     쪽이면 된다 — **다른 열의 후위 선수보다 깊어도 합법**이다. 예) 전위 OH(z2)가 리베로(z6)·백OH(z5)보다
     깊어도 무방, **세터(z1, 같은 우열 짝)보다만** 네트 쪽이면 OK. ⇒ "전위 패서가 후위밴드(y≥0.72)에
     들어감"은 **오버랩 위반이 아니다**(자기 열 짝보다 깊을 때만 위반). `_dv_receive`의 "후위밴드 침범"은
     룰이 아니라 **전술 프록시**일 뿐(룰 49 재고 — 아래 변경이력 2026-06-24).
   - **서브 팀은 오버랩 제약 없음(FIVB 2025–2028)**: 서브 순간 **서브 팀 전원**이 아무 위치나 가능
     (받는 팀만 로테이션 순서 유지). ⇒ 코드 검사(`auditBoard`)가 **서버만** 면제하는 건 과한 제약 —
     서브 팀 전원 면제가 룰에 맞다(별도 정리 대상). 출처: FIVB Official Rules 2025–2028.
2. **역할이 자리를 정한다**: passer / setter / blocker / digger / attacker. 위치는 역할의 *일*에서 나온다.
3. **보드 좌표**: 네트 y=0.5, 우리 진영에서 **전위=네트 쪽(0.5 근처), 후위=베이스라인 쪽(1.0 근처)**.
   (홈=하단, 원정=상단 점대칭.)

---

## 2. 역할 정의

| 역할 | 일 | 위치 원리 |
|---|---|---|
| **세터 S** | 세트 전담. **받으면(서브/디그) 공격을 못 짠다** → 가능한 한 첫 터치를 피하고 네트-우 타깃으로 | 받기: 패스 레인에서 빠짐 / 수비: 후위 디그 |
| **패서** | 서브 리시브 = **리베로 + OH 2명**(3-패서, 5-1 표준) | 좌·중·우 레인, 3~5m |
| **블로커** | 전위 3인, 상대 히터 정면에서 블록 | 네트, x=상대 히터 |
| **디거** | 후위, 블록 뒤/옆 공간 디그 | 페리미터 베이스(깊은 코너+중앙) |
| **공격수** | 전위 비패서(MB 속공·OP 우측), 패스 후 어택 | 네트 앞 자기 공격 출발점 |
| **리베로 L** | 후위 MB 대체. 항상 패서·최전방 디거 | 전위·서브·공격·블록 **불가** |

## 3. 시나리오 A — 우리가 받을 때 (상대 서브)

목표: **서브를 잘 패스 + 세터를 네트로 보내 공격을 짠다.**

### A-1. 세터가 **후위**일 때 (핵심 — 사용자 교습)
| 선수 | 자리 | **왜** |
|---|---|---|
| **패서 3(L+OH 2)** | 좌·중·우 레인, 3~5m | 서브 코스를 3등분해 "마이볼". 리베로 중앙 깊게(최고 패서), OH 좌우 |
| **세터(후위)** | 자기 열 **전위 파트너 뒤**(더 깊게) | **받으면 세트 못 함** → 패스 레인에서 빠진다. 전위 파트너 뒤 = 오버랩 합법(세터>전위) + 서브 회피 |
| **세터 앞 전위 파트너** | 한 발 내려와 세터 앞 | 세터가 숨을 수 있게. 공 오면 **리시브**, 안 오면 **공격 전환**, 리시브 깨지면 **커버** |
| **전위 MB/OP(비패서)** | 네트 앞 공격 출발점 | 패스 안 함. 패스 후 바로 때리러 대기 |
| **서브 순간(전환)** | 세터만 **네트-우 타깃 침투** | 나머지는 제자리에서 받거나 전환. **전위가 세터 자리로 가는 게 아니다 — 세터가 뒤→앞** |

### A-2. 세터가 **전위**일 때
- 세터가 이미 네트 근처 → **침투 불필요, 문제 없음.** 패서는 L+OH 2(또는 로테이션 상황).

---

## 4. 시나리오 B — 우리가 서브할 때 (서브 후 수비 전환)

목표: **상대 공격을 블록 + 빠진 공 디그.**

### B-1. 세터가 **후위**일 때 (핵심 — 사용자 교습)
| 선수 | 자리 | **왜** |
|---|---|---|
| **전위 3인** | **블록 라인**(네트) | **세터가 후위라 블록을 못 뛴다 → 전위 3명이 블록 라인.** x=상대 히터 정면(MB=상대 속공, 윙=핀) |
| **세터(후위)** | **우측 후위 디그** | 블록 못 뛰니 디거. 우측 후위 = 디그 후 바로 세트하러 네트-우 타깃에 가깝게 |
| **리베로+나머지 후위** | 페리미터 베이스(깊은 코너+중앙) | 리베로가 최전방 디펜더 |
| **좌/우 전위** | 기본 블록 준비, 코스 보이면 **팁(페인트) 커버** 풀오프 | 가운데 팁 무방비 방지(룰 48과 연결) |

> **핵심**: 세터 후위 = **전위 3명 블록, 세터가 받는다(디그)**. 상대 셋업 중엔 전위가 리시브 안 함(블록 전념).

### B-2. 세터가 **전위**일 때
- 세터도 전위라 **블록 가능**(3블록에 세터 포함될 수 있음). 후위는 리베로 포함 디그. **[디테일 확인 필요]**

## 5. 자동화 방향 (구현 시 — magic 숫자 제거)

자리마다 숫자를 박는 대신 **역할 → 위치 함수**로:

```
positionOf(player, rotation, phase) =
  role(player, rotation, phase) 에 따라:
    passer            → 레인 x(자기 열) · 패스 깊이 y
    setter(후위·받기)  → 전위 파트너 뒤 → (서브 컨택) 네트-우 침투
    setter(후위·수비)  → 우측 후위 디그
    front 비패서       → 네트 앞 공격/블록 출발점
    blocker(수비)      → 네트, x=상대 히터 정면
    digger             → 페리미터 베이스
```

코드 매핑:
- `receiveFormation` ← 시나리오 A
- `serveFormation`·`servingDefBase`·`switchedSpots` ← 시나리오 B
- `courtDirector` 서브 순간 전환 ← A의 "세터만 침투" 로직

## 6. 검증 (_dv 측정 가드 — 전 로테이션·양 사이드)

구현 후 반드시:
- **A 세터 후위 받기**: 세터가 전위 파트너보다 *깊은가* + 서브 순간 *세터만* 네트로 이동했는가.
- **B 세터 후위 수비**: 전위 3인이 *블록 라인*에 있는가 + 세터가 *우측 후위 디그* 자리인가.
- **오버랩**: 모든 시작 대형 룰 Q 위반 0.
- A/B 자가검증(깬 대형 주입 시 검출).
