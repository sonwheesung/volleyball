# MOCAP_TEST_GUIDE — 비디오 모캡 실현성 테스트 (사용자 직접 실행)

> 목적: 유튜브 1인 스파이크 영상 **한 클립**을 무료 모캡 툴에 넣어 3D 모션(FBX/GLB)을 뽑고,
> "배구명가 3D 경기화면이 현실적인가"의 **실현성만 반나절 안에 판정**한다.
> 비전·근거·엔진 준비도는 자매 문서 [MATCH_3D_VISION](./MATCH_3D_VISION.md).
> **성격: 비상업 기술 테스트.** 앱 미배포·소스 교체 전제(상업화 시 직접 촬영/구매 애셋으로 교체 — VISION §3).
> **모든 무료티어 수치·약관은 2026-07 기준이며 수시 변동 — 시작 직전 각 공식 pricing 재확인.** 미확인 항목은 그렇게 명기.

## 0. 시작 전 3줄 요약

1. **AI는 대신 못 한다**: 계정 가입·영상 업로드·다운로드는 **사용자 본인이 직접** 한다(로그인·업로드 대행 불가).
2. **톱픽 = DeepMotion Animate 3D**(단일 영상 드래그 → FBX, 무료로 제일 쉬움). 공동 후보 = Rokoko Vision(싱글캠 영구 무료 + 내장 발고정 보정).
3. **판정은 스파이크 1클립으로 충분**: 발 미끄러짐·관절 튐·점프 높이·타이밍만 보면 "쓸만/보정필요/폐기"가 갈린다(§4).

### 톱픽 판단 근거

- **첫 테스트는 DeepMotion**: "단일 MP4 드래그 → 처리 → FBX 다운로드"가 가장 단순·문서화 잘 됨. 무료 60초/월로 스파이크 여러 클립 판정 가능.
  단, 무료 티어가 **비상업 전용**이라 딱 이 테스트 용도에만(우리 결정과 정합).
- **바로 이어서 Rokoko Vision 병행 권장**: 싱글캠이 **영구 무료**이고, 무료 Rokoko Studio에 **발고정·리타깃 보정**이 내장 → §4 판정에서 "보정하면 쓸만한가"까지 한 툴에서 확인 가능.
- **GLB로 r3f를 곧장 만져보고 싶으면 Plask**(무료 GLB 직출력). Move.ai는 무료로 시작하기엔 부적합 → 결과 좋을 때 유료 정밀화용.

## 2. 좋은 테스트 영상 고르기 (모캡 성패의 절반)

단일 카메라 AI 모캡은 **사람을 화면에서 얼마나 깨끗이 보느냐**로 품질이 갈린다. 아래를 만족하는 유튜브 클립을 고른다.

### ✅ 좋은 조건
- **1인 단독**(다른 사람이 프레임에 안 겹침) — 배구 **스파이크 연습/드릴/슬로우모션** 영상이 적합한 이유.
- **전신이 프레임 안에** 계속 들어옴(머리~발끝, 점프해도 발이 안 잘림).
- **카메라 고정**(삼각대) · **측면 또는 정면** 고정 앵글.
- **배경 단순**(체육관 벽 등) · 조명 균일 · 선수와 배경 대비 뚜렷.
- **5~15초**(도움닫기→점프→스파이크→착지 한 사이클) · 해상도 720p~1080p면 충분.
- 옷이 몸선을 가리지 않음(펄럭이는 헐렁한 옷·유사색 배경 회피).

### ❌ 나쁜 예 (모캡 깨짐)
- **경기 중계 광각**(선수가 작게·여러 명·공수 전환).
- **카메라가 움직임**(팬/줌/핸드헬드 흔들림).
- 다인·오버랩(블로킹 2~3명 붙는 장면), 네트·기물에 몸 가림.
- 발이 프레임 밖으로 나가는 점프, 역광·어두움.

> 팁: "volleyball spike drill slow motion", "배구 스파이크 연습 슬로우", "spike approach side view" 류로 검색해 **측면 고정·1인·전신** 클립을 고른다.

## 3. 단계별 절차 (DeepMotion 기준, 반나절)

> Rokoko Vision/Plask도 흐름은 동일: 가입 → 영상 업로드 → 처리 대기 → FBX/GLB 다운로드. 화면 명칭만 다름.

### 3-1. 테스트 영상 확보 (사용자 직접)
- 위 §2 기준의 유튜브 클립을 하나 고른다.
- **유튜브 영상 다운로드는 사용자 본인의 몫**(다운로드 방법·도구는 이 문서 범위 밖 — 유튜브 약관·저작권은 본인 확인). 필요한 5~15초만 잘라 MP4로 준비(≤20초).
- 더 확실하게: **직접 스마트폰으로 스파이크 동작을 측면 고정 촬영**하면 저작권·초상권 걱정 없이 최상의 소스(프로덕션에도 이 방식으로 교체 — VISION §3).

### 3-2. 가입 (사용자 직접 — AI 대행 불가)
- deepmotion.com → Animate 3D → 무료(freemium) 계정 가입. **로그인·업로드는 반드시 본인 계정으로** 한다.

### 3-3. 업로드
- 새 Animation Job → 준비한 MP4 업로드. 옵션:
  - **Full Body Tracking** 켬(스파이크는 전신). 손/얼굴 트래킹은 크레딧 추가 소모(1초당 +0.5크레딧)라 이 테스트엔 **꺼도 됨**.
  - "Foot Locking"/지면 보정 옵션이 있으면 켜서 발 미끄러짐 완화(툴마다 명칭 다름).

### 3-4. 처리 대기
- 클라우드 처리(보통 수십 초~수 분). 완료되면 미리보기 재생이 뜬다.

### 3-5. 다운로드
- **FBX** 익스포트(리타깃·Blender 후처리 표준). GLB가 무료로 되면 GLB도(r3f 바로 재생용).
- 파일을 로컬에 저장 → §4 판정.

> **크레딧 아끼기**: DeepMotion 무료는 월 60초. 한 번에 완벽을 노리지 말고, 먼저 **짧은 10초 클립 1개**로 판정 → 감 잡히면 rerun/추가 클립.

## 4. 결과 판정 체크리스트 (스파이크 1클립으로)

미리보기(또는 Blender에 임포트)에서 아래를 본다. 각 항목 상·중·하로 매겨 종합.

| 항목 | 무엇을 보나 | 좋음(쓸만) | 나쁨(폐기 신호) |
|---|---|---|---|
| **발 미끄러짐**(foot sliding) | 착지·도움닫기 때 발이 지면에서 미끄러지나 | 발이 지면에 붙어 고정 | 스케이트 타듯 미끄러짐(가장 흔한 문제) |
| **관절 튐**(jitter) | 팔·다리·척추가 프레임 사이 떨리나 | 매끈한 궤적 | 손목·무릎이 순간 튐/떨림 |
| **손목·발목 꺾임** | 말단 관절이 부자연스레 꺾이나 | 자연스런 각도 | 180° 뒤집힘·과신전 |
| **점프 높이** | 실제 점프만큼 뜨나 | 도움닫기→점프 높이 반영 | 발이 바닥에 붙어 안 뜸/과장 |
| **전체 타이밍** | 도움닫기→점프→타점→착지 리듬이 맞나 | 원영상과 동기 | 동작이 밀리거나 뭉개짐 |
| **스윙 팔 궤적** | 타점에서 팔이 제대로 휘두르나 | 스파이크 스윙 인지 | 팔이 몸에 붙거나 엉킴 |

### 판정 기준
- **쓸만** = 발 미끄러짐·튐이 경미하고 점프·스윙·타이밍이 살아있음 → 파이프라인 실현성 **긍정**. VISION §5 [3]~[7] 승격 검토.
- **보정필요** = 골격 동작은 맞으나 발 미끄러짐/튐이 있음 → **Rokoko Studio 발고정** 또는 **Blender 클린업**(§5)으로 살릴 수 있는지 30분 더 시도 후 재판정.
- **폐기** = 골격이 무너짐(사지 뒤엉킴·타이밍 붕괴) → **영상 문제일 확률 높음**(§2 재점검, 더 깨끗한 측면 고정 클립으로 재시도). 두세 클립 모두 무너지면 → 풀 3D 보류, **2.5D 완충안**(VISION §1.4)부터.

> 한 클립만으로 단정하지 말 것: **최소 2~3개 클립**(각도·선수 다르게)으로 재현되는 패턴인지 본다(N 표본 정신 — 우연 1건으로 판단 금지).

## 5. 다음 단계 (결과 좋으면 — 개략, 후속 과제)

이 테스트가 "쓸만"이면 아래는 **출시 후** 별도 과제로(VISION §5 파이프라인). 지금 하는 일 아님.

1. **FBX 클린업** — Blender에서 발 미끄러짐 보정(foot lock)·지터 스무딩·불필요 프레임 정리. (Rokoko Studio는 이 일부가 무료 내장.)
2. **리타게팅** — 우리 선수 모델(Meshy/Tripo 생성)에 이 모션 입히기. **공통 스켈레톤(Mixamo 규격)**이면 Mixamo 자동 리타깃, 아니면 Blender Auto-Rig Pro/Rokoko.
3. **GLB 익스포트** — 애니 클립 포함 GLB로 번들(모바일 예산: 선수당 ≤1.5만 tri, 본 ≤40).
4. **r3f/native 재생 PoC** — `expo-gl`+`@react-three/fiber`+`drei/native`의 `useGLTF`/`useAnimations`로 클립 재생. **Expo Go 불가 → dev client 필요**(VISION §1.3).
5. **엔진 이벤트 배선** — `engine/events.ts`의 `RallyEvent`(액션+좌표)를 애니 트리거로 매핑(VISION §2). 로직 재작성 아님 — 배선+높이/타이밍 합성.
