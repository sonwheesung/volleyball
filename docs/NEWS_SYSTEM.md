# 뉴스 피드 (News) — 설계/구현 문서 (캡스톤)

> 자동으로 흘러간 리그를, 신문이 대신 이야기해준다(①관전형 전달 + ②서사 종합).
> 매 경기를 안 보는 단장이 "무슨 일이 있었는지"를 아는 수단. **관전형 1순위 산출물.**

## 0. 원칙 (불변)
- **새 저장 없음.** archive(시상)·milestones·hallOfFame·injuries·경기별 생산에서 **순수 파생**(결정론).
- **가짜 드라마 금지** — 기록에 근거한 **사실만**. 변주는 *표현*에서만, *사실*은 절대 지어내지 않는다.
- **중요도(big)** 로 헤드라인/단신 구분. 최신 시즌 우선, 같은 시즌 내 헤드라인 우선.
- 관전형 [[idle-definition]]: 푸시 없음. 켜면 쌓여있는 피드.
- 도메인 용어·규칙은 **출처 확인 후** 구현([[verify-domain-definitions]]) — 추정 금지.

## 1. 문제 진단 (2026-06-21) — 왜 빈약한가

코드 기준(추정 아님)으로 두 가지 결함:

### 1.1 종류가 시즌 경계·장기 누적에 몰려, 초반 피드가 거의 빈다
실시간(현재 시즌)으로 뜨는 건 **부상·사건·구단주 3개뿐**. 우승·시상·기록·HOF는 전부 **시즌을 끝내야**
archive에 쌓여 비로소 나온다. → **첫 시즌 진행 중엔 archive가 비어 부상·사건만 보인다**(실측 증상).
새 플레이어가 첫 인상을 받는 초반이 가장 비어 보이는 역설.

### 1.2 이미 저장된 데이터의 절반 이상을 안 쓴다
`endSeason`이 매 시즌 archive에 저장하지만 뉴스가 **무시**하는 것:

| 저장돼 있는데 미사용 | 묻힌 기사 |
|---|---|
| `awards.titles` 6종 中 득점왕만 사용 | 스파이크·블로킹·서브·디그·세팅·리시브왕 5종 |
| `awards.best7` | "시즌 베스트7 발표" |
| `awards.roundMvps` | 라운드별 MVP |
| `archive.streaks` (연승·연패) | "10연승 파죽지세 / 10연패 수렁" |
| `archive.series` (플옵 W/L) | "리버스 스윕 / 3-0 스윕 / 업셋" |
| `archive.record` (시즌 승패) | "전승 시즌 / 30승 돌파 / 무승 굴욕" |
| `archive.standings` (최종 순위) | "최하위 탈출 / 통한의 2위 / 가을야구 진출" |
| 경기별 생산(`allProdRows`, 미노출) | 트리플 크라운 / 데뷔전 / 커리어하이 |

### 1.3 본문이 분류당 템플릿 1개 → 같은 종류가 이름만 바뀐 복붙
10시즌치 MVP 기사가 사실상 동일 문장. **반복 = 빈약함의 체감 원인.**

## 2. 해결 방향 (3축)

A. **실시간 소재 추가** → 초반·시즌 중 피드를 채운다(§3 실시간 표).
B. **시즌 경계 종류 확대** → 묻힌 데이터를 전부 기사화(§3 경계 표).
C. **본문 풍부화 + 변주 엔진** → 같은 종류라도 최대한 다른 내용(§4). **가장 중요.**

## 3. 기사 소재 카탈로그 (전체)

`kind`(NewsItem.kind) = `champion|award|milestone|hof|injury|scandal|owner|match|debut|streak|standing`
(신규 kind: `match`·`debut`·`streak`·`standing` 추가)

### 3.1 실시간 (시즌 중) — 초반을 채운다

| 소재 | kind | 트리거 | big | 데이터원 |
|---|---|---|---|---|
| **트리플 크라운** | match | 한 경기 **서브에이스≥3 · 블로킹≥3 · 후위공격(백어택)≥3** (KOVO 정의, §5) | ★ | 경기별 ProdLine(`aces·blocks·backSpikes`) |
| **신인/유망주 데뷔전** | debut | 신인의 **첫 선발 출전**(서브교체·가비지 출전 제외) + 그 경기 성적 | 평판높은 기대주=★ | 경기별 선발(`splitLineup`) + ProdLine |
| **한 경기 폭발(커리어하이)** | match | 한 경기 개인 득점이 임계 이상(예: 30+) 또는 자기 최고 경신 | 35+=★ | 경기별 ProdLine |
| **연승 행진** | streak | 팀 시즌 중 연승 ≥ 임계(예: 8, 10) | 10+=★ | 시즌 중 경기 결과(라운드 경계 평가) |
| **연패** | streak | 팀 시즌 중 연패 ≥ 임계 | 10+=★ | 동상 |
| **업셋** | match | 하위팀이 상위팀을 꺾음(순위차/OVR차 임계 이상) | 큰격차=★ | 경기 결과 + 그 시점 순위 |
| **선두 등극·순위 변동** | standing | 라운드 경계에서 1위 등극/추락 등 변동 | 1위등극=★ | 라운드별 순위 |
| 부상(중상·시즌아웃) | injury | seasonInjuryReport (기존) | 시즌아웃=★ | seasonInjuryReport |
| 사건·사고/영구제명 | scandal | seasonScandals/expelled (기존) | ★ | dynamics/expel |
| 간판선수 벤치 술렁 | owner | benchDirectives, pop≥60 (기존) | pop≥78=★ | benchDirectives |

> 라운드 경계 평가: 시즌은 라운드(leg)로 나뉘므로(라운드 MVP 존재), 연승·순위 변동은 **라운드 경계**에서
> 산정해 실시간성 + 결정론을 둘 다 만족(매 경기 재평가 비용·노이즈 회피).

### 3.2 시즌 경계 (종료 후) — 종류 확대

| 소재 | kind | 트리거 | big |
|---|---|---|---|
| 우승 | champion | archive.championId (기존) | ★ |
| **준우승·리버스스윕 피해** | champion | series에서 결승 패/2승후 3패 | 리버스스윕피해=★ |
| 정규 MVP / 챔프 MVP | award | awards.mvp / finalsMvp (기존) | mvp=★ |
| 신인상 / 기량발전상 | award | awards.rookie / mostImproved (기존) | 단신 |
| **기록왕 6종** | award | awards.titles.{scoring,spike,block,serve,dig,set,receive} — **전부** | 단신 |
| **베스트7 발표** | award | awards.best7 (7인 명단) | 단신(내 팀 다수=★) |
| **라운드 MVP 종합** | award | awards.roundMvps | 단신 |
| **시즌 기록** | standing | record: 전승/30승+/한자릿수승/무승 | 전승·무승=★ |
| **순위 서사** | standing | standings: 최하위 탈출/통한의 2위/가을야구(3위내) | 최하위탈출=★ |
| 기록 경신 | milestone | milestones (기존) | m.big |
| 명예의전당 헌액 | hof | hallOfFame (기존) | legend=★ |

## 4. ★ 본문 풍부화 + 변주 엔진 (핵심)

> 목표: **같은 kind·같은 사실이라도 기사 본문이 최대한 다르게.** 단, 사실은 절대 변형 금지.

### 4.1 조립식 본문 (combinatorial body)
본문을 **조각(fragment) 조합**으로 생성한다:
```
body = opener[i] + factCore(불변·사실) + context[j] + closer[k]
```
- `opener`/`context`/`closer`는 각 분류별 **3~5개 풀**. 사실(factCore: 선수·팀·수치·포지션)은 항상 정확.
- 조합 수 = opener×context×closer (예 4×4×4 = **64가지**) → 조각 12개로 분류당 수십 변형.
- 헤드라인도 동일하게 **2~4개 변형 풀**.

### 4.2 결정론적 변주 선택 (replayable variety)
변형 인덱스는 **안정 시드**로 고른다 — 같은 기사는 항상 같은 표현(리플레이 일치), 다른 기사는 다른 표현:
```
seed = hash(season, kind, subtype, playerId|teamId)   // 안정 필드만
i = seed % openers.length
j = (seed / 7)  % context.length      // 독립 스트림(서로 다른 자리수)
k = (seed / 53) % closer.length
```
- **인접 비중복**: 같은 시즌 같은 kind 기사가 여럿이면, 직전 기사와 **같은 opener/closer를 피해** 한 칸 시프트(연속 노출 시 반복 체감 제거).
- 수치·맥락에 따라 **분기 표현**: 압도적(전승)·신기록·근소차 등 사실의 *정도*에 맞는 풀을 선택(예: 30승은 "압도적", 9승은 "분투").

### 4.3 사실 다양화 (본문에 담는 사실 자체를 늘림)
표현뿐 아니라 **담는 사실의 가짓수**를 늘려 정보 밀도를 올린다. 분류별 사용 가능 사실:
- **우승**: 정규 순위·승패·MVP·챔프MVP·플옵 시리즈 결과(스윕/풀세트)·연속 우승 여부.
- **MVP**: 시즌 득점/블록/디그 등 대표 스탯·팀 순위·전년 대비·수상 횟수(통산 N번째).
- **기록왕**: 수치·2위와의 격차·생애 몇 번째 타이틀.
- **HOF**: 통산 스탯 3종·시즌수·소속 구단·영구결번 여부·동일 구단 N번째 헌액.
- **데뷔전**: 드래프트 순번·평판·그 경기 득점/스탯·팀 승패.
- **트리플 크라운**: 세 부문 각 수치(에이스 N·블록 N·백어택 N)·시즌 통산 N번째 트리플크라운.
- **연승/업셋**: 연승 수·꺾은 상대·순위차.

> 원칙 재확인: **사실의 가짓수를 늘리는 것**(밀도)과 **표현을 변주하는 것**(반복 제거)은 둘 다 한다.
> 없는 사실을 지어내는 것(가짜 드라마)은 절대 안 한다.

### 4.4 newsKey 안정화(권장 변경)
현재 `newsKey = season:kind:headline` — 헤드라인이 변주되면 읽음/안읽음 추적이 흔들릴 수 있다.
→ **`season:kind:subtype:playerId|teamId`** 같은 *불변 식별 필드*로 키를 바꾼다(변주는 표현일 뿐 동일 기사).

## 5. 트리플 크라운 — 정의 (출처 확인)

KOVO V-리그 한정 기록(2005-06 도입). **한 경기에서 한 선수가 아래 3개를 각 3회 이상**:
1. **서브 에이스** ≥ 3
2. **블로킹 득점** ≥ 3
3. **후위공격(백어택) 득점** ≥ 3

→ 판정: 경기별 ProdLine에서 `aces≥3 && blocks≥3 && backSpikes≥3`.
(`backSpikes`는 production.ts가 **바로 이 판정용으로** 이미 분리 집계 중. spikes의 부분집합.)
출처: 나무위키 V-리그/기록/트리플 크라운, KOVO 공식 기록 페이지, 뉴시스·더스파이크 기사(2026-06-21 확인).

## 6. 데이터 노출 (구현 전제)

신규 소재 대부분은 **경기 단위 생산**이 필요하나, 지금은 `data/production.ts`의 `allProdRows()`가
내부 캐시로만 존재(미노출). → **경기 단위 셀렉터를 export**한다(엔진 변경 0, 순수 파생):
```ts
// data/production.ts
export function seasonMatchProds(uptoDay): { dayIndex, homeTeamId, awayTeamId, lines: Map<id,ProdLine>,
                                             starters: Set<id> }[]
```
- `starters`는 경기별 `splitLineup`으로 산출(데뷔=첫 선발 판정용).
- 트리플크라운·데뷔·커리어하이·업셋은 이 위에서 파생. 연승·순위는 라운드 경계 결과에서.

## 9. 검증 (구현 후 필수)
`tools/simNews.ts` 확장 — A/B 자가검증 포함([[no-guessing-run-stats]]):
- **매달린 참조 0**: 모든 기사 playerId/teamId가 실제 존재.
- **중복 0**: 같은 (season,kind,subtype,식별자) 기사가 두 번 안 나옴.
- **사실 정합**: 트리플크라운 기사 ↔ 실제 경기 ProdLine 재확인(에이스·블록·백어택 각 3+), 데뷔=첫 선발 맞는지.
- **변주 커버리지**: 장기 시뮬에서 분류별 사용된 opener/closer 분포가 한쪽에 쏠리지 않는지(반복 체감 정량화).
- **결정론**: 같은 시드 → 같은 본문(변주 포함). 표본 충분히(여러 시즌·시드).
