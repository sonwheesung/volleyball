## 0. 왜 트라이아웃 모델인가

- **1년 계약 = 매년 강제 도박.** 다년 보장이 없으니 용병 서사가 매 시즌 리셋된다 —
  "작년의 에이스를 재지명할 것인가, 새 얼굴에 걸 것인가"가 연례 행사가 된다.
- **연봉 고정(전 구단 동일) = 레버는 돈이 아니라 안목.** 부자 구단이 못 사는 구조라
  스카우팅 시스템(스카우터 역량별 정보 공개)이 그대로 트라이아웃의 게임이 된다.
- 매년 새 풀이 유입되므로 **멸종 문제가 구조적으로 해소**된다.

## 1. 연례 트라이아웃 (오프시즌)

- **재계약 우선권(2026-06-11 추가)**: 드래프트 전에 구단이 자기 외인과 갱신 가능(실제 KOVO).
  AI는 "확실한 에이스급(국내 평균+15, 32세 이하)"만 잔류 — 애매하면 새 얼굴 도박. 사용자는
  자동(추천)/재계약/풀로 보냄 3택. 계약은 1년 단위지만 갱신이 쌓이면 한 용병이 3~5시즌 함께 간다.
- **풀 생성**: 매 오프시즌 외국인 후보 10명 시드 생성(FRESH_POOL_SIZE)(`foreign:{season}` 결정론).
  전원 OP 중심(OH 소수). 능력 분포는 국내 상위권~슈퍼스타(도박의 재료 — 대박과 쪽박).
  **전 시즌 외인 중 재참가자 포함**(잔류 의사 — 기록·나이 보존, 재지명 가능).
- **지명 순번 = 추첨**(시드 결정론, 성적 무관 — 현실 그대로. 꼴찌도 1픽을 못 받을 수 있는 비정함).
- **지명**: 팀당 정확히 1명. AI는 니즈+노이즈 추정(draft AI 재활용), 나는 위시리스트
  (`tryoutWish[]` — 드래프트 픽과 같은 UX, 순번에서 뺏기면 차순위).
- **계약**: 1년 고정, 연봉 `FOREIGN_SALARY = 41,000만`(상한 30만 달러 상당) 균일.
- 미지명 외인은 리그를 떠난다(국내 FA 풀·드래프트에 절대 섞이지 않음).

## 2. 캡과 지갑 — 현실 규정 그대로

- **외인 연봉은 샐러리캡 제외**(실제 KOVO와 동일). 모든 캡 합산(payroll)에서
  `isForeign` 제외 → 기존 국내 FA 밸런스 무영향(통합 지점 최소화의 핵심).
  - **표시도 국내 전용(EC-CAP-01, 2026-06-30)**: 캡 비교를 띄우는 **모든 화면**(대시보드·단장실·시즌중이동·FA·계약관리)이
    `!isForeign` 필터를 거친 국내 페이롤을 쓴다. 한때 계약관리만 옳고 나머지 4곳이 외인 포함 전체를 국내 캡과 비교해
    멀쩡한 팀이 허위 빨강(인천 타이드 37.7/35.0)·시즌중 영입 차단까지 났다 → 4곳 수정. 가드 `_dv_capdomestic`.
- 대신 **운영 자금(FINANCE)에서 고정 지출** — 외인 4.1억은 지갑이 감당한다.
  시즌 중 교체 시 잔여 연봉 + 대체 외인 연봉 이중 부담(교체 결정의 재정 무게).
- **현금 게이트(2026-06-14)**: 오프시즌 트라이아웃도 시즌 중 교체와 동일하게 **운영 자금이
  `FOREIGN_SALARY` 이상일 때만** 외인을 영입/재계약한다(`runTryout`의 `myCash` 게이트). 자금이
  바닥난 구단은 그 시즌 **외인 공석**으로 시작 — 망한 구단의 대가이자 안티 과금. 모기업 보전이
  적자를 메우되 외인까지 거저 주지는 않는다(전엔 게이팅 없이 보전으로 무조건 영입 = 과지출,
  `tools/simBrokeSign.ts`로 발견·수정). AI 팀은 모기업 무한 보전(캡만) — 게이트 무관.

## 3. 시즌 중 교체 (가장 극적인 결정)

- 부진·부상 외인을 **퇴출하고 대체 외인 영입**(시즌당 1회). 대체 풀 = 그 해 트라이아웃
  미지명자 중 잔류 희망자(시드 파생, 3~5명 — 등급은 한 단계 아래가 보통).
- 구현: TRANSACTION_SYSTEM 재활용 — release+sign tx, 날짜 인지 명단·리플레이 안전 그대로.
  단 외인 release는 FA 풀로 가지 않고 리그를 떠난다(타 팀이 주울 수 없음 — 현실 규정).
  > **버그 수정(2026-06-25)**: 이 "리그 떠남"은 **두 곳**에서 지켜야 한다 — forward-pass `applyTx`(faAvail)
  > **와** on-demand 셀렉터 `availableFAsOnDay`. 후자에 `!isForeign` 가드가 빠져 있어 `replaceForeign`/
  > 매뉴얼 방출 후 옛 외인이 인시즌 FA 풀(`app/transactions.tsx`)·`signInSeason`으로 샜다(재영입·UI 누수).
  > 셀렉터에 동일 가드 추가로 해소. 가드 `tools/_dv_foreign_fa_leak.ts`(A/B). 아시아쿼터는 `isForeign=true`라 동일 수정이 커버.
- 새 외인은 경기감각 0에서 시작(FORM — 합류 직후 1~2주 무뎌짐, 공짜 디테일).

## 5. ★ 진행 중 세이브 호환 (마이그레이션 불필요 — 자가 치유)

- **시즌 중 세이브**: 트라이아웃은 시즌 경계에서만 발동 — 현 시즌은 아무 변화 없음.
  기존 리플레이(currentDay+results) 그대로 재생(결정론 보존).
- **다음 endSeason에서 자동 합류**: 트라이아웃이 전 구단에 1명씩 배정 → 외인이 0~7명
  어떤 상태든 **한 번의 오프시즌으로 정상화**. 별도 마이그레이션 코드 없음.
- store 신규 필드는 전부 옵셔널+기본값(zustand persist merge가 처리 — 기존 패턴).
- 외인이 이미 있는 팀: 그 외인이 재참가 풀에 들어가므로 재지명하면 그대로 잔류(서사 연속).

## 7. ★ 아시아쿼터 (V2) — 구현 설계 (2026-06-19, 사용자 요청 "풀 구현")

KOVO 여자부: 팀당 **외인 1(OP 트라이아웃) + 아시아쿼터 1 = 수입 2명**. 아시아쿼터는 별도
트라이아웃, 포지션 유연(OH/MB/OP/S), 연봉 더 낮음. 아래는 외인 시스템을 **재사용**하는 안전 설계.

### 7.1 핵심 설계 결정 (경제 무파급의 열쇠)
- **플래그**: 아시아쿼터 = `isForeign:true` **+** `isAsianQuota:true`. → 캡 제외·FA 비대상·면담
  제외가 `!isForeign` 필터로 **자동 적용**(통합 지점 0). 외인 *라이프사이클*만 `!isAsianQuota`로 분리.
- **로스터 16칸 유지(중요)**: 아시아쿼터는 **17번째가 아니라 16칸 중 1칸을 차지**한다(국내 14 +
  외인 OP 1 + 아시아쿼터 1 = 16). → `ROSTER_TOTAL=16`·FA room·드래프트 충원·`positionGap`
  **무변경**(경제 수학 불변 — 검증 리스크 제거). 외인 OP가 이미 16칸 안에 사는 것과 동일 패턴.
- **엔진 재사용**: `resolveTryout`(풀+순번→지명)은 슬롯 무관 — 아시아쿼터 풀로 그대로 호출.

## 8. 인명(이름) 풀 — 확장 + 동명이인 방지(dedup) (2026-06-30 설계·구현)

**문제**: 표시 이름이 작은 고정 배열(`data/names.ts`)에서 선수마다 **독립적으로 인덱스 추첨**(`rng.int`)돼,
한 화면(트라이아웃/드래프트 풀)이나 리그 안에 **동명이인**이 흔히 떴다(예: 파울라×2·말리완×2·자오펀×2 —
실기기 관찰 2026-06-30). 풀이 작을수록(외인 14·아시아 24) 생일역설로 충돌이 거의 보장.

**GitHub 조사 결론(사용자 요청 "괜찮은 이름 생성기 찾아봐")**: philipperemy/name-dataset, sigpwned/
popular-names-by-country, IES-platform/r4r_gender(WGND) 등 국제 인명 데이터셋은 **전부 라틴 문자**(영문/현지
표기). 게임 UI는 전(全) 한글이고 외인은 한글 음역(바네사 등)으로 표기 → **라틴 데이터를 그대로 꽂으면 화면이
어긋난다**. 그래서 이 데이터셋들을 **출처로만** 써서 한글 음역 풀을 대폭 큐레이션(확장)하고, **결정론 dedup**으로
중복을 없앤다(실존 유명선수·셀럽·남성명 회피 규칙 유지 — 메가왓티·옐레나·나탈리아·카리나·드라간 류 금지).

**~~A. 풀 확장(1차 시도 — 폐기)~~** — ~~`FOREIGN_NAMES` 14→90·`ASIAN_IMPORTS` 24→76·`GIVEN` 30→60 큐레이션.~~
**폐기 사유(2026-06-30, 사용자 지적)**: 고정 *완성 이름* 리스트는 아무리 늘려도 **유한 → 100시즌+면 고갈/반복**.
dedup이 "현존 전부 회피"라 풀이 마르면 뽑을 게 없어진다. "무조건 유동적으로(절차적으로) 뽑아야" → 아래 A'로 교체.

**A'. 절차적 음절 조합 생성 (정본)** — 이름을 *완성 리스트*에서 고르지 않고 **음절 빌딩블록을 조합**해 만든다.
공간이 수만 가지라 사실상 무제한(어떤 플레이스루보다 큼) + 은퇴자가 이름을 비워 **영구히 고갈 안 함**.
- **국내**: 성(`SURNAMES` ~60 실성) + 이름 = `GIVEN_SYL1`(~35) + `GIVEN_SYL2`(~20)(가끔 1음절 `GIVEN_SOLO`).
  공간 ≈ 60×35×20 ≈ **42,000**. `genKoreanName(id)`.
- **외국인**: 두 갈래(전수 음절 감사 2026-06-30 후 — 경계 품질). ①열린접두 `FGN_OPEN`(받침無 34)+`FGN_SUFFIX`(27, 75%)
  ②받침접두 `FGN_CODA`(8)+`FGN_CODA_SUFFIX`(자음시작 안전 8, 25%). 공간 ≈ **1,100**. `genForeignName(id)`. 받침+유음/모음
  충돌(실네사·줄르타)·중복음절(바바나·미미라, **재추첨**)·모음 hiatus(미안나)·약접두(프·글)를 제거해 자연스러운 음역만.
  실존선수·셀럽·남성명 회피.
- **아시아쿼터**: 국적(`ASIAN_NATS` 8개국)별 음절 풀 `ASIAN_SYL[nat].a + .b`. 국적 태그 동반. `genAsianIdentity(id)`.
- **결정론 핵심**: 이름은 **id 시드 서브스트림**(`createRng(strSeed('kname:'+id))` 등)에서만 추첨 → makePlayer의
  메인 rng(스탯/재능)와 **완전 분리**. 이름 로직을 바꿔도 스탯/ID 0 영향. 이름은 순수 `f(id)`. 기존 세이브는
  스냅샷에 이름 영속 → 불변.

**B. 결정론 dedup (절차적 재생성)** — `data/seed.ts dedupeNames(players, seedKey, taken)`. 생성 **직후 1회**만
(선수 경력 중 이름 불변). 이미 쓰인 이름이면 **시드를 교란해(`id#k`) 새 이름을 절차적으로 재생성**(유한 리스트
소진 없음 — k가 늘수록 새 조합). 아시아는 이름+국적 묶음 재생성. 순수·결정론(같은 입력·시드 → 같은 결과).
- 적용: `generateLeague`(초기 리그 1배치) · `generateForeignPool`/`generateAsianPool`(fresh + `taken`=현존 외인/아시아) ·
  `generateDraftClass`(클래스 + `taken`=현 국내) · `fillRosters`(충원 + `taken`). dedup은 *현존(living)* 집합만 회피
  하면 됨(은퇴자는 스냅샷서 제거) → 수만 공간에서 충돌 자체가 희박, dedup은 0 보장용.
