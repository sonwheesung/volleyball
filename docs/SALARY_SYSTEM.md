**실제 시장가치 공식(`marketValue`)** — 단위: 만원, `BASE=24000`(~2.4억). 2026-06-17 개편:
**능력(abilityMul)을 주동력으로 가파르게** + 나이는 **연속 serviceFactor**(루키 절벽 제거):
```
marketValue = BASE × abilityMul × serviceFactor(age) × POSITION_MUL × foreignMul × perfFactor
  abilityMul   = clamp(0.35 + (overall-55)/28, 0.35, 2.0)  // 가파름 — 연봉이 OVR을 따라감
  serviceFactor= ≤19:0.58 → 27:1.0(점진) → 31:0.84 → 그 이후 감소  // 22→23 절벽 없음
  POSITION_MUL = S1.12 · OP1.18 · OH1.05 · MB0.92 · L0.82
  foreignMul   = 외국인 1.6
  perfFactor   = 생산 3경기↑일 때 0.8~1.3 (S=세트/경기, L=디그/경기, 그 외=득점/경기), 없으면 1.0
→ clampSalary(MIN_SALARY=3000, 개인상한)  // 캡은 cap.ts
```
> **2026-06-17 개편 동기·측정**: 기존엔 OVR을 "보정만" 써서 연봉↔OVR 상관 r=0.16(거의 무관),
> 루키캡 22→23세 3.5배 절벽으로 같은 OVR 쌍 48.8%가 연봉 2배+ 차이 → "연봉 체계가 없다"는 피드백.
> abilityMul 가파르게 + serviceFactor(절벽 제거)로 **r=0.16→0.47, 2배+ 차이 48.8%→6.9%, 절벽
> 3.5배→1.37배** (tools/simSalaryOvr.ts, N=200리시드·22,400선수). 계약은 여전히 서명 시점 고착
> (노장 과대·신인 저평가 서사 보존) — 다만 능력이 주도해 OVR과 분명히 연동.

> **0장 ↔ FA 충돌 주의(해소됨):** 0장은 "팀 총예산만(느슨), 정식 캡 없음"으로 결정했으나,
> 이후 FA 시스템에서 **정식 샐러리캡이 도입**되었다(`engine/cap.ts`: `LEAGUE_CAP=350000`/35억,
> `MAX_SALARY=80000`/8억, 프랜차이즈 예외 `FRANCHISE_MAX=110000`/11억). 현재 기준은 **정식 캡**이며,
> 연봉 산정 결과는 `clampSalary`로 개인 상한을 적용받는다. 상세는 `docs/FA_SYSTEM.md` 5장.

## 1. 선행 시스템 — 개인 생산 기록 (PRODUCTION)

연봉의 1순위 동력이 "실적"이므로, 경기에서 **선수별 생산을 집계**한다(지금은 팀 OVR 기반
경기라 미집계). 풀 랠리 엔진 전이라 **간이 귀속(attribution)** 으로 시작.

### 1.1 득점 귀속 (간이, 결정론)

경기 시드에서 파생된 RNG로, 득점 1점마다 유형·선수를 귀속한다.

```
유형 분포(placeholder): 공격 0.62 / 블로킹 0.13 / 서브에이스 0.10 / 상대범실 0.15(무귀속)
공격자 선택 = 가중[포지션 공격점유 × skSpike]   (OP 1.0·OH 0.9·MB 0.6·S 0.1·L 0)
블로커 선택 = 가중[포지션 블록점유 × skBlock]   (MB 1.0·OH 0.6·OP 0.6·S 0.3·L 0)
서버 선택   = 가중[skServe]                      (L만 0.1)
```

- **세트(assist)**: 공격 득점마다 팀 세터에게 1 → 세터의 생산 지표.
- **디그(dig)**: 상대 공격 점수에 비례해 후위 수비수에게 분배(L 1.3·OH 0.6·S 0.5·MB 0.4·OP 0.3).
- 포지션마다 의미 있는 실적이 남는다(공격수=득점, 세터=세트, 리베로=디그).

> **KOVO 여자배구 분포 맞춤(2026-06 측정·재확인)**: 득점유형 비율을 엔진 실측에 맞춤
> (공격킬 57.9% / 스터프 9.0% / 에이스 5.7% / 상대범실 27.4%, N=2,571경기·44.4만 랠리 — `tools/simStatRecord.ts`).
> 공격 점유 `ATTACK = {OP 1.38, OH 0.98, MB 0.42, S 0.08}` + 집중도 `ATK_FOCUS=2.0`
> (좋은 공격수에게 세트 몰림 → 1옵션 에이스 부각). 한 시즌 측정: OP(아포짓) 득점 ~29%·톱 ~5.3점/세트(~21/경기),
> OH 좌우 ~47%, MB(센터) ~21%·블록 0.5/세트, 세터 ~12 assist/세트, 리베로 ~4.7 dig/세트 — 실제 여자배구 수렴.
> **⚠ box 경로 재측정 교정(2026-06-24, N=600경기·box 단일화 후·독립세션 drift 검출)**: 위 수치는 **legacy
> production 경로(`simStatRecord`)** — 블록 점유를 여러 포지션에 분산(BLOCK 가중 §1.1)한 값이다. **게임 truth인
> box 경로(2026-06-24 단일화, `rally.ts` 실제 블로커=리드블로커 MB ~97% 귀속)에선 MB 블록 ≈0.98/세트**(문서 0.5 대비
> +96%, STATS_PROTOCOL §3 stale 교정). 세터 어시 11.7≈12·리베로 디그 4.55≈4.7은 box=legacy 일치(1인 포지션).
> ⇒ §1.1의 BLOCK 점유 가중(MB1.0·OH0.6…)·블록 0.5/세트는 **legacy 한정**이고, 게임 표시/시상(블로킹왕)은 box 기준.
>
> **스탯→생산 기울기 검증(252판 측정)**: 능력치 3분위로 분리해 생산이 따라 오르는지 확인.
> OP 스파이크 71→3.8 / 83→4.8, OH 75→3.1 / 89→3.9, MB 스파이크 77→1.5 / 89→1.9, 리베로 디그 78→4.3 / 84→4.7 —
> 모두 단조 증가. **블록은 집중도 `BLK_FOCUS=2.5`** 추가(`BLOCK[pos]×skBlock**2.5`) → MB 블록스킬 72→0.50 / 88→0.73
> 기울기 확보(이전엔 평탄·역전). **세터 assist는 코트 위 세터 1명에게 전량 귀속 → skSet이 아니라 팀 공격량의 프록시**
> (현실의 세터 어시스트와 동일 성질). 세터 스킬은 랠리 엔진의 세팅 승수로 팀 공격 전체에 반영된다.

### 1.2 시즌 집계 (저장 없이 재계산)

- 진화 셀렉터와 동일 철학: **치른 경기(results) + 경기일 로스터 + 시드 → 결정론**.
  선수별 시즌 기록을 저장하지 않고 재계산(캐시).
- `ProdLine = { matches, points, spikes, blocks, aces, assists, digs }`
- 명예의 전당·통산 기록(6장)의 토대도 된다.

### 1.3 통계 단일화 — 스코어박스 = 생산 = 통산 (2026-06-24 ✅ 구현)

> **문제(사용자 보고 "통계는 차이가 한 개도 없어야")**: 같은 경기의 선수별 기록이 **두 곳에서 따로** 계산된다 —
> ① **스코어박스**(`engine/rally.ts` box, 관전 보드·경기 상세가 그리는 기록, `data/matchBox.ts`), ② **생산**
> (`engine/production.ts` `attributeProduction`, 통산·시즌·시상·연봉이 먹는 기록). 둘은 **독립 난수·다른 모델**이라
> 팀 합계만 KOVO로 맞춰져 있고 **선수 개인은 크게 어긋난다**. → "보면서 본 기록 ≠ 영원히 남는 기록".

**측정(`tools/_ev_statsource.ts`, N=2,000경기·32선수, A/B 자가검증 통과 — box vs box=0·box vs prod>0):**

| 통계 | 팀합 차이 | **선수 개인 분기율(\|box−prod\|/box)** |
|---|---|---|
| 디그 | 3.9% | **58.4%** |
| 블록 | 0.2% | **62.6%** |
| 에이스 | 8.0% | **62.6%** |
| 스파이크(킬+블록아웃) | 3.3% | **25.3%** |
| 어시(세트) | 3.3% | **22.0%** |

> 핵심: 팀합은 비슷한데 개인은 22~62% 어긋남. 또 생산은 한 점당 spike=assist=dig를 묶어 셈(192,995 동일)하나
> 박스는 사건별 독립(디그 200,893 ≠ 스파이크 199,553 — 랠리 루프의 비종결 디그까지 포함) → **박스가 사건 단위 진실**.

**단일화 설계(채택 방향)**: **스코어박스(box)를 단일 진실로** 삼고, `attributeProduction`이 자기 난수로
재귀속하는 걸 폐기 → 생산은 **경기별 box를 집계**(matches·garbage·backSpikes·subUse XP 등 생산 고유 오버레이만 유지).
그러면 관전 보드 = 스코어박스 = 통산/시즌/시상/연봉이 **구성상 한 선수도 안 어긋난다**(보드↔박스는 이미 byId/recvId/
setId/touches로 묶는 중 — 그 사슬의 종점을 통산까지 연장).
- **카테고리 정의 통일**: box `atkKill`→spikes, `blockPt`→blocks, `srvAce`→aces, `assist`→assists, `digSucc`→digs,
  `recvAtt`→receives(정의 차 — box는 범실 포함 전 리시브, prod는 패서 픽 → box 기준으로 통일).
- **과거 세이브 불변**: 통산은 시즌 끝에 선수에 박혀 저장(`accrueCareer`) → **앞으로 쌓이는 시즌부터** 새 집계.
  과거 career는 동결(소급 변경 없음). 모델 교체 시점 불연속은 수용(STATS_PROTOCOL — 신규만 새 모델).
- **밸런스 재검증(엄수)**: 연봉·시상 산출이 생산을 먹으므로 KOVO 밴드·생산 기울기(스탯→생산 단조)·시상 분포를
  N≥1만 재시뮬(이 1장 본문 측정 재실행). 팀합은 박스가 이미 KOVO 정렬(`_ev_box_audit`)이라 개인 재분배가 골자.
- **가드**: `_ev_statsource`(분기율 → 통합 후 **0** 목표) + 기존 `_ev_box_audit`·`simKovo`·생산 기울기 측정.

**✅ 구현(2026-06-24)**: `engine/production.ts` `productionFromBox`(box 카테고리→ProdLine 직접 집계 +
matches/backSpikes/subUse 오버레이) + `attributeProduction(…, box?)` 분기. `data/production.ts`가 경기마다
`box`를 채워 넘김 → 전 구단 통산/시즌이 스코어박스를 단일 진실로 집계.
- **검증**: `_ev_statsource`(통합 prod vs box **선수별 0 분기** 5카테고리 전부 · 레거시는 분기 → 도구 민감) ·
  `simAwards`(새 경로 — 디그왕=리베로·세트왕=세터·블로킹왕=MB·득점왕=OH·베스트7 포지션 정상 = 기울기 보존) ·
  `checkRecords`(리더보드 PASS) · 205 테스트 · tsc · auditBoard 0(보드 무관). box가 KOVO 정렬(`_ev_box_audit`)이라
  통산도 자동 정렬.
- **과거 세이브 불변 확인**: career는 시즌말 `accrueCareer`로 동결 저장 → 미래 시즌부터 새 집계(소급 없음).
- **레거시 경로 잔존**: `attributeProduction`의 box 없는 분기는 `simStats`·`simStatRecord` 참조용으로만 유지
  (게임은 항상 box 경로). 추후 그 도구도 box 경로로 옮길 수 있음.

> 보드↔스코어박스 단일화 = `docs/BOARD_RULES.md` 충실도 절. 본 절은 그 한 단계 위 = 스코어박스↔통산 단일화.

## 2. 시장가치 (Market Value) — 계약/재계약 시 산정

*지금 새로 계약하면 받을 값.* **현재 OVR을 직접 쓰지 않고 실적을 쓴다.**

```
실적점수 = wP·정규화(시즌 득점/경기)        // 공격수
         + wA·정규화(세트/경기)             // 세터
         + wD·정규화(디그/경기)             // 리베로/수비
         + 수상보너스(MVP·베스트7·우승)      // 누적 서사 (추후)
marketValue = BASE
            × (0.4 + 0.6 × 실적점수)         // 실적이 주동력
            × ageEarnMul                     // ≤21 0.45 / 24~29 1.0 / 32~33 0.6 / 34+ 0.4
            × positionMul                    // S 1.15 · OP 1.2 · OH 1.05 · MB 0.9 · L 0.8
            × foreignMul                     // 외국인 프리미엄(별도 캡 대신 시장가치로)
            × 협상난수(0.9~1.12)
```

- 신인/무실적은 실적점수 낮음 → marketValue 낮음(루키스케일과 맞물림).
- 잠재력은 어린 선수 한정 소폭 가산(미래 기대).

## 3. 계약 (Contract) — 고착성

```ts
interface Contract { salary: number; years: number; signedAtAge: number; signedSeason: number; }
```

- **연봉 = 서명 시점 marketValue로 고정**, 잔여 기간 불변 → 현재 능력과 갭 발생.
- **루키스케일**: 신인 1~2년차는 marketValue 무시, 낮은 고정 연봉 → 떡상해도 싸게 묶임(꿀).
- **FA**: 계약 만료 시 현재 marketValue로 재계약/이적 → 단장 결정의 무게(pillar 4).
- **시드 생성**: 기존 선수는 "0~3시즌 전 서명"으로 흩뿌려 → 현재 OVR과 연봉이 자연 불일치.

## 4. 팀 총예산 (느슨한 제약)

- 팀마다 **총예산(budget)** 하나. 국내/외국인 구분 없음, 최저연봉 강제 없음.
- 신규 계약은 예산 잔여 안에서. 초과 시 영입 불가/방출 유도.
- (후순위) KOVO식 정식 샐러리캡·외국인 별도 캡으로 강화 가능.

## 6. 로드맵

1. **개인 생산 추적**(1장) — 득점/세트/디그 귀속 + 시즌 집계  ← 지금 여기
2. 시장가치·계약(2·3장) + 시드 연봉 흩뿌리기 + 팀 예산
3. FA/재계약/방출 (단장 업무, Phase 5)
4. 수상·통산 기록(6장) → 실적보너스·명예의전당

## 7. 밸런싱 체크리스트

- [ ] 같은 OVR이라도 신인 ≪ 전성기 ≪ (실적 쌓인) 노장 순으로 연봉이 갈리는가
- [ ] 떡상한 신예가 계약 만료 전까지 "싸게 묶인 꿀선수"로 보이는가
- [ ] 노쇠한 고연봉 노장이 "오버페이"로 부담이 되는가
- [ ] 세터·리베로가 득점은 적어도 세트·디그 실적으로 정당한 연봉을 받는가
- [ ] 외국인 아포짓이 최상위 연봉대인가
- [ ] 팀 예산이 영입 결정을 실제로 제약하는가
