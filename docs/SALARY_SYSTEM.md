## ★ 구현 현황 (2026-06 기준)

| 영역 | 상태 | 실제 파일 / 메모 |
|---|---|---|
| 개인 생산 귀속(1장) | ✅ 구현 | `engine/production.ts`(귀속) + `data/production.ts`(`leagueProduction` 캐시). 명단은 `availableTeamPlayers`(부상·시즌 중 이동 반영, INJURY/TRANSACTION_SYSTEM) — 결장은 생산 기회 손실 |
| 시장가치/계약 고착(2·3장) | ✅ 구현 | `engine/salary.ts` — `marketValue`/`computeSalary`/`contractStatus`/`formatMoney` |
| 루키스케일 | ✅ 구현 | `ROOKIE_CAP=6000`(0.6억), 서명나이 ≤22 적용 |
| 시드 연봉 흩뿌리기 | ✅ 구현 | `data/seed.ts` — 0~3시즌 전 서명으로 분산 |
| 팀 예산(느슨) | ⚠️ 대체됨 | **FA 단계에서 정식 샐러리캡으로 강화**(아래) |
| 수상·실적보너스(2장 일부) | ❌ 미구현 | 시상식 자체는 구현됨(AWARDS_SYSTEM) — 수상→연봉 연동만 미구현 |

**실제 시장가치 공식(`marketValue`)** — 단위: 만원, `BASE=28000`(~2.8억):
```
marketValue = BASE × abilityMul × ageEarnMul(age) × POSITION_MUL × foreignMul × perfFactor
  abilityMul  = clamp(0.5 + (overall-55)/45, 0.4, 1.6)   // OVR을 "보정"으로는 사용(주동력 아님)
  ageEarnMul  = ≤21:0.45 / 22~23:0.7 / 24~29:1.0 / 30~31:0.82 / 32~33:0.62 / 34+:0.42
  POSITION_MUL= S1.12 · OP1.18 · OH1.05 · MB0.92 · L0.82
  foreignMul  = 외국인 1.6
  perfFactor  = 생산 3경기↑일 때 0.8~1.3 (S=세트/경기, L=디그/경기, 그 외=득점/경기), 없으면 1.0
→ clampSalary(MIN_SALARY=3000, 개인상한)  // 캡은 cap.ts
```
> 0장 결정("OVR 직접 안 씀")의 현 구현: OVR은 `abilityMul`로 **보정만** 하고, 실적(`perfFactor`)과
> 나이·포지션·외국인이 함께 결정. 계약은 서명 시점 값으로 고착(`computeSalary`).

> **0장 ↔ FA 충돌 주의(해소됨):** 0장은 "팀 총예산만(느슨), 정식 캡 없음"으로 결정했으나,
> 이후 FA 시스템에서 **정식 샐러리캡이 도입**되었다(`engine/cap.ts`: `LEAGUE_CAP=350000`/35억,
> `MAX_SALARY=80000`/8억, 프랜차이즈 예외 `FRANCHISE_MAX=110000`/11억). 현재 기준은 **정식 캡**이며,
> 연봉 산정 결과는 `clampSalary`로 개인 상한을 적용받는다. 상세는 `docs/FA_SYSTEM.md` 5장.

## 0. 확정된 설계 결정

| 항목 | 결정 |
|---|---|
| 연봉 묶는 방식 | **다년 계약 고착** — 서명 시점 가치로 고정, 잔여 기간 불변(루키스케일·FA 포함) |
| 리그 제약 | **팀 총예산만**(느슨) — 별도 샐러리캡/외국인 캡 없음. 외국인은 시장가치 프리미엄으로만 |
| 실적 반영 | **생산 추적 먼저** 만들고 → 실적 기반 연봉 |

> 왜 OVR ≠ 연봉인가: ① 연봉이 **과거 실적**에 묶임(현재 능력 아님) ② **계약 고착**(서명 시점에 고정)
> ③ **나이/루키스케일**(신인은 떡상해도 싸게 묶임) ④ 포지션/외국인 프리미엄.

## 1. 선행 시스템 — 개인 생산 기록 (PRODUCTION)

연봉의 1순위 동력이 "실적"이므로, 경기에서 **선수별 생산을 집계**한다(지금은 팀 OVR 기반
경기라 미집계). 풀 랠리 엔진 전이라 **간이 귀속(attribution)** 으로 시작.

### 1.1 득점 귀속 (간이, 결정론)

경기 시드에서 파생된 RNG로, 득점 1점마다 유형·선수를 귀속한다.

```
유형 분포(placeholder): 공격 0.62 / 블로킹 0.13 / 서브에이스 0.10 / 상대범실 0.15(무귀속)
공격자 선택 = 가중[포지션 공격점유 × skSpike]   (OP 1.0·OH 0.9·MB 0.6·S 0.1·L 0)
블로커 선택 = 가중[포지션 블록점유 × skBlock]   (MB 1.0·OH 0.6·OP 0.6·S 0.3·L 0)
서버 선택   = 가중[skServe]                      (L만 0.1)
```

- **세트(assist)**: 공격 득점마다 팀 세터에게 1 → 세터의 생산 지표.
- **디그(dig)**: 상대 공격 점수에 비례해 후위 수비수에게 분배(L 1.3·OH 0.6·S 0.5·MB 0.4·OP 0.3).
- 포지션마다 의미 있는 실적이 남는다(공격수=득점, 세터=세트, 리베로=디그).

> **KOVO 여자배구 분포 맞춤(2026-06 측정)**: 득점유형 비율을 엔진 실측에 맞춤(공격킬+블록아웃 62% / 스터프 10% /
> 에이스 6% / 상대범실 22%). 공격 점유 `ATTACK = {OP 1.38, OH 0.98, MB 0.28, S 0.08}` + 집중도 `ATK_FOCUS=2.0`
> (좋은 공격수에게 세트 몰림 → 1옵션 에이스 부각). 한 시즌 측정: OP(아포짓) 득점 ~29%·톱 ~5.3점/세트(~21/경기),
> OH 좌우 ~47%, MB(센터) ~21%·블록 0.5/세트, 세터 ~12 assist/세트, 리베로 ~4.7 dig/세트 — 실제 여자배구 수렴.
>
> **스탯→생산 기울기 검증(252판 측정)**: 능력치 3분위로 분리해 생산이 따라 오르는지 확인.
> OP 스파이크 71→3.8 / 83→4.8, OH 75→3.1 / 89→3.9, MB 스파이크 77→1.5 / 89→1.9, 리베로 디그 78→4.3 / 84→4.7 —
> 모두 단조 증가. **블록은 집중도 `BLK_FOCUS=2.5`** 추가(`BLOCK[pos]×skBlock**2.5`) → MB 블록스킬 72→0.50 / 88→0.73
> 기울기 확보(이전엔 평탄·역전). **세터 assist는 코트 위 세터 1명에게 전량 귀속 → skSet이 아니라 팀 공격량의 프록시**
> (현실의 세터 어시스트와 동일 성질). 세터 스킬은 랠리 엔진의 세팅 승수로 팀 공격 전체에 반영된다.

### 1.2 시즌 집계 (저장 없이 재계산)

- 진화 셀렉터와 동일 철학: **치른 경기(results) + 경기일 로스터 + 시드 → 결정론**.
  선수별 시즌 기록을 저장하지 않고 재계산(캐시).
- `ProdLine = { matches, points, spikes, blocks, aces, assists, digs }`
- 명예의 전당·통산 기록(6장)의 토대도 된다.

## 2. 시장가치 (Market Value) — 계약/재계약 시 산정

*지금 새로 계약하면 받을 값.* **현재 OVR을 직접 쓰지 않고 실적을 쓴다.**

```
실적점수 = wP·정규화(시즌 득점/경기)        // 공격수
         + wA·정규화(세트/경기)             // 세터
         + wD·정규화(디그/경기)             // 리베로/수비
         + 수상보너스(MVP·베스트7·우승)      // 누적 서사 (추후)
marketValue = BASE
            × (0.4 + 0.6 × 실적점수)         // 실적이 주동력
            × ageEarnMul                     // ≤21 0.45 / 24~29 1.0 / 32~33 0.6 / 34+ 0.4
            × positionMul                    // S 1.15 · OP 1.2 · OH 1.05 · MB 0.9 · L 0.8
            × foreignMul                     // 외국인 프리미엄(별도 캡 대신 시장가치로)
            × 협상난수(0.9~1.1)
```

- 신인/무실적은 실적점수 낮음 → marketValue 낮음(루키스케일과 맞물림).
- 잠재력은 어린 선수 한정 소폭 가산(미래 기대).

## 3. 계약 (Contract) — 고착성

```ts
interface Contract { salary: number; years: number; signedAtAge: number; signedSeason: number; }
```

- **연봉 = 서명 시점 marketValue로 고정**, 잔여 기간 불변 → 현재 능력과 갭 발생.
- **루키스케일**: 신인 1~2년차는 marketValue 무시, 낮은 고정 연봉 → 떡상해도 싸게 묶임(꿀).
- **FA**: 계약 만료 시 현재 marketValue로 재계약/이적 → 단장 결정의 무게(pillar 4).
- **시드 생성**: 기존 선수는 "0~3시즌 전 서명"으로 흩뿌려 → 현재 OVR과 연봉이 자연 불일치.

## 4. 팀 총예산 (느슨한 제약)

- 팀마다 **총예산(budget)** 하나. 국내/외국인 구분 없음, 최저연봉 강제 없음.
- 신규 계약은 예산 잔여 안에서. 초과 시 영입 불가/방출 유도.
- (후순위) KOVO식 정식 샐러리캡·외국인 별도 캡으로 강화 가능.

## 6. 로드맵

1. **개인 생산 추적**(1장) — 득점/세트/디그 귀속 + 시즌 집계  ← 지금 여기

## 7. 밸런싱 체크리스트

- [ ] 같은 OVR이라도 신인 ≪ 전성기 ≪ (실적 쌓인) 노장 순으로 연봉이 갈리는가
- [ ] 떡상한 신예가 계약 만료 전까지 "싸게 묶인 꿀선수"로 보이는가
- [ ] 노쇠한 고연봉 노장이 "오버페이"로 부담이 되는가
- [ ] 세터·리베로가 득점은 적어도 세트·디그 실적으로 정당한 연봉을 받는가
- [ ] 외국인 아포짓이 최상위 연봉대인가
- [ ] 팀 예산이 영입 결정을 실제로 제약하는가
