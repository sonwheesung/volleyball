# 구단 정체성 시스템 (CLUB_IDENTITY_SYSTEM)

> CLAUDE.md 2장(데이터 누적 서사)·6장(장기 서사)·7장(단장 의사결정)을 확장한다.
> **목적**: 게임 시작(구단 선택) 시점부터 각 구단이 "역사·서사"를 갖게 한다 —
> 명문/신흥 강호/황혼의 명가/만년 약체/신생팀처럼. 선택 화면에 구단 설명이 뜨고,
> 그 설명에 **선수단 구성이 실제로 맞물린다**(말뿐인 라벨이 아님).

---

## 0. 확정된 설계 결정 (2026-06-18)

| 항목 | 결정 | 이유 |
|---|---|---|
| 역사 깊이 | **구단 프로필**(창단연도·통산우승·전통·최근5시즌 가상순위·한줄서사) | 서사는 충분히 풍부하되 명전/기록/노쇠 등 기존 시스템을 안 건드림 — 안전 |
| 과거 데이터 | **실제 시뮬 안 함.** 프로필 수치는 정체성에서 결정론 생성 | 시작 전 N시즌 시뮬은 명전·영구결번과 깊게 얽혀 파급·검증 비용 큼 |
| 배정 | **고정** — TEAM_NAMES 순서와 1:1, 매 게임 동일 | 현실 리그처럼 "전통의 강호"가 일관 → lore 축적·선택의 무게 |
| 엔진 파급 | **시드 생성 + 표시(UI)만.** 경기/생산/연봉/노쇠 엔진은 정체성을 **모름** | SOLID — 정체성은 "시작 조건"이지 영구 보정이 아님 |
| 영속성 | 정체성은 **시작 조건일 뿐.** 수 시즌 뒤 드래프트·노쇠·FA로 평균회귀 | 만년 약체도 리빌딩으로 반등 가능(parity 보존) |

> **핵심**: 정체성은 두 개의 생성 노브(`strengthBias`·`ageRange`)로만 선수단에 반영된다.
> 나머지(포텐셜·연봉·미래 궤적)는 기존 공식이 **나이·능력에서 파생**한다 — 새 노브 최소화.

## 1. 정체성 7종 (고정, TEAM_NAMES 순서 1:1)

| # | 구단 | key | 라벨 | strengthBias | ageRange | 서사 궤적 |
|---|---|---|---|---|---|---|
| 0 | 인천 타이드 | `dynasty` | 명문 | **+3.5** | 23~33 | 두꺼운 선수층·우승 DNA, 지금도 강함 |
| 1 | 수원 페어리스 | `aging` | 황혼의 명가 | +1.5 | 26~35 | 주축 노쇠 — 강하지만 곧 하락(노쇠 엔진이 끌어내림) |
| 2 | 대전 블레이즈 | `rising` | 신흥 강호 | +2.0 | 19~26 | 젊은 코어+높은 포텐 → 미래가 더 무섭다(성장 엔진이 끌어올림) |
| 3 | 광주 페퍼스 | `cellar` | 만년 약체 | −2.0 | 20~33 | 수년째 하위권, 반등 계기 필요 |
| 4 | 김천 코메츠 | `midpack` | 중위권 | 0 | 21~32 | 꾸준한 중위권, 봄배구 문턱 |
| 5 | 화성 윙스 | `expansion` | 신생팀 | −3.5 | 19~23 | 막 창단, 전력 약하나 유망주로 가득(최고 포텐) |
| 6 | 서울 스파이커스 | `rebuild` | 리빌딩 | −1.5 | 19~28 | 전성기 후 어린 선수로 판 다시 짜는 중 |

> **strengthBias 합 = 0** → 리그 평균 전력 보존(기존 대칭 TIER_SPREAD와 동일 취지).
> **나이가 미래를 만든다**: 노쇠(`aging.ts`)·성장(`progression.ts`) 엔진이 이미 나이에 반응하므로,
> ageRange만으로 "황혼의 명가는 내려가고 신흥 강호는 올라오는" 궤적이 **공짜로** 생긴다.
> 외국인(OP)은 정체성과 무관하게 전성기(24~31)로 영입 — 현실의 용병 수급.

## 2. 구단 프로필(표시 전용) 필드

```ts
interface ClubIdentity {
  key: ClubIdentityKey;       // dynasty | aging | rising | cellar | midpack | expansion | rebuild
  label: string;              // "명문" 등 짧은 칩
  tagline: string;            // "전통의 명가" 등 한 어구
  blurb: string;              // 한 줄 서사(선택 화면·상세)
  foundedYear: number;        // 창단연도(신생=최근)
  titles: number;             // 통산 우승(서사 — 실제 기록 아님)
  tradition: number;          // 0~100 전통/팬덤 척도
  recentRanks: number[];      // 최근 5시즌 가상 순위(1=우승) — 서사 그래프
  strengthBias: number;       // [생성] OVR ± (랜덤 티어 대체)
  ageRange: [number, number]; // [생성] 선수 나이 분포
}
```

- `data/clubIdentity.ts`에 `CLUB_IDENTITIES`(7) + `clubIdentity(teamId)` 셀렉터.
- `recentRanks`·`titles`·`tradition`은 **순수 표시**(엔진 무관). 정체성 서사의 "과거".
- 알려진 단순화: 창단 직후 신생팀도 `clubTenure`는 기존 공식(나이−19)을 따른다 —
  프랜차이즈 표시에 약간의 lore 불일치가 있으나 밸런스 무해(향후 노브 후보).

## 3. 생성 연동 (`data/seed.ts`)

- 기존: 팀별 전력 티어를 `TIER_SPREAD` 대칭 분포로 만들어 **랜덤 셔플** 배정(정체성 없음).
- 변경: 셔플 제거 → 팀 인덱스 `ti`의 `CLUB_IDENTITIES[ti]`에서
  `strengthBias`(→`makePlayer`의 `bias`)와 `ageRange`를 가져와 결정론 적용.
- `makePlayer`에 `ageRange?: [number,number]` 인자 추가(기본 [19,34], 기존 호출 불변).
- 외국인은 `[24,31]`로 오버라이드.

> 결정론·시드 불변. `reseedLeague`(시뮬)에서도 정체성은 인덱스 고정이라
> 모든 유니버스에서 team0=명문 — 단, 수 시즌 평균회귀로 우승은 분산된다(아래 검증).

## 5. 검증 (STATS_PROTOCOL — 추정 금지, 시뮬로 확인)

정체성이 선수단에 **실제로** 맞물리는지 + 기존 밸런스를 깨지 않는지 측정한다.

| 검증 | 도구 | 합격 기준 |
|---|---|---|
| 정체성→선수단 매핑 sanity | `tools/clubIdentity.ts` | 명문>중위>약체 팀OVR 단조, 신생/신흥 평균나이 최저·포텐 최고 |
| 초기 OVR 분포 | `tools/ovrDist.ts` | 표시 신입~70·평균~80·90+ 리그당 ~2 유지(붕괴 없음) |
| 장기 parity | `tools/simLeague.ts 100 30` | 우승 분산·평균회귀 r≈0(명문 영구독주 아님), 만년약체 반등 발생 |
| 재정 | `tools/simFinance.ts` | 영구 침몰 없음, 캡·자금 연출 유지 |
| 무결성 | `tools/qaIntegrity.ts` | 위반 0(로스터·OVR·나이·예산) |

### 측정 결과 (N · 2026-06-18)

| 검증 | 결과 | 판정 |
|---|---|---|
| 매핑 sanity (`clubIdentity.ts`, 리시드 400) | 팀OVR raw **명문 75.5 → 신흥 73.5 → 황혼 73.7 → 중위 71.7 → 리빌딩 70.8 → 약체 69.9 → 신생 68.9** (단조 ✅). 평균나이 신생 21.4 최저·황혼 30.3 최고. 포텐헤드룸 신생 +7.4 최고·황혼 +3.9 최저 | ✅ 라벨대로 |
| 초기 OVR 분포 (`ovrDist.ts`, 리시드 300) | 개인 표시 평균 80.1, 팀 표시 78~90. **90+ 리그당 2.7명**(0명 5%·5+ 13%) — 스펙 "2~3명, 가끔 0/5+" 부합(기존 1.8→2.7, 정체성이 스타를 강팀에 집중) | ✅ |
| 무결성 (`qaIntegrity.ts`, 150시즌) | 위반 0건, OVR 50~94, 나이 18~40 | ✅ |
| 재정 (`simFinance.ts`, 120시즌) | 잔고 평균 8.7억(0~29.5), 영구 침몰 없음, 모기업 보전 5% | ✅ |
| 장기 parity (`simLeague.ts 100 8`) | **순위 지속성 r=−0.10**(≈0, 초기서열 고착 없음)·우승경험 **7/7팀 100%**·꼴찌→우승 **100%**·1위팀 점유 26%·최장왕조 평균 7.4 | ✅ 평균회귀 |

> **결론**: 정체성은 인덱스 고정(모든 유니버스 team0=명문)이지만 **persistence r≈0**으로
> 초기 서열이 고착되지 않는다 — 노쇠가 황혼의 명가를 끌어내리고 드래프트·성장이 신생/리빌딩을
> 끌어올려 모든 팀이 우승을 경험한다(100%). 정체성은 "시작의 색깔"일 뿐 영구 권력이 아니다.
