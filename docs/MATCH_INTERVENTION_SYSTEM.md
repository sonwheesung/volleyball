### 2.2 결과 정합 — 순수 로그 방식 (2026-07-12 조정)

> **정정(2026-07-12, 구현 판단)**: 독립 검증은 ~~"개입 경기 결과+박스를 스냅샷으로 박제"(대안 B)~~를 권고했으나,
> 구현 착수 시점에 **순수 로그(pure-log) 방식**으로 조정한다. 근거:
> - 이 게임은 **이미 재튜닝(ENGINE_VERSION) 시 전 경기를 재시뮬**한다(기존 동작). 스냅샷은 개입 경기만
>   "동결"시켜 나머지 경기(재튜닝 시 드리프트)와 **불일치**를 만들고, 재관전용 로그도 여전히 필요해 코드만 는다.
> - 검증이 우려한 split-brain은 "결과는 저장하되 생산은 재시뮬"하는 **절반 조치**에서 생긴다. 개입 로그를
>   **모든 sim 호출부에 동일하게 실으면**(관전·순위·생산·플옵) split-brain이 **원천 소멸**한다 — 전부 같은
>   입력으로 재시뮬하니 바이트 동일. 이걸 `_dv_intervention_consistency` 가드로 **증명**한다(추정 금지).
> - 남는 한계: 재튜닝 시 과거 개입 좌표가 어긋나 개입이 오발화/소멸할 수 있다. 그러나 **드물고(재튜닝 자체가
>   드묾)·자기일관적(split-brain 없음)·크래시 없음**(좌표 불일치 = 개입 미발화, subIn 가드가 방어). 기존
>   "재튜닝은 전 역사를 재편"이라는 게임 동작과 일관. **스냅샷 동결은 출시 후 "관리한 경기 영구 보존"이
>   필요해지면 하드닝으로 재검토**(post-launch).

**순수 로그 방식:** 개입 로그(§2.1)만 세이브에 영속하고, **경기를 시뮬하는 모든 호출부**(관전 `data/matchBox.ts`,
순위 `data/standings.ts`, 생산 `data/production.ts`, 그리고 플옵 확장 시 `engine/playoffs.ts`+`data/postseason.ts`)가
`interventionsFor(fixtureId)`로 그 경기의 로그를 조회해 `opts.interventions`로 넘긴다. 어느 호출부라도 빠뜨리면
그 경로만 갈라지므로 — **단일 헬퍼 + 정합 가드**로 전 호출부가 로그를 싣는지 박제한다.

### 2.3 캐시 무효화 축 등록 (P0 — 안 하면 순위 스테일)
캐시 키(`standings.ts` / `production.ts`의 `${baseVersion()}:${currentTxVersion()}`)는 개입 로그가 바뀌어도

## 3. 결정론 불변식 — "프리픽스 불변" (반드시 지킴)

독립 검증: 교체가 점수 P에서 일어나면 **P 이전 경기 전개는 바이트 동일**하다. 성립 근거 —
`simulateMatch`은 4개 **독립** RNG 스트림(`rng`·`boxRng`·`digRng`·`cointossRng`, 상태 비공유)을 쓰고,
모듈 전역 가변상태가 없으며, 교체 평가는 랠리 루프 최상단(다음 랠리 전)에서 일어난다. 따라서 P를 만든
랠리는 이미 끝났고 교체는 다음 랠리부터 코트를 바꾼다 → P 이전 네 스트림 전부 불변.

**보드 인터랙션 = 결정 시 전체 재계산**: 플레이어가 P에서 교체를 누르면 로그에 append 후
`simulateMatch(seed, roster, {interventions})`를 **처음부터 재계산**. 프리픽스가 바이트 동일하므로 보드는
P부터 새 전개로 매끄럽게 이어진다. 경기 1회 시뮬은 밀리초라 매 결정 재계산이 저렴.

**성립을 깨뜨리는 3대 함정 (P0):**
1. **자동조종 해제는 forward-only.** "이 경기 유저 지휘 → 전 경기 자동 교체 전역 끔"으로 구현 금지.
   P 이전에 발화했을 자동 피로/핀치 교체가 사라져 프리픽스가 바뀐다. **자동 교체는 항상 켜두고 수동 개입을
   가산**(엔진이 이미 예산·FIVB 재진입 가드로 충돌 처리). 감독 자동 결정은 "그 시점에 유저 개입이 없으면"만 발화.
2. **주입 지점 = 랠리 루프 최상단, 직전 기록 점수를 키로.** 랠리 중간(hop 루프) 금지.
3. **좌표는 경로 의존적.** 개입 P2 좌표는 P1이 반영된 궤적 위에서만 유효 → 로그는 항상 재생 순서대로 전체 재주입.
   로그를 순서 밖에서 편집·검증하는 도구는 깨진다.

**좌표 유일성**: 한 세트 내 점수 (h,a)는 단조증가 격자경로라 각 (setNo,h,a)를 정확히 1회 방문 → 개입 좌표로 유일·안정.

**부상 교체 상호작용**: 중상 코트 교체는 루프 하단(점수 P 기록 후). 같은 점수 P에 수동 교체가 겹칠 때
수동 subIn은 injuryReplaced 슬롯을 거부(기존 엔진). 메인 rng 프리픽스는 불변. UX 순서 규칙만 명세: **부상 교체 우선 처리 후 수동 교체**.

  AI 감독이 이미 자동으로 쓰던 메커니즘을 플레이어 개입 옵션으로 노출. **서브권이 있을 때만** 노출/활성(없으면 즉시
  복원돼 실효 없음 → UI가 서브권 판정(직전 득점 팀=다음 서브)으로 게이팅). `MatchIntervention.subKind: 'manual'|'pinch'`.

### 4.1 타임아웃·교체 완전 수동 — 설정 토글(2026-07-15 사용자 결정)

~~초기 착수 서술: "내 팀 정규시즌은 감독 자동(타임아웃·교체)을 **유지·가산**한다(함정1)"~~ → **정정(2026-07-15, 사용자 결정)**:
내 팀 정규시즌 경기에서 **감독의 자동 개입을 끌 수 있다.** 단 "항상 완전 수동"이 아니라 **게임 설정 "경기 지휘" 토글 파생**이다:

| 설정 "경기 지휘" | 내 팀 정규시즌 경기의 타임아웃·작전교체 |
|---|---|
| **감독 자동(기본)** | 종전 그대로 — 감독이 연속실점 임계로 타임아웃, 작전 4종 자동 교체(현행 동작·바이트 동일) |
| **구단주 직접** | 감독 자동 타임아웃·작전 4종 교체 결정을 **끔** — 세트당 타임아웃 2회·교체 전부 구단주 몫(§1 개입 UI로 직접) |

**엔진 축(`engine/match.ts opts.manualSide?: Side`)**: 지정된 사이드는 아래 결정 블록을 **스킵**한다 —
① 감독 자동 타임아웃(연속실점 임계 `TO_THRESHOLD`) ② 작전 교체 4종 **결정** 블록(피로 rest·핀치 pinch·블로킹 block·수비 def).
미지정(undefined)이면 스킵 판정이 전부 거짓 → **바이트 동일**(새 기본 = 감독 자동). 엔진은 설정을 모른다(순수) — 사이드만 받는다.

**끄지 않는 것(반드시 유지)**: TTO 테크니컬 타임아웃(리그 규정 자동), 부상 교체(`pendingSevere` — FIVB 강제),
**복원 루프**(pinch 서브권 상실 복귀·block 후위행·def 전위행 — 유저가 넣은 pinch도 이 루프로 복귀하므로 필수),
세트말 원복, 개입 적용 블록(§3). 즉 **자동 "결정"만 끄고 자동 "복원·강제·정리"는 유지**한다.

**forward-only 설정(§3 함정1과의 관계)**: 함정1이 금지한 것은 *"이 경기 관전하니 전 경기 자동조종 전역 끔"* 처럼
**관전 여부에 조건부**로 프리픽스를 흔드는 것이다. 이 토글은 다르다 — **경기 날짜(dayIndex) 시점의 설정값**으로만 판정하는
**save-stable 결정론 플래그**다. 설정 변경은 `(day, value)` 체인지로그로 남고 **그 경기가 그날 어떤 설정이었나**만 재생하므로
관전 여부와 무관하고, 프리픽스/순수 로그 정합을 흔들지 않는다. **토글 변경은 이후 경기부터 적용**(과거 소급 재계산 방지 — forward-only).

**적용 경기 = 개입 가능 경기와 정확히 일치**: 정규시즌 내 팀 경기만(§1). 플레이오프(개입 보류)·타팀끼리 경기는 감독 자동 유지.
구현: `data/dynamics.ts manualSideFor(homeId, awayId, dayIndex)`가 "내 팀 경기 여부 × 그 날짜 시점 설정값"으로 판정,

### 4.2 FIVB 15.6.1 — 나간 선발의 타슬롯 재진입 차단(2026-07-15, F2 봉인)

`subIn()`은 IN 후보가 "이번 세트 이미 아웃된 선발"(`usedStarterOut`)인지 검사하지 않아, 나갔던 선발 X가 같은 세트

### 4.3 표시 스테일 카운트 봉인 — 주입 좌표 오프바이원(2026-07-21, 테스터 실보고)

**버그**: 개입 시트의 잔여 교체 예산·교체 후보·핀치 차단 판정이 **같은 데드볼에 방금 커밋한 내 개입을 절대 못 셌다**
(테스터: "남은 5/6에서 3명 교체해도 5/6 그대로"). 원인 = **주입 좌표와 표시 컷오프의 오프바이원**:
- 유저 sub 개입의 `SubEvent.point`는 랠리 루프 최상단 주입이라 `points.length = ptIdx+1`(직전 기록 점수 `points[ptIdx]` 직후 iteration).
- 그런데 표시 산출(`ivBudget`·`benchCands`·`outCands`·`pinchBlock`)의 필터가 `point <= ptIdx`(재생 완료 접두)라 **주입 좌표(ptIdx+1)를 1 차이로 배제**.
- 개입 패널이 열리면 보드 pause로 `ptIdx`가 안 올라 → 연속 교체가 **전부** 미반영. 형제 3곳도 같은 컷오프라 이미 나간 선발이 후보에 잔존·활성 슬롯 재교체 "먹통".

**단순 완화가 금물인 이유(실측)**: 감독 자동 교체(rest/pinch/block/def)도 **같은 iteration에 point=ptIdx+1**로 기록된다(`_tmp_probe` 400/400).
그래서 `point<=ptIdx+1`로 넓히면 **아직 화면에 안 나온 미래 자동 교체까지 미리 세는** 부작용이 있다. 자동 교체는 개입 블록보다 **뒤**에
실행돼(match.ts) 유저 다음 개입의 실예산을 깎지 않으므로, **가산하면 안 된다.**

**수정(순수 로그 방식과 동형)**: 표시 산출을 순수 셀렉터 `data/matchInterventionView.ts`로 추출하고, 컷오프를
**재생 완료(point≤ptIdx) + 아직 재생 안 된 내 지시(pending)** 이중 항으로 바꿨다. pending = 현재 좌표 유저 sub 지시가 실제 적용된
enter(point===ptIdx+1 & inId이 유저 지시 inId) — 자동 교체는 `usedSubIn` 때문에 유저 inId를 재사용 못 하므로 자연 분리된다.
타임아웃은 `TimeoutEvent.point = points.length−1 = ptIdx`(개입 sub와 달리 −1)라 `point≤ptIdx`가 이미 포함 → pending 항 불필요(이중카운트 방지).
injury 축은 유저 개입 무관이라 기존 컷오프 보존.

**동반 발견·수정(코트 모델 dvPhilosophy 불일치)**: `buildMatchBox` 시뮬은 `buildLineup(squad, dvPhilosophy)`인데 보드 `myBaseSix`는
dv를 생략해, **육성철학 높은 감독(실측 dv97) 팀**에서 근소차 슬롯(OP U23)이 어긋나 `applySubsToSix`가 재생하는 코트가 시뮬과 통째로
불일치했다(개입 후보·표시 코트 오염). 보드가 **시뮬과 동일 인자(dvPhilosophy)로 라인업**을 구성하도록 수정.

정본 산식·좌표 규약은 `data/matchInterventionView.ts` 헤더 주석. 사각 분류는 `docs/TEST_METHODOLOGY.md §4`("주입 좌표 ↔ 표시 컷오프 오프바이원").

## 6. 관전형·밸런스 방어

- **관전형**: "기본 자동 + opt-in"이 유효 방어. `opts.interventions`가 비면 현재 코드 그대로. "매 순간 손이 가는"
  유혹은 실재하므로 UI는 개입을 강요하지 않고 조용히 제공(빅매치에서 부각은 선택).
- **밸런스 리스크**: 유저 마이크로관리 > AI면 내 승률이 KOVO 분포/ΔOVR 승률곡선 위로 이탈(왕조 편향).
  인간 개입은 시뮬 곤란 → **출시 후 내 팀 승률 텔레메트리 모니터**. 개입 효과는 엔진 예산으로 상한.
- **butterfly**: 개입이 내 초반 경기 결과를 바꾸면 clinch 러닝순위→타팀 휴식→리그 후속 스쿼드가 바뀐다.
  결정론 문제 없으나 blast radius 큼 → 스냅샷 세트스코어를 러닝 테이블에 정확히 접합하면 통제됨.
