# 경기 개입 시스템 (MATCH_INTERVENTION_SYSTEM)

> 정본. 구단주가 **내 팀 경기 중** 타임아웃·교체를 직접 조작하고, **선발/벤치를 감독 게이트 없이 직접 확정**하는 시스템.
> 설계 결정: 2026-07-12 (사용자·테스터 요구 — "이거 없으면 안 한다"). 5개 병렬 조사 + 독립 검증(표준 작업 순서 1.5) 반영.
> 이 문서가 개입 관련 최종 권위다. 충돌 시 이 문서 우선, 단 게임 두 기둥(관전형·누적서사)·무저장 결정론 제약은 상위.

## 0. 무엇을 뒤집는가 (명시)

이 시스템은 CLAUDE.md에 못 박힌 결정 여러 개를 **의도적으로** 뒤집는다. 안티과금 기둥을 다이아로
뒤집은 전례처럼 **사용자 결정**이며, 취소선 정정으로 보존한다.

- **기둥 #1 관전형** — "조작 아니라 보는 게임". → 유지하되 **"기본 자동 관전 + opt-in 개입"** 예외를 둔다.
  개입 안 하는 유저 경험은 100% 보존(개입 로그가 비면 현재 코드 그대로 = 자동조종).
- **3장 권한표 "타임아웃·교체 = 감독 고유 권한, 월권 금지(🚫)"** → **직접 뒤집힘.** 내 팀 경기 개입 시 구단주 직접.
- **3장 "선발 기용은 건의(감독 수락/거절)"** → 선발/벤치는 **직접 확정**(감독 거부 롤 제거), 단 감독 관계 가벼운 대가 유지.
- **취소선 결정 2건**: 타임아웃 건의 제거(2026-06-16)·작전 토글 제거(2026-06-18) → **재도입.**
- 원설계 `MATCH_SYSTEM §8.3`("중요 경기+플레이어 개입 → 직접 호출·작전")은 06-16/06-18 때 취소선이
  누락된 잔재였고, 이 시스템은 그 원구상의 부활에 가깝다.

**감독의 남은 역할**: 내가 개입하지 않을 때의 자동조종(전 경기 기본값), 경기 스타일(공격/수비/밸런스),
그리고 개입 라인업에 대한 **관계 반응**(언짢음). "감독이 마음에 안 들면 교체"라는 구단주의 칼(기둥 #4)은 유지.

## 2. 아키텍처 — 스냅샷 + 개입 로그

독립 검증 권고(대안 B). 두 개의 저장물:

### 2.1 개입 로그 (input, 재관전 재생용)
플레이어 결정을 per-fixture 작은 로그로 세이브에 영속. 내 팀 경기만이라 bounded(시즌 ~36경기).
선발건의 `benchDirectives`(data/dynamics.ts)와 동형 패턴.

```ts
interface MatchIntervention {
  at: { setNo: number; h: number; a: number };  // 직전 기록 점수 = 주입 좌표(§3 프리픽스 불변)
  kind: 'timeout' | 'sub';
  subOut?: string; subIn?: string;               // kind==='sub'
}
// 세이브: interventions: Record<fixtureId, MatchIntervention[]>   (내 팀 경기만)
```

### 2.2 결과 정합 — 순수 로그 방식 (2026-07-12 조정)

> **정정(2026-07-12, 구현 판단)**: 독립 검증은 ~~"개입 경기 결과+박스를 스냅샷으로 박제"(대안 B)~~를 권고했으나,
> 구현 착수 시점에 **순수 로그(pure-log) 방식**으로 조정한다. 근거:
> - 이 게임은 **이미 재튜닝(ENGINE_VERSION) 시 전 경기를 재시뮬**한다(기존 동작). 스냅샷은 개입 경기만
>   "동결"시켜 나머지 경기(재튜닝 시 드리프트)와 **불일치**를 만들고, 재관전용 로그도 여전히 필요해 코드만 는다.
> - 검증이 우려한 split-brain은 "결과는 저장하되 생산은 재시뮬"하는 **절반 조치**에서 생긴다. 개입 로그를
>   **모든 sim 호출부에 동일하게 실으면**(관전·순위·생산·플옵) split-brain이 **원천 소멸**한다 — 전부 같은
>   입력으로 재시뮬하니 바이트 동일. 이걸 `_dv_intervention_consistency` 가드로 **증명**한다(추정 금지).
> - 남는 한계: 재튜닝 시 과거 개입 좌표가 어긋나 개입이 오발화/소멸할 수 있다. 그러나 **드물고(재튜닝 자체가
>   드묾)·자기일관적(split-brain 없음)·크래시 없음**(좌표 불일치 = 개입 미발화, subIn 가드가 방어). 기존
>   "재튜닝은 전 역사를 재편"이라는 게임 동작과 일관. **스냅샷 동결은 출시 후 "관리한 경기 영구 보존"이
>   필요해지면 하드닝으로 재검토**(post-launch).

**순수 로그 방식:** 개입 로그(§2.1)만 세이브에 영속하고, **경기를 시뮬하는 모든 호출부**(관전 `data/matchBox.ts`,
순위 `data/standings.ts`, 생산 `data/production.ts`, 그리고 플옵 확장 시 `engine/playoffs.ts`+`data/postseason.ts`)가
`interventionsFor(fixtureId)`로 그 경기의 로그를 조회해 `opts.interventions`로 넘긴다. 어느 호출부라도 빠뜨리면
그 경로만 갈라지므로 — **단일 헬퍼 + 정합 가드**로 전 호출부가 로그를 싣는지 박제한다.

### 2.3 캐시 무효화 축 등록 (P0 — 안 하면 순위 스테일)
캐시 키(`standings.ts` / `production.ts`의 `${baseVersion()}:${currentTxVersion()}`)는 개입 로그가 바뀌어도
안 변하면 옛 결과를 반환한다. 개입 등록 시 **`setOwnerContext` 동형으로 `txVersion++` + `recordBump(fixture.dayIndex)`**.
개입은 항상 관전 중(=currentDay) 경기라 minAffectedDay=그 경기일 → forward-only 접미 bump, 기존 스플라이스 불변식과 정합.

## 3. 결정론 불변식 — "프리픽스 불변" (반드시 지킴)

독립 검증: 교체가 점수 P에서 일어나면 **P 이전 경기 전개는 바이트 동일**하다. 성립 근거 —
`simulateMatch`은 4개 **독립** RNG 스트림(`rng`·`boxRng`·`digRng`·`cointossRng`, 상태 비공유)을 쓰고,
모듈 전역 가변상태가 없으며, 교체 평가는 랠리 루프 최상단(다음 랠리 전)에서 일어난다. 따라서 P를 만든
랠리는 이미 끝났고 교체는 다음 랠리부터 코트를 바꾼다 → P 이전 네 스트림 전부 불변.

**보드 인터랙션 = 결정 시 전체 재계산**: 플레이어가 P에서 교체를 누르면 로그에 append 후
`simulateMatch(seed, roster, {interventions})`를 **처음부터 재계산**. 프리픽스가 바이트 동일하므로 보드는
P부터 새 전개로 매끄럽게 이어진다. 경기 1회 시뮬은 밀리초라 매 결정 재계산이 저렴.

**성립을 깨뜨리는 3대 함정 (P0):**
1. **자동조종 해제는 forward-only.** "이 경기 유저 지휘 → 전 경기 자동 교체 전역 끔"으로 구현 금지.
   P 이전에 발화했을 자동 피로/핀치 교체가 사라져 프리픽스가 바뀐다. **자동 교체는 항상 켜두고 수동 개입을
   가산**(엔진이 이미 예산·FIVB 재진입 가드로 충돌 처리). 감독 자동 결정은 "그 시점에 유저 개입이 없으면"만 발화.
2. **주입 지점 = 랠리 루프 최상단, 직전 기록 점수를 키로.** 랠리 중간(hop 루프) 금지.
3. **좌표는 경로 의존적.** 개입 P2 좌표는 P1이 반영된 궤적 위에서만 유효 → 로그는 항상 재생 순서대로 전체 재주입.
   로그를 순서 밖에서 편집·검증하는 도구는 깨진다.

**좌표 유일성**: 한 세트 내 점수 (h,a)는 단조증가 격자경로라 각 (setNo,h,a)를 정확히 1회 방문 → 개입 좌표로 유일·안정.

**부상 교체 상호작용**: 중상 코트 교체는 루프 하단(점수 P 기록 후). 같은 점수 P에 수동 교체가 겹칠 때
수동 subIn은 injuryReplaced 슬롯을 거부(기존 엔진). 메인 rng 프리픽스는 불변. UX 순서 규칙만 명세: **부상 교체 우선 처리 후 수동 교체**.

## 4. 개입 규칙 (엔진 기존 한도 재사용)

- **교체 예산**: 세트당 6회(FIVB), 재진입 금지(usedSubIn/usedStarterOut) — **엔진 기존 로직 그대로**. UI가 남은
  예산을 표시하고 소진 시 버튼 비활성(엔진 no-op을 "먹통"으로 오인 방지).
- **교체 시점**: 데드볼(랠리 사이)만. 랠리 중 금지.
- **서브 교체(핀치, 2026-07-12 사용자 요청)**: 교체 시 "서브 교체" 선택 가능 — 엔진 기존 `kind='pinch'`로 적용돼
  **서브권을 잃으면(공격권이 상대로 넘어가면) 자동복원 루프가 원선발로 되돌린다**(match.ts 261행 `pinch && side !== serving`).
  AI 감독이 이미 자동으로 쓰던 메커니즘을 플레이어 개입 옵션으로 노출. **서브권이 있을 때만** 노출/활성(없으면 즉시
  복원돼 실효 없음 → UI가 서브권 판정(직전 득점 팀=다음 서브)으로 게이팅). `MatchIntervention.subKind: 'manual'|'pinch'`.
- **타임아웃**: 세트당 한도 엔진 기존값, 세트 중 호출 가능.
- **리베로 제외**: 리베로는 `defenders()`에서 후위 MB를 대체하는 파생(six[] 슬롯 아님) → **수동 교체 UI에서 제외.**
- **같은 포지션만 (2026-07-12 사용자 결정)**: 뺄 선수와 **같은 포지션 벤치만** 후보로 표시(OH 자리에 세터가 들어가는
  비현실 라인업 방지). 같은 포지션 벤치가 없으면 후보 0 → "같은 포지션에 투입할 수 있는 벤치 선수가 없어요" 안내.
- **부상/injured 선수 제외**: 교체 대상 목록에서 미리 제외(엔진이 막지만 UX 혼란 방지).
- **밸런스 상한**: 개입 효과는 위 예산 한도로 자연 상한. 상대는 AI 자동이라 비대칭 → §6 텔레메트리 모니터.

## 5. 선발/벤치 직접 확정 (#2)

**"즉시"의 정의**: 현재도 선발/벤치 건의는 수락돼도 **다음 경기부터** 적용(forward-only, 관전 중 경기 소급 금지).
#2는 그 결정을 **감독 거부 롤 없이 직접 확정**하는 것 — 관전 중 경기의 라인업을 바꾸는 게 아니다(그건 #1 인게임 교체).

**변경:**
- 스토어 `suggestBench`/`suggestStart`: 수락 롤 결과를 **무시하고 항상 적용**(`ok=true`, BenchDirective append).
  엔진 `benchAccept`/`startSuggestAccept`/`benchP`/`startP`는 **그대로 둔다**(테스트·아래 플레이버용) — 스토어만 우회.
- 쿨다운(`BENCH_COOLDOWN_DAYS`)·동시 상한(`BENCH_MAX`)은 **유지**(직접이라도 남용 방지 — 매 경기 라인업 난사 차단).
- **카리스마 사문화 방지 (2026-07-12 정정 — 감독 관계 값은 코드에 없음, 신설 안 함)**: 새 감독-신뢰 스탯을
  도입하지 않는다(범위 확대 회피). 대신 카리스마는 **이미 자동조종 층에서 의미를 유지**한다 — 내가 개입하지
  않을 때의 자동 타임아웃(`TO_THRESHOLD[style]`)·자동 교체는 그대로 감독 성향/카리스마가 좌우한다. 라인업
  게이트만 없어질 뿐 감독은 경기 운영에서 여전히 살아있다.
- **결정의 무게 = 팬심·불만 파급 유지**: 스타를 직접 벤치해도 `benchedOn`→출전율→`benchAngerPenalty`(팬 분노)·
  `discontentOf`(선수 불만)는 **그대로 흐른다**. 감독이 안 막아줄 뿐, 대가는 남는다.
- **플레이버(선택)**: `benchAccept`/`startSuggestAccept` 롤을 UI 표시용으로만 계산해 "감독도 동의합니다" vs
  "감독은 내키지 않지만 지시를 따릅니다"를 보여준다(기계 효과 0, 감독 존재감 보존 — 가짜 드라마 아님, 실제 성향 반영).
- 결정론 기계(setOwnerContext+splice)는 불변.

## 6. 관전형·밸런스 방어

- **관전형**: "기본 자동 + opt-in"이 유효 방어. `opts.interventions`가 비면 현재 코드 그대로. "매 순간 손이 가는"
  유혹은 실재하므로 UI는 개입을 강요하지 않고 조용히 제공(빅매치에서 부각은 선택).
- **밸런스 리스크**: 유저 마이크로관리 > AI면 내 승률이 KOVO 분포/ΔOVR 승률곡선 위로 이탈(왕조 편향).
  인간 개입은 시뮬 곤란 → **출시 후 내 팀 승률 텔레메트리 모니터**. 개입 효과는 엔진 예산으로 상한.
- **butterfly**: 개입이 내 초반 경기 결과를 바꾸면 clinch 러닝순위→타팀 휴식→리그 후속 스쿼드가 바뀐다.
  결정론 문제 없으나 blast radius 큼 → 스냅샷 세트스코어를 러닝 테이블에 정확히 접합하면 통제됨.

## 7. 다운스트림 영향 (교체·직접라인업이 흔드는 곳)

박스 파생 통계(득점·블로킹·디그 등)는 엔진이 실제 교체된 코트로 box를 채우므로 **자동 흡수**. 스냅샷이
결과를 박제하므로 재시뮬 경로 대부분이 자동 해소. 남는 것:

**P0 (결과 정합 — 스냅샷으로 해소, 접합 확인 필수)**
- `data/standings.ts` 순위 — 개입 경기는 스냅샷 세트스코어 읽기(러닝 clinch/rest 테이블 접합 정확히).
- `data/production.ts` — 통산/시즌/시상/연봉 단일진실(box). 개입 경기는 스냅샷 박스 읽기.
- `data/matchBox.ts` 관전 보드 — 개입 로그 재생(§3).
- `engine/playoffs.ts` + `data/postseason.ts` buildPlayoffBox — **바이트공유 급소.** 보드·시리즈 점수판이
  같은 스냅샷을 읽어야 `_dv_playoffs` 가드 통과. 통과 못 하면 플옵 개입 폴백(§1).

**P1 (귀속·서사 왜곡 — 손봐야)**
- `engine/production.ts` `matches`/`starters` — `starters`는 splitLineup(OVR 상위 7)이지 실제 코트가 아님.
  수동 교체로 데뷔한 벤치 선수가 `starters`에 안 들어감 → 실제 코트타임 기준으로 보정.
- `data/news.ts` 데뷔 판정 — `starters` 기반. 수동 교체 첫 출전을 데뷔로 볼지 정의.
- `data/awardSalary.ts`·marketVal — production 기반 연봉/시장가(money-path, 특히 검증).
- `data/matchAward.ts` 경기 MVP — box 파생 자동 흡수, 이중 저장 시 확인.

**P2 (자동 흡수 — 확인만)**
- 경기감각 form(`data/dynamics.ts`) — `played`가 buildLineup onCourt 기반(경기결과 무의존) → 인게임 교체가
  form 안 흔듦(격리라 안전).
- 시상 matches>0 필터 — subUse 크레딧으로 이미 충족.
- 중계 현수막·마일스톤·업적·시즌결산 — points[]/box 타임라인 재생성으로 자동 흡수.

## 8. 개발 단계 + 검증 게이트

- **0단계 (문서)** ✅ 이 문서 + 기존 문서 취소선 정정(CLAUDE 3장·10장, MATCH §7.4·8.3·1.3b, OWNER §2.1·4.7,
  SAVE 신규 필드, REALTIME 스냅샷 정책, BOARD_RULES 신규 룰) + README 색인.
- **1단계**: `simulateMatch`에 `opts.interventions` 추가. **비면 완전 무동작 = 바이트 동일** — `_dv` A/B로
  "interventions=[] vs 미지정" 0불일치 증명(회귀 가드).
- **2단계**: 개입 로그 영속 + 캐시 키/스플라이스 등록(§2.3) + 개입 경기 결과·박스 스냅샷(§2.2).
- **3단계**: #2 직접 라인업(accept 롤 우회 + 감독 관계 대가).
- **4단계**: 보드 인게임 교체/타임아웃 UI(데드볼 교체·세트중 타임아웃, 예산 표시, 리베로/부상 제외).
- **5단계 (검증)**:
  - 프리픽스 불변 A/B 가드 — 개입 P 이전 points[] 바이트 동일 (`_dv_prefix`, N≥10,000).
  - 스냅샷==보드 정합 가드 — 플옵 `_dv_playoffs` 동형으로 개입 경기 보드 재생 == 스냅샷 세트스코어.
  - KOVO 분포 재측정 — 개입 없는 시즌이 킬 56%·에이스 6% 등 불변(개입은 유저 선택이라 기저 분포 무영향 확인).
  - 내 팀 승률 텔레메트리 — 개입 유무 승률 이탈 측정.
  - 풀 배터리 0건 + tsc + downstream(뉴스·통계·기록·스코어보드) 정합.

> 검증 안 된 수정은 커밋하지 않고 되돌린다(CLAUDE §11 추정 금지). 각 단계 착수 전 이 문서에 계획이 있는지 확인.

## 9. 신규 가드 (검증 루틴 등록 대상)

- `_dv_prefix` — 개입 P 이전 프리픽스 바이트 불변 A/B.
- `_dv_intervention_empty` — interventions=[] vs 미지정 바이트 동일(1단계 회귀).
- `_dv_snapshot_board` — 개입 경기 스냅샷 == 보드 재생 결과(플옵 포함).
- 기존 `_dv_playoffs`(바이트공유)·`checkSubs`(net-zero 교체회계)·splice/evosig 가드 — 개입 축 반영 후 재통과.
