# 기록 경신 마일스톤 (Milestones) — 설계/구현 문서

> 통산 숫자가 박물관 유물이라면, 마일스톤은 그 유물이 *태어나는 순간*이다.
> "통산 5,000득점"이 데이터라면 "오늘 5,000을 넘었다 — 역대 3번째"는 이야기다(②데이터 누적 서사).

## 0. 범위 (확정: 개인+구단+리그)
| 종류 | 내용 | big |
|---|---|---|
| **개인 통산** | 득점·블로킹·디그·에이스·출전 임계 돌파, 장수(10·15·20·25시즌) | 최상위 임계만 |
| **구단** | 프랜차이즈 통산 1위 등극(득점/블로킹/디그, floor 이상) | 항상 |
| **리그** | 명예의전당 레전드(영구결번급) 통산 추월 | 항상 |

## 1. 작동 (결정론)
- `endSeason` 에서 **before(시즌 시작 base.career) vs after(이번 시즌 생산 누적, seasons+1)** 비교.
- 개인: `personalMilestones` — `CAREER_THRESHOLDS` 임계 교차.
- 구단: 활성 명단+해당 구단 HOF 중 자신 제외 최대(clubMaxOther)를 넘고 floor 이상이 되는 순간. floor(득2000·블700·디2000)로 inaugural 남발 차단.
- 리그: `passedValues` 로 영구결번급 HOF 통산을 추월.

### 1.1 불변식 — 정확중복 금지 (season|playerId|text)
`detectSeasonMilestones` 출력은 같은 `(season, playerId, text)` 조합을 **두 번 담지 않는다.** 시즌 경계마다 유일한 사건이어야 한다.
- **근본 봉인(리그 경로)**: 리그 마일스톤의 기준값 `marks`는 **동점 레전드**(같은 stat에 동일 통산값을 가진 두 HOF)가 있으면
  값이 중복되어, `passedValues`가 같은 값을 2회 반환 → `legends.find`이 매번 첫 레전드를 골라 **동일 push 정확중복**이 났다.
  `marks`를 `[...new Set(...)]`로 값 dedup하여 봉인(삽입순서 보존 = 결정론 유지, 서로 다른 값엔 무영향). `data/milestones.ts`.
- **방어(store)**: `endSeason` 적립 시 `(season|playerId|text)` 키로 정확중복을 한 번 더 거른다(첫 등장 유지 → 과거 시즌 보존, 신규 중복만 제거).
  개인·구단 경로는 임계/선수·stat당 1회라 내부중복이 불가하지만, 전 경로를 방어로 덮는다. big 마일스톤은 영구 미프루닝이라 중복이 누적되면
  장기 세이브에서 위반이 쌓이므로(관측: 시즌 ~246부터 발현), 두 겹으로 차단.

### 1.2 티어드 롤링 캡 (big 초선형 무한증가 봉인 — 2026-07-14)
**문제(실측)**: `store.milestones`가 무한증가한다 — big을 전부 영구보존하던 §2 정책 아래 100시즌 **2249건 → 250시즌 13796건**(초선형).
비-big은 최근 300으로 이미 유계라, 폭증의 주범은 **big**이다. big을 kind별로 뜯어보면 성장법칙이 다르다:

| 티어 | kind | 사건 | 성장법칙 | 캡(최근 N) |
|---|---|---|---|---|
| **T3** | `league` | 명예의전당 레전드 통산 추월 | **시즌² 폭증**(레전드 풀이 매 시즌 커지고 현역이 계속 추월) — 무한증가 주범 | **500** |
| **T1** | `career`(big) | 헤드라인 개인 통산 임계 + 20·25시즌 롱런 | **선형**(현역 수에 비례, 시즌당 대략 상수) | **2000** |
| **T2** | `club` | 프랜차이즈 통산 1위 등극 | **로그 자기제한**(floor·1위 1명이라 자연 포화) — 사실상 유계 | **1000**(안전망) |

- 각 티어 **최근 N건 롤링**(전 시대 헤드라인은 보존, 오래된 하위 사건부터 밀려남). 비-big 300 유지. **총 상한 ~3800건(~2MB), 시즌 수평선과 무관하게 유계.**
- 구현: `endSeason`에서 `allMs`(삽입순서=시즌 오름차순이라 `slice(-N)`="최근 N" 정확)를 kind별 filter+slice로 자른 뒤 season 오름차순 정렬. **결정론·순수 보존**(순서 의존만, 상태 없음).
- 근거: **사용자 결정("역대 상위 N 롤링 캡")** + 독립검토(kind별 성장법칙 진단 — T3가 시즌² 주범, T2는 로그라 안전망만) 반영.

> **⚠ 업적 결합 제약(반드시 준수)**: `engine/achievements.ts`는 `big_milestone_5`(내 팀 big수 ≥5)·`milestones_20`(내 팀 총수 ≥20)·`big_milestone`(내 팀 big 존재)을
> **`milestones` 배열에서 카운트 파생**한다. 현 캡은 넉넉해 내 팀 잔존 big(~380) ≫ 타겟(5·20)이라 단조 회귀(재잠금)가 나지 않는다.
> **그러나 향후 캡을 업적 타겟 근처로 조이려면 반드시 그 카운트를 `careerLog`류 단조 카운터로 디커플한 뒤에 조여라** — 안 그러면
> 프루닝이 카운트를 끌어내려 이미 달성한 업적이 **재잠금(achMono 위반)** 된다. 이 순서를 어기지 말 것.

## 2. 영속 바운딩 (방치형 장기)
~~big(역대·구단·레전드)은 **영구 보존**~~ → **정정(2026-07-14, §1.2)**: big 영구보존은 T3 `league`의 시즌² 초선형 폭증을 유발해 폐기.
big도 **kind별 티어드 롤링 캡**(§1.2: league 500·club 1000·career-big 2000)으로 유계화. 일반 통산 임계(비-big)는 **최근 300건**만(`store.milestones`). 100년+ 운영에서도 총 ~3800건 상한, 저장 폭주 없음.

## 3. 한계 / 추후
- 구단 기록은 "현 명단+HOF 기준 통산 최다" — 은퇴 비-HOF 선수 기록은 추적 누락(근사). 프랜차이즈 영구 최대를 별도 보존하면 정밀해짐.
- 단일시즌 신기록·연속 수상 경신은 추후.
- 동명이인은 이름풀 재사용으로 발생(playerId 기준 식별은 정확).

## 4. 코드 맵
- `engine/milestones.ts` — 순수 임계 감지(`crossedThresholds`/`personalMilestones`/`passedValues`).
- `data/milestones.ts` — `detectSeasonMilestones(season, hof)` 로 Milestone[] (개인+구단+리그, 한국어 문구).
- `store` — endSeason 적립 + 바운딩.
- `app/(tabs)/history.tsx` — 마일스톤 타임라인(최근 40, big 강조).
- `tools/simMilestones.ts` — 통산 누적 포함 N시즌 sanity.

### 신규 개념 = "주장(captain)" 하나뿐 — 설계 검토 필요(review-plan 1.5단계 권장)

`captain`/주장은 코드·타입·엔진 어디에도 없다(추정 아님, grep 대조). 도입 시 확정할 것:
- **선임 규칙**: 누가 주장이 되나(원클럽맨·근속·`isFranchise`·인기 파생?) — **무저장 결정론 파생**으로 뽑을지, 구단주가
  임명하는 **영속 상태**(새 세이브 필드)로 둘지. 후자면 SAVE_SYSTEM 마이그레이션·drift 가드 필요.
- **엔진 무파급 원칙**: 주장이 경기 능력치(rally.ts)에 영향을 주면 밸런스·결정론 관문(simKovo/simLeague)을 새로 타야 함
  → **RELATIONSHIP·CLUB_IDENTITY 선례처럼 "표시·서사 전용, 엔진 무파급"** 으로 좁히는 것을 권고(리뷰 대상).
- **가짜 드라마 금지**: "주장이라서 잘한다"는 없는 인과 금지 — 주장은 서사 라벨이지 성능 레버가 아니다.
- 착수 시 별도 판단 필요 → **표준 작업 순서 1.5단계(독립 리뷰)** 를 태워 결정.

### 문서 배치 결정
본 항목은 **MILESTONE_SYSTEM에 둔다**(선수 커리어 마디의 재조합이 본질 — 통산 출전·구단 1위·리그 추월이 여기 정의됨).
플레이어(구단주) 본인 트로피는 **ACHIEVEMENT_SYSTEM**(별개 레이어), 헌액 연출·번호는 **BROADCAST §8**, 은퇴 세리머니
뉴스는 **NEWS §3.4**가 각각 담당 — 레전드 화면은 이들을 **읽어서 한 인물로 모으는 뷰**이지 새 데이터 원천이 아니다.

### 원칙 합치
- **무저장 결정론**: 주장을 파생형으로 두면 새 영속 0(마일스톤·HOF·clubTenure·jersey 전부 이미 있음). 임명형이면 최소 1필드.
- **관전형·누적 서사**: 레전드는 "세월이 한 사람에게 쌓인 이야기"(기둥 2)의 캡스톤 — AVATAR(얼굴)·헌액과 함께 유대감 자산.
