# 선수 아바타 시스템 (AVATAR_SYSTEM)

> 유대감 강화(독립리뷰 2026-07-04, 4에이전트 합의 1순위 공백)의 첫 조치. "전원 회색 아이콘"이라 정 붙일
> 얼굴이 없던 문제를 해결한다. 관전형·데이터 누적 서사 두 기둥과 정합(얼굴이 있어야 기록·활약이 "정든 개인"에 쌓인다).

## 결정 (2026-07-04, 사용자 선택 B)

- **온브랜드 레이어 아바타** — 외부 라이브러리 대신 `react-native-svg`로 얼굴 레이어를 직접 그린다(LegendIllustration 스타일 재사용).
  - ~~multiavatar/DiceBear 등 생성 라이브러리~~ 반려: DiceBear는 `<style>` CSS라 react-native-svg에서 검은 실루엣, multiavatar는 렌더는 되나 **장난스러운 톤·성별 섞임**(여자 V리그 부적합). 온브랜드가 톤·성별·유니폼을 통제.
- **여자 V리그 톤** — 여성형 헤어스타일 위주, 유니폼(팀색) + 브이넥.
- **id 시드 결정론·무저장** — 같은 선수 = 항상 같은 얼굴. 100시즌+ 세대교체로 이름이 바뀌어도 id 고정. 저장 없이 재계산(기둥 정합).

## 구조

- `data/playerFace.ts` — `faceFeatures(id)`: id+salt 해시로 **피부(5)·헤어색(7)·배경(6)·헤어스타일(5)·눈(3)·입(3)** 인덱스를 독립 추출(런타임 캐시, 저장 X).
- `components/PlayerAvatar.tsx` — `<PlayerAvatar id size jersey trim />`: 피처로 SVG 레이어를 그린다.
  순서: 배경 → 뒷머리 → 목 → 유니폼(어깨)+브이넥 → 머리+귀 → 앞머리(뱅) → 눈썹·볼터치 → 눈 → 입.
  헤어스타일 0 롱·1 포니테일·2 단발·3 올림(번)·4 숏. 유니폼색은 `jersey`/`trim` prop(기본 틸, 팀색 주입 가능).

## 노출 현황 / 향후

- ✅ **선수 상세 히어로**(`app/player/[id].tsx`) — 회색 아이콘 → 아바타.
- ⏳ 향후: 선수단 목록·드래프트·경기 MVP·헌액 등 다른 표면으로 확대(유대감 후속 로드맵의 얼굴 토대).

## 검증

- `tools/_dv_face.ts` — 결정론(같은 id 동일 피처) + 변형(5스타일·5피부·7헤어·6배경 전부 등장·편중<40%). tsc 0.
- 렌더는 react-native-svg 프리미티브(Path/Circle/Ellipse/Rect)만 사용 — 앱 곳곳(Legend/Champion/Award 일러스트·레이더)에서 이미 검증된 경로.

## 원칙 합치

- 관전형: 보는 경험 1순위 투자처와 정합(얼굴은 몰입 자산). 손 안 감·무푸시 무관.
- 누적 서사: 얼굴이 기록의 "그릇" — 통산·마일스톤·헌액이 정든 개인에 귀속.
- 결정론·무저장: id 파생 재계산. 가짜드라마 무관(외형 표시).
