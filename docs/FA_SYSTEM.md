## 1. 단계 ① — 선수 생애 토대 (선행)

FA가 돌아가려면 선수가 빠지고(은퇴) 채워져야(유망주) 한다.

### 1.1 경력 누적
- `career.seasons` 를 롤오버마다 +1. 시드 생성 시 베테랑은 `career.seasons ≈ age - 19`(데뷔 추정).
- FA 자격 타이머의 기준.

- **세이브 호환(오프시즌 재계산)**: 오프시즌 해석은 로드 시 base+seed로 재계산되므로(REALTIME_SIM), 앱 업데이트로 공식/상수가 바뀌면 **진행 중 오프시즌의 은퇴자 명단이 바뀔 수 있다**(엔진버전 게이트 밖). 수용(사용자 결정): 은퇴는 오프시즌 1회 확정 사건이고, 릴리스 사이 진행 중 오프시즌은 드문 경계 케이스 — 별도 게이트 없이 재계산 결과를 채택(관전형 재현 일관성 우선). 시즌 진행 중 확정된 과거 은퇴 기록(`retirements`)은 영속이라 불변.

### 2.4 원소속 우선협상 / 프랜차이즈 ★

**2. 오퍼 시트 프리뷰 (3택 각 옵션 밴드 + 선수 단위 캡션 3분기).** `data/owner.ts resignOptionOutlooks`가 3택 각각을
  override로 `resignOutlookNow`에 넣어 **옵션별 밴드**를 산출(재구현 아님·엔진 위임). 선수 단위 캡션은 불만 topic으로 3분기:

> 사용자 스펙: 재계약 3 프리셋(표준/후하게/짧게)을 **FA식 오퍼 빌더**로 격상 — 연봉 스텝(시장가 배율)·기간 1~5·주전보장 토글.
> **원탭 기본값(표준=시장가·3년·보장off) 유지** — 레버는 옵트인(관전형). 3택 액션시트 → 슬라이더 빌더(`app/contracts.tsx`, `engine/salary.resignSalaryBounds`).

**두 신규 엔진 primitive(`engine/owner.ts`) — `buildOwnerFx`·`resignOutlookNow`가 공유(미리보기=결과 자동):**

1. **저연봉 오퍼 거부 가산 `lowOfferRefuse(salaryRatio, wMoney) = K·wMoney·max(0, R0−ratio)`** (`R0=0.95`·`K=2.0`, 측정 고정).
   - 기존 money 문턱(`discontentOf`: `salaryRatio<0.75 && w.money≥0.25`)과 **독립 가산** — 비-money 아키타입도 `w.money`만큼 반응해
     **'최저가+주전보장' 콤보의 공짜 락업(우회)을 차단**. `ratio = offer.salary / marketVal(p, prod)`(오퍼를 만든 시장가와 같은 기준).
   - **표준(1.0×)·무오퍼는 정확히 0** → `+ 0.0`이라 전후 **bit-동일(0드리프트)**. `ratio ≥ R0`이면 `max(0, 0.95−ratio)=0` (by construction, 측정 표준low≠0=0).
2. **주전보장 완화 `guaranteeRelief(refuseBias) = clamp0(min(0.25, 0.25 − max(0,−refuseBias)))`** — 이번 오퍼의 주전보장이 **출전 불만(minutes)만 완화**.
   - **완화 대상 = `refuseResignProb`의 minutes 기여 + `sustainedBenchRefuse`뿐.** **breach(파기) 항은 절대 완화 불가**(`prob = min(0.95, max(FLOOR, base+accum − relief) + breach + low)` — breach는 완화 뒤 add) → **보장→벤치→재보장 세탁 봉인**.
   - 면담 starter 카드 하향(`refuseBias<0`, −0.18)과 **합산 상한 0.25**(`GUARANTEE_RELIEF_CAP`) — 카드 −0.18이면 완화 0.07(합 0.25). 완화 후에도 `MINUTES_RELIEF_FLOOR=0.05` 잔여 거부(**약속 ≠ 확정 잔류**).

**채널 분리(혼동 금지 — 문서 명기):** **파기 판정 = 기존 계약** `p.contract.starterGuarantee`(이번 시즌의 배신) · **완화 판정 = 이번 오퍼** `override.starterGuarantee`(미래 약속).
  둘은 다른 필드다 — 이번 오퍼로 과거 배신(breach 0.5)을 세탁할 수 없다(가드 ⑤ 세탁봉인 A/B). 롤오버 시 override가 계약에 그대로 반영(`rollover.ts:73`)돼 **이번 오퍼의 보장이 다음 시즌 벤치 시 breach의 출처**가 된다.

**재계약 수락 모델 ≠ FA 채점 모델 (비대칭 정당화).** 재계약은 **잔류(단장 결정)한 만료자를 시즌말 `refuses()` 거부 롤**로 판정(불만·면담·오퍼 레버가 거부 확률을 올리고 내림) — **"떠날 위험"의 모델**.
  FA 영입은 **`offerScore`→`acceptProb` 경쟁 입찰**(여러 팀 중 argmax·롤) — **"끌어올 매력"의 모델**. 둘이 다른 이유: 재계약엔 경쟁 입찰이 없다(내 팀 대 시장 이탈, 이분법) → 점수화가 아니라 **불만 발원 거부 확률**이 자연스럽다. 반대로 FA는 N팀 경합이라 **상대 점수 순위**가 필수.
  같은 레버(주전보장·연봉)가 **재계약에선 거부 확률을 낮추고**(lowOfferRefuse↓·guaranteeRelief), **FA에선 offerScore를 올린다**(playT↑·moneyT↑) — 방향은 대칭이나 **판정 수식이 도메인에 맞게 다르다**(재구현이 아니라 각 도메인 primitive). `engine/owner`(재계약)와 `engine/faMarket`(FA)의 분리가 이 비대칭을 코드로 강제.

### 2.7.1 오퍼 점수 (player, team) ∈ [0,100]
선수별 가중치(`faPref.w` — 2.5b, 합 ~1)로 각 요인 점수를 합산해 100점화. **요인(가산/감점)**:
| 요인 | 방향 | 출처 |
|---|---|---|
| 우승권(전력+prestige) | + | winT (2.5) |
| 연봉(제시/요구) | + | moneyT |
| 장기근속·원소속·프랜차이즈 | + | loyT |
| 주전 보장(포지션 구멍) | + | playT |
| **좋은 팀메이트 / 싫은 선수** | **±** | relT = `teamAffinity`(RELATIONSHIP §2) — 친구 +, 라이벌/앙숙 − |
| 면담(내 팀) | + | talkBias |
```
score = 100 × ( w.money·moneyT + w.win·winT + w.loyal·loyT + w.play·playT + w.home·homeT + w.rel·relT ) + talkBias·k
clamp [0,100]   // relT가 −면 감점(싫은 선수 있는 팀 점수↓). 선수 성향(w)이 "의리파/우승파"를 가른다
```

### 2.7.2 점수 → 확률 (완만한 S곡선 — 사용자 결정)
`P(수락) = smoothstep(SIT_FLOOR, CERTAIN, score)` — `t=clamp((score−FLOOR)/(CEIL−FLOOR),0,1); P=t²(3−2t)`.
- `score ≤ FLOOR`(예 30) → P≈0(거의 거절), `score ≥ CEIL`(예 80) → P≈1(거의 확정), 중간은 확률적(업셋 드라마).
- 높은 점수는 거의 확정·중간만 확률 → 강팀이 대체로 이기되 가끔 업셋(parity 안전).

### 2.7.3 여러 오퍼 해결 (선수 1명 기준)
1. **오퍼 팀 모음**: 그 포지션을 원하고(gap>0 또는 내 지명) **그 순간** 슬롯·캡·자금 가능한 팀만.
2. 각 팀 score→P 계산(점수는 **그 시점 로스터** 기준 — 친구 연쇄 반영).
3. **오퍼 0개 → 미입단**(FA 잔류 — 팀이 안 원해서. 양방향).
4. **최고 점수 < SIT_FLOOR → 시즌 아웃**(잔류, 사용자 결정 — "그럴 바엔 안 뛴다", 드물게).
5. 아니면 **P 내림차순 정렬 → 위에서부터 시드 롤 → 첫 성공 팀 입단**.
6. **전부 실패 → 가장 높았던 P 팀 입단**(fallback).
7. **오퍼 1팀 → fallback상 결국 그 팀**(전부 실패해도 최고=그 팀) → 자동 입단(단 4의 SIT_FLOOR는 적용).

### 2.7.4 엣지 케이스 / 동시성 (순차 결정론으로 해소)
실제 병렬 아님(순수 함수). 위험 = **논리적 동시성**(한 번에 여러 선수·팀, 공유 자원=슬롯·캡·"한 선수 한 팀"). 해법 = **고정 순서 + 매 선수 가용성 갱신**:
| 엣지 | 처리 |
|---|---|
| 같은 선수 ↔ 여러 팀 | 그 선수 1회 해결 → 한 팀만. 나머지 팀은 다음 선수로 |
| 같은 팀 ↔ 여러 선수 | **OVR 내림차순 순차** — 계약 시 그 팀 캡·슬롯 즉시 차감 → 다음 선수는 꽉 찬 팀 제외(이중계약·캡초과 0) |
| fallback 최고 팀이 그새 꽉 참 | 해결 시점 **계약 가능 팀만 재필터** 후 정렬·롤 |
| 친구 연쇄(A 입단→B의 그 팀 점수↑) | 점수를 **그 시점 로스터**로 계산 → "친구 따라간다" 자연 발생. 순서 고정(OVR)이라 결정론 |
| 내 팀 미리보기=결과 | 같은 시드·순서 → FA 센터 예상==실제(기존 보장 유지) |
| 보상선수 | 행선지 확정 후 적용(순서 무관) |
| 로스터 하한 | 시즌 아웃·이탈 누적으로 부족 시 드래프트·신인 충원(ROSTER_MIN 게이트) — SIT_FLOOR 낮게 둬 드물게 |
| 외인/아시아 | 관계·이 모델 비대상(별 트랙) |
| 결정론 | 모든 롤 `seed(playerId:season[:team])` → 같은 세이브 같은 결과(리플레이) |

### 2.7.5 밸런스 관문
확률·시즌아웃은 **변량(업셋)·로스터 구멍**을 만들 수 있어 **parity 측정 필수**(`simLeague` 다중 유니버스 — 도입 전과 동등). 깨지면 S곡선 기울기·SIT_FLOOR·w.rel 하향. KOVO는 경기 무파급(불변). 미리보기=결과·결정론 회귀.

> 구현은 RELATIONSHIP_SYSTEM Phase 2(이 §2.7이 그 phase의 상세). `engine/faMarket.offerScore`→`offerScoreOf(0~100)`+`acceptProb`,
> `data/offseason.resolveFAMarket` 입찰 루프를 argmax→정렬·롤·fallback·SIT로 교체. 계수 전부 placeholder(측정 튜닝).

### 2.7.6 FA 센터 전달(UX) — 지연 구조·실패 사유·보상 개념 (✅ 2026-07-09 구현)

> 설계(경쟁·확률·지연·보상)는 KOVO/§2.2~2.7대로 정상인데 **화면이 그걸 안 알려줘** "이해가 안 됨"(확정 UX 결함).
> 엔진·`resolveFAMarket` 로직은 **불변** — 표시·카피·실패 사유 전달만 보강한다(`app/fa.tsx` + `data/offseason.ts` 관측 추가).

**(1) 지연 구조 1줄 안내(항상 표시)** — FA는 즉시 계약이 아니라 **지명(faSignings) → 시즌 시작 때 경쟁 입찰 해석
(resolveFAMarket) → 선수가 offerScore로 수락**하는 지연·확률 구조다. 상단 배너에 "지금은 '지명' 단계 — 결과는 시즌
시작 때 다른 구단과의 경쟁으로 확정"을 못박아 "영입 눌렀는데 실패"라는 오해를 없앤다. 헤더 카피도 `영입 성공 / 지명`으로,
"질 수도·잔류할 수도·캡/자금 안에서만"을 명시. 탭하면 상세 모달(지연·경쟁·잔류·자금 3경로 설명).

**(2) 실패 사유 세분화** — `resolveFAMarket`이 내가 지명한 FA의 **내 팀 입찰 게이트 결과**(myGate)를 관측해
`FAMarketResult.faFail: Record<id, FAFailCode>`로 노출(`FAPreview.faFail`로 전달). 화면은 '경합/불발' 뭉뚱그림을 제거하고
사유별로 표기:

| 코드 | 게이트(resolveFAMarket) | 화면 문구 |
|---|---|---|
| `ROSTER` | `ROSTER_TOTAL − 로스터 ≤ 0`(입찰 전 컷) | 선수 자리 없음 — 정원 초과 |
| `CAP` | `!canAfford(payroll, offer)`(국내 캡) | 샐러리캡 초과 — 입찰 못 함 |
| `CASH` | `offer + compCost > cashLeft`(운영 자금) | 운영 자금 부족 — 입찰 못 함 |
| `LOST` | 내 입찰 성사(BID) 후 다른 팀이 롤 승 | {팀}에 뺏김 (경쟁 입찰 패배) |
| `SIT_OUT` | 최고 점수 < `SIT_OUT`(0.14) → 선수가 전 오퍼 거절(어느 팀과도 미계약) | ~~{선수} 선수가 잔류를 택함~~ → {선수}, 계약 없이 시즌(무소속) |

> **정정(2026-07-10, 사용자 결정)**: `SIT_OUT`은 "잔류"가 **아니다** — 원소속 재계약처럼 오해된다. 실제 상태 = **어느 팀과도 계약하지 못한
> 무적(무소속) 신분으로 그 시즌을 보냄**(방출로 이미 FA 풀에 나온 선수가 모든 오퍼를 거절해 시즌 아웃). "잔류" 단어를 표면(뉴스·배지·안내)에서 전면 제거.

**우선순위 = 게이트 먼저**: 내 입찰이 아예 안 들어간 경우(ROSTER/CAP/CASH)는 AI가 그 선수를 뽑아 `lostTo`가 찍혀도
'뺏김'이 아니라 **그 사유**로 보여준다 — "자금 없어 입찰조차 못 함 ≠ 경쟁 패배"(옛 화면이 자금부족을 '경합'으로
오표시하던 핵심 결함 수정). 내 입찰이 성사(BID)된 경우만 `lostTo`면 LOST, 아니면 SIT_OUT.

**(3) 보상선수·'돈만' 인라인 설명** — "보상선수란?"/"'돈만'이 뭔가요?" 안내를 모달로 제공: **A·B 등급 FA를 영입하면
원소속팀이 우리 비보호 1명을 데려간다(§2.2~2.3)**, 보호명단(6명) 밖 최고 OVR부터 유출, **'돈만'은 보상선수 없이 돈을 더
내고(A 300%·B 200%) 선수단을 지키는 옵션**, **C 등급은 보상이 없어 토글이 안 뜬다**(왜 A·B만 토글이 뜨는지).

**모달은 전부 커스텀 다크 글래스(`Popup`) — 네이티브 `Alert` 금지(UI-21)**.

> 검증: `tools/_dv_fafail.ts`(신규 상설 가드) — (A) 합성 직접호출로 CASH/CAP/ROSTER 게이트를 강제해 코드 대조 +
> A/B 민감도(자금 뒤집으면 코드 사라짐) + 자금0인데 AI가 데려간 케이스가 CASH 우선(오표시 수정 확인), (B) 자연 장기런에서
> WON(코드 없음·내 팀)·LOST(lostTo=실제 소속)·SIT_OUT(어느 팀에도 없음)을 **관측 가능한 최종 로스터**로 교차검증.
> `_dv_uictx`(등급 pre-FA 무회귀)·`simFaDup`·`simMoneyOnly` 그린 유지, tsc 신규 0.

- **미리보기=결과 일관(하드)**: 만족도의 `OfferCtx` 재료(teamOvr·prestige·posGap·isOriginal·isFranchise·isPreferred·asking·talkBias·relT)는
  화면이 이미 메모한 **`base.off`(pre-FA 스냅샷·로스터)+`base.prestige`+`base.prevTeamOf`+ownerFx+bonds** — `resolveFAMarket`이 읽는 **동일 소스**에서 뽑는다.
  UI가 다른 값을 재계산해 두 숫자가 어긋나면 안 됨. **만족도 표시 경로는 엔진 `offerScore`→`acceptProb`에 그대로 위임**(재구현 금지 — 허위 오라클 방지).
- **결정론 표시용 rand=0.5 고정**: 만족도는 `rand=DISPLAY_RAND(0.5)` 표시값(±0.05·`rand` 항 제거한 중앙값). 실제 시장 해소는 엔진의 순차 `rng.next()` 그대로(만족도는 결과가 아니라 "끌림 정도" 단일 속성).

**① 상세 펼침** — 행의 정보(ⓘ) 아이콘/자세히 토글로 펼침. 안에:
  - **능력**: `deriveRatings(p)`의 6종(스파이크·블로킹·디그·리시브·세팅·서브)을 `StatBar`(reveal 인지)로 + **최근 성적** `p.seasonLines` 최근 1~2줄(재활용, 새 컴포넌트 없음). 전체 상세는 기존 ⓘ→`/player/[id]` 유지.
  - **성향 별점**: `prefWeightsOf(p)`의 5축을 ★1~5로. **축·라벨**: money→"돈", win→"우승", play→"출전", **loyalty→"의리"**(원소속 잔류 성향 — §2.8 원문 "감독" 축은 loyalty 가중이라 선수-대면 정확 라벨 "의리"로 표기), home→"연고". (rel 가중은 이미 행의 친구/라이벌 배지로 노출 — 6번째 별 미표시.)

### 2.8.6 Phase 6 — 선수 역제안 카운터(counterTolerance) + FA 뉴스 강화 (2026-07-10) ✅ (빌드 순서 ⑥ = §2.8.1 "⑥ 카운터(마지막·결정론)")
> §2.8 Phase 4(선수 역제안 1회)·§2.8.1 ①(카운터=결정론 안전형 — 라이브 왕복 금지·`counterTolerance` 사전 커밋)의 엔진+UI 구현.
> **핵심 = 선수가 "조금 더" 요구하면, 내가 미리 정한 한도 안에서 자동으로 받아들이는 사전 커밋 레버.** 실시간 haggling이 아니라
> 순수함수 1패스로 해소돼 preview=result·무저장 결정론이 유지된다. **카운터 시드는 메인 rng 밖**(`facounter:id:season` 해시).

**엔진(`data/offseason.ts resolveFAMarket` — 내 팀 입찰만):**
- `FAOffer.counterTolerance?: { salaryUp: number }`(persisted 입력, salary와 동일 단위 만원). **yearsUp은 v1 제외**(리뷰 기각 — v2에 "요구 조건"형으로).
- `counterAsk = min( round100( asking × (1+δ) ), maxSalaryFor(p) )` — δ는 `facounter:{playerId}:{season}` 문자열 해시 시드(`strSeed`→`createRng`)에서 **0.05~0.25 균등**. **메인 rng 미소비**(traits/injury와 같은 id 시드 서브스트림 패턴).
- **발동 조건(전부 AND)** — 내 팀 BID 성립 && `grade∈{A,B}` && `offerFinal < counterAsk` && `baseScore < CERTAIN(0.60)` && `offerFinal + salaryUp ≥ counterAsk`(**all-or-nothing** — 부분 상향 금지) && 상향액이 캡·운영자금 게이트 통과(`canAfford(payroll, counterAsk)` && `counterAsk + compCost ≤ cashLeft`). 초과면 **미발동·원 오퍼 유지**(myGate 코드 변경 없음 — 원 오퍼가 이미 BID 통과).
  ```
  const rand = rng.next();                 // ← rng 소비를 1회로 hoist(발동 유무 무관·byte-동일)
  let bidOffer = offer, bidScore = offerScore({…offerSalary: offer, rand});   // baseScore
  if (isMe && cfg?.counterTolerance && (grade==='A'||grade==='B')) {
    const δ = 0.05 + createRng(strSeed(`facounter:${id}:${season}`)).next()*0.20;
    const counterAsk = Math.min(round100(asking*(1+δ)), maxSalaryFor(p));
    const up = cfg.counterTolerance.salaryUp;
    if (bidOffer < counterAsk && bidScore < CERTAIN && bidOffer + up >= counterAsk
        && canAfford(payroll[t], counterAsk) && counterAsk + compCost <= cashLeft) {
      bidOffer = counterAsk;
      bidScore = offerScore({…offerSalary: counterAsk, rand});   // ← 같은 rand 재사용
      counterFired[id] = { from: offer, to: counterAsk };        // 선클램프 → to === 최종 계약 연봉
    }
  }
  ```
- **효과**: 오퍼를 정확히 `counterAsk`로 상향 → **같은 rand로** score 재계산 → 정상 롤(자동수락 아님, 경쟁 유지 — 상향해도 라이벌이 먼저 롤하면 뺏길 수 있음).
- **rng 소비 불변(0드리프트 핵심)**: 그 선수·그 팀 판정에서 `rng.next()`(rand)는 정확히 **1회** — tolerance 유무·발동 여부와 무관. baseScore 판정과 발동 후 재계산이 **같은 rand**를 재사용한다. AI는 tolerance 없음(전량 미발동). all-auto·기존 세이브 = **바이트 0드리프트**.
- **관측 필드** `FAMarketResult.counterFired: Record<playerId, { from; to }>`(→ `PreDraft`·`DraftContext`·`FAPreview`로 faCompete와 동일 배선). 순수 관측 — 미발동이면 키 없음. `to === 최종 계약 연봉`(선클램프로 자동, 서명 시). ②용 `faSatOut: string[]`(SIT_OUT + `bids.length>0`인 풀 FA)도 같은 채널로 전달.

**UI(`app/fa.tsx` 오퍼 폼):**
- 연봉 한도 스텝퍼 1개: "선수가 더 요구하면 여기까지 양보"(+0 ~ +1.0억, step 0.1, 기본 +0=미설정). +0이면 `counterTolerance` **undefined**로 저장(기본 오퍼 0드리프트).
- **counterAsk·δ는 절대 은닉**(만족도는 현재 오퍼 기준만 — 카운터 재료 미반영). tolerance 설정 시 정성 캡션만: "요구가 오면 한도 안에서 자동으로 받아들입니다".
- 카드 결과 배지: 프리뷰 `counterFired[id]` 있고 `signedByMe`면 "요구를 수용해 {to}에 계약 예정" 톤.

**뉴스(`data/news.ts`):**

## 2.9 대체 FA 안전망 — 필러 파이프라인 (📋 설계 확정·미구현 2026-07-09)

> 배경: §1.5 가변 로스터 + §1.6 floor 하향 + §1.7 attrition 강화로 **포지션 구멍이 자연 발생**하는데,
> organic FA 풀에 그 포지션 대체급이 없으면 팀이 그 자리를 못 메운다(경기 성립 불가·불변식 붕괴). 이를 막는 **2층 안전망**.
> 가짜 드라마 금지: 필러는 **매시즌 발동하면 floor 튜닝 오류 신호**다. 커리어·수상 0, 밋밋한 배관공 선수.

### 파이프라인 4층 (역할 분담)
`(C) 신인 공급(드래프트 §3)` → `(A) 시장 바닥 공급` → `fillRosters 소비(§1.6)` → `(B) 최후 방어`
필러는 (A)·(B) 두 층이다. (A)는 시장을 조용히 두껍게, (B)는 그래도 뚫리면 마지막에 강제로 막는다.

### (A) 시장 바닥 (organic 부재 시에만 1명 톱업)
- **발동 조건**: organic FA 풀에 **그 포지션 대체급이 하나도 없을 때만** 해당 포지션에 **1명** 주입(항상 스폰 아님 —
  시장에 대체급이 있으면 발동 안 함).
- **스펙**: OVR **raw 58~63(중심 60)** · 나이 **27~31** · **포지션당 최대 1명**. 평범한 즉시전력 백업.
- **비영속 시드**: `fa-filler:{시즌}:{포지션}` — **저장 안 함**, 매 오프시즌 결정론 재파생(리플레이 일치).
- **통계 격리(중요)**: 미서명 필러는 **medianOvr·리그 통계 산정에서 제외**한다 — 미서명 배관 물량이 리그 OVR
  앵커(§1.2 은퇴·연봉·AI잔류의 medOvr)를 오염시키면 안 됨. **뉴스 미노출**(시장 배관이라 서사 아님).
- **서명 시 승격**: 어떤 팀이 이 필러를 서명하면 **신규 고유 id로 승격**(필러 시드 id와 분리 → id 충돌 방지)하고
  **그때부터 실선수**(커리어 누적·은퇴·HOF 파이프라인 정상 진입). 미서명이면 다음 오프시즌 시드 갱신으로 사라짐.

### (B) 최후 강제 주입 (오프시즌 해석의 고정 마지막 순수 스텝)
- **위치**: 오프시즌 해석 순서 `은퇴 → FA → 드래프트 → fillRosters` **뒤의 고정 마지막 순수 스텝**.
- **발동 조건**: 그래도 로스터 < floor(§1.6) **또는 특정 포지션 전멸**인 팀에 강제 충원. **AI 팀 포함 리그 전역**
  (내 팀만이 아니라 전 구단 — 어느 팀도 경기 불가 상태로 시즌 진입 금지).
- **시드**: `fa-filler:{시즌}:{teamId}:{포지션}`(팀 종속 — (A)와 달리 팀에 귀속).
- **인시즌 금지(하드)**: (B)는 **오프시즌에만**. **인시즌 강제 영입은 금지** — base 스냅샷과 어긋나 결정론이 붕괴한다.
  인시즌 부상 구멍은 **floor의 −3 버퍼(주전+1부상흡수, §1.6)** 가 흡수한다.
- **불변식(방어 가드)**: 정상 리그에선 **발동하지 않아야 한다**(`buildLineup` throw-guard와 같은 결 — 뚫리면 버그 신호).
  (B)가 자주 발동하면 floor/attrition/클래스 규모 튜닝 오류 → §E 검증으로 감시.

#### 현행(우리) vs KOVO 대조 — 유일 일치는 "순위 역순 추첨 순번", 나머지 전부 불일치

| 항목 | 현행(우리) | KOVO 실제 | 개정 방향 |

| **팀당 지명 수** | 빈자리 수(보통 0) | **가변**: 매 라운드 지명 or **패스**(실측 팀당 2~4명, 7팀 총 19~21명) | 4라운드 참여, 라운드마다 지명/패스 |

#### 개정 결정

1. **지명 근거 = 유망주 발굴** (로스터 인원 무관). ~~빈자리(16−현재)만큼 뽑는다~~ 폐기. 로스터가 꽉 차 있어도
   미래를 위해 뽑을 수 있고, 그래서 팀당 지명 수가 "빈자리=보통 0"에 묶이지 않는다.
2. **4라운드 고정, 각 라운드 순번대로.** 팀은 매 라운드 **지명하거나 패스**한다(팀당 지명 수 = 가변).
   - **AI 패스 판단**: ①계약 상한(20) 근처 ②원하는 유망주가 남아있지 않음 ③후반 라운드(3R·4R에서 니즈 충족됨).
     → KOVO 실측 팀당 2~4명·7팀 총 19~21명에 수렴하도록 튜닝(§E 검증).
3. **지명 순번 = 순위 역순 가중 추첨**(유지) — **KOVO 실제 확률로 재보정**. 7팀 꼴찌~1위 =
   **35 / 30 / 20 / 8 / 4 / 2 / 1**(합 100) 방향. 현행 `lotteryRound1`(꼴찌 1픽률 25%)을 이 분포로 재캘리브레이션.
   > ※ 리그가 7팀이 아닐 경우(확장) 이 7-슬롯 표를 팀 수에 맞춰 재정규화(합 100 유지)하는 일반식 필요 — OPEN Q.
4. **드래프트 클래스 확대 = ~40명 유망주/년**(KOVO 지원 규모). 현행 소규모 클래스를 확대(§3.3 스카우팅 2.0의
   `makeProspect`/`prospectArc` 분포는 유지, **개수만** 확대 — 밸런스 회귀로 왕조·parity 무영향 확인).
5. **라운드별 루키 연봉 밴드(옵션·기록만)**: KOVO는 1R>2R>3R>4R로 계약 조건이 다르다. 도입 시
   `renewedContract`/신인 초봉 산식에 라운드 계수를 곱한다. **v1은 균일 유지 가능**(연봉 정합 리스크 낮은 쪽 우선).

#### ★ 이 재설계가 "드래프트 표시 버그 3종"을 근본 해소

앞서 확정한 드래프트 UI 버그 3종은 전부 **"빈자리=픽 수" 모델의 증상**이라, 진짜 라운드제 전환으로 **자동 소멸**한다:
- **① 순번이 "타/타/내"로 이상**하게 뜸 → 픽 수가 빈자리라 팀마다 픽 개수가 들쭉날쭉해 순서가 꼬였다.
  4라운드 균일 참여 + 라운드 진행바로 순번이 정직해진다.
- **② 1명만 찜하면 드래프트가 조기 종료** → 내 픽 수 = 내 빈자리라, 찜 1명 소진 시 내 참여가 끝났다.
  매 라운드 참여(패스 포함)로 조기 종료 없음.
- **③ 찜 3명인데 지명권 0** → 로스터가 차 빈자리 0이면 지명권이 0이었다. 유망주 발굴 모델은 로스터 무관하게
  4라운드 지명권을 준다(원치 않으면 패스).

> 위 3종은 미봉책(표시 패치)이 아니라 **모델 교체로 근본 해결**한다는 게 이 개정의 요지(§3.2.1 라이브 재설계와 연동).

### 3.1 AI 픽 우선순위 — 3티어 (2026-06-25 개선)
> "OVR만 높다고 뽑지 않는다." AI는 자기 팀 사정에 맞춰 뽑되, 세대급 재능은 예외로 무조건 잡는다.
> `engine/draft.ts` `pickWithReason` — 순수 함수, 결정론. 픽마다 **사유(reason)** 를 함께 반환(라이브 콘솔 표시용).

순번이 오면 남은 유망주 중에서 **아래 순서로** 고른다:

1. **슈퍼 유망주 예외 (BPA, reason=`super`)** — 남은 후보에 **특급 유망주**(`prospectValue ≥ SUPER_PV=81`)가
   있으면 **포지션 필요와 무관하게** 그중 최고를 잡는다(가치=`prospectValue`×감독성향×스카우트).
   세대급 재능은 자리가 차 있어도 놓치지 않는다(현실 BPA).
   > **컷 정의 주의**: maxPot(스탯 14개 중 최대) 단독은 **포화**(클래스 71%가 ≥88 — 누구나 한 스탯은 99 근처)라
   > "특급"을 못 가린다. **드래프트가치(`prospectValue`=현재OVR×0.4+maxPot×0.6)** 로 정의 → 클래스 **상위 ~10%**
   > 만 특급(측정 `_tmp_pv`: pv≥80=18%·≥82=3%, median 75.6). 라이브 콘솔로 "전부 특급 BPA"를 잡아 교정(2026-06-25).
2. **포지션 필요 (need, reason=`need`)** — 특급이 없으면, **부족 포지션(gap>0)** 후보만 보고 그중 최고
   (`prospectValue`×`needWeight(gap)`×감독성향×스카우트). 자리가 남는 포지션은 이 단계에서 안 뽑는다(`aiFillFromPool`과 동일 정책).
3. **필요 없음 → OVR + 성격 (best, reason=`best`)** — 부족 포지션이 하나도 없으면(이상 구성 충족),
   **현재 실력(OVR)과 성격**으로 고른다(`overall`×`personalityFactor`×감독성향×스카우트).
   - **성격(personalityFactor)** = 멘탈(집중·기복·VQ 평균) + 특성(긍정 가산/부정 감산). 범위 ~0.85~1.15(소폭).
     리더·클러치·노력형 같은 좋은 성격은 동급에서 우대, 새가슴·유리몸은 기피. *placeholder — 측정으로 튜닝.*

> `prospectValue`(포텐 비중 0.6)는 1·2티어(육성 베팅)에, `overall`(현재 실력)은 3티어(즉시 전력+성격)에 쓴다.
> 인간 단장의 위시리스트(reason=`wish`)는 모든 티어보다 우선(순번 내에서 가능한 첫 위시).
> 측정 가드 `tools/_ev_draftpick.ts`(슈퍼 무조건 선발·필요 포지션 우선·무필요 시 성격 영향·결정론·A/B).

### 3.3 아마추어 성적표 — 스카우팅 2.0 (📋 설계 확정 2026-07-03 · GPT+Claude 합의 완료 · **구현 착수 신호 대기** · 코드 미착수)
> **사용자 결정(설계 방향)**: 현실 스카우팅처럼 유망주마다 **고교/대학(아마추어) 성적표**를 보여주되, **성적이 진짜 실력·천장을 정확히 말해주지 않게**(노이즈) 해서 발굴 도박·반짝 함정·숨은 보석을 만든다. 독립 리뷰(general-purpose, N=20,000 실측) + GPT 검토 + Claude 보강으로 **최종 합의(A/95점)**. **이 절이 곧 빌드 스펙 — 사용자 "시작" 신호에 착수.**
> **최종 목표(한 줄)**: 드래프트를 **"정답 맞추기"가 아니라 "정보를 해석하는 과정"** 으로. 성적=즉시전력 신호, 스카우터=미래를 조금 더. **끝까지 100% 정답은 없다** — 그 불확실성이 관전 재미·서사.

**핵심 판타지(왜)**: ① **반짝(flash)** — 아마추어 성적은 좋은데 포텐 낮은 유망주(뽑으면 bust). ② **숨은 보석(gem)** — "유망주 소리도 안 듣던 선수를 순위 높아 아무나 뽑았는데 포텐이 엄청 높은". 성적만 보면 둘을 못 가린다 → 천장은 **시즌이 지나며** 드러나는 도박(관전형·누적 서사 보상). 실측: `corr(현재OVR, maxPot)=0.51`, 중앙값 4분면에서 flash·gem 각 ~16%(극단 각 ~2%).

> **★ 사용자 정제(2026-07-03) — 리뷰의 "corr 0.51 유지"와 갈림**: 원하는 건 **"포텐은 현재(=성적)에 대체로 비례하되 *가끔* 이상치(대기만성·반짝)"** 다. 즉 **대부분은 예측 가능**(좋은 성적=대체로 좋은 포텐 → 스카우팅 판단이 대체로 보상됨)하고, **극단은 드물게(예: 5~10%)** 나와야 기억에 남는 서사가 된다. 현행 0.51(flash/gem 각 ~16%=꽤 잦음)은 "가끔"보다 흔함 → **목표: `corr(현재,포텐)`을 더 높이고(예 ~0.7, 대체로 비례) + 소수 이상치를 heavy-tail로 명시 주입**(대기만성=현재 낮은데 포텐 top / 반짝=현재 높은데 포텐 낮음). **이건 `makeProspect`의 talent→포텐 링크 조정 + 이상치 주입이라 리뷰의 "makeProspect 손대지 마라"와 상충** → 검증(GPT+사용자) 때 **목표 corr·이상치 비율**을 확정하고, 아래 검증 밴드(역산불가 corr 밴드)는 그 결정에 맞춰 재설정. **성적 자체는 여전히 현재+노이즈(포텐 항 0)** — 바뀌는 건 "현재↔포텐 분포"뿐이다.

**설계 원칙(리뷰 확정)**:
1. **성적 = 현재 윗단 스탯(포지션별 생산) × 학교강도 인플레(id 시드) × 노이즈.** **포텐 항 0 — 순수 현재 기반.** 포텐을 성적에 섞으면 gem이 좋은 성적을 내 정체가 드러나 도박이 붕괴(채택 금지). 현실 아마추어 박스도 "한 것(생산)"을 재지 "천장"을 안 잰다.
2. **약체 리그 인플레**: 아마추어는 약한 상대끼리 → **절대수치를 프로 규범 위로** 민다(공격성공 프로 40~55%→아마 55~65%·에이스↑). realism 향미 + **프로 규범에 앵커한 순진한 플레이어를 낚는 함정**. 학교/리그 강도 라벨("2부 대학 리그")로 서사 정당화. **은폐 주력은 노이즈, 인플레는 향미/함정**(대체로 순위 안 바꿈).
3. **노이즈(제일 중요)**: **곱셈 ±15%, `id × 스탯카테고리`별 독립**(단일 배수면 카테고리 비율 보존돼 삼각측량 가능 → 금지), **스카우터·팀·reveal 무관(seed=id만)** — 플레이어·AI 동일 조건, 스카우터로 성적 노이즈 못 지움. 목표: `corr(성적,현재)≈0.5`(역산 불가·현재분산 ~25%만 설명)·`corr(성적,maxPot)≈0.25`(천장 거의 안 샘)·특급 평균 최상위(성적top5%→실제 상위 적중 ~0.4 ≫ 무작위 0.10).
4. **집계 포텐 별(★) 표시 제거**("별 1개만 뜨면 좀 그렇다"). 남는 표시 = **아마추어 성적표 + (스카우터로 선명해지는) 현재 OVR 범위 + (스카우터가 공개한) 개별 스탯 잠재력 일부**(#5). **합성 "아마추어 OVR" 숫자 표시 금지**(거의 역산됨) — 박스만 주고 판단은 플레이어.
