**실제 상수(문서 예시값과 차이):**
- 샐러리캡 `LEAGUE_CAP=350000`(**35억**, 문서 예시 30억), 개인상한 `MAX_SALARY=80000`(8억),
  프랜차이즈 `FRANCHISE_SEASONS=6`·`FRANCHISE_OVR=74`·`FRANCHISE_MAX=110000`(11억).
- **프랜차이즈 판정(2026-06-18 재정의)**: `isFranchise = 국내선수 AND clubTenure≥6 AND overallRaw≥74`.
  근속만 보면 시드 `clubTenure=나이−19` 탓에 25세↑ 전원이 프랜차이즈가 돼(측정 59%) 무의미했다.
  **장기근속 + 스타급 기량 + 외국인 제외**로 좁혀 "구단의 간판"만 남긴다(측정 ~7%). 캡 예외도 진짜
  간판에게만 적용돼 의미가 산다. 표시(계약관리·선수상세)·캡 예외 모두 이 한 함수로 일관(`engine/cap.ts`).
- 자격: `FIRST_FA_SEASONS=6` — `career.seasons≥6 AND contract.remaining≤1`. (문서 "이후 3시즌마다"는
  현재 계약 만료 기준으로 단순화.)
- 등급: 연봉 순위 **상위 35% A / 다음 35% B / 나머지 C**.
- 요구연봉 프리미엄 `PREMIUM = A 1.25 / B 1.15 / C 1.1`(2.2표의 보상 배수와는 별개).
- 보상금 `compensationMoney = A 2.0× / B 1.0× / C 0`(보상선수 동반), `compensationMoneyOnly = A 3.0× / B 2.0×`('돈만'·선수 보호). 보상선수는 **A·B만** 필요.
- `offerScore = w.money·연봉 + w.win·우승권 + w.loyalty·충성 + w.play·출전 + w.home·연고 + 0.05·rand`.
  **선수별 가중치 `w`** 가 동기를 차등화(2.5b). `우승권 = 0.7·전력 + 0.3·prestige`(최근 성적).
- AI 잔류 판단 `aiKeepsFA`: 나이<32 AND OVR≥70. 로스터 이상치 `ROSTER_IDEAL{S3,OH5,OP2,MB4,L2}=16`.
- **재계약에도 샐러리캡 적용**: 만료 FA 잔류는 가치순으로 팀 캡(35억) 한도 내에서만. 초과분은 잔류 못 하고
  FA 시장으로(`data/offseason.ts buildOffseason`). 재계약 연봉은 개인상한 클램프(`renewedContract`→`clampSalary`).
  단, 현재 연봉 수준에선 캡이 거의 안 걸려 왕조 억제 효과는 미미(향후 연봉 인플레 시 작동).

> 6장 데이터 모델(`clubTenure`/`career.seasons` 등)은 반영됨. 7장 모듈 매핑의 `leagueState.ts`는
> 실제로는 `data/league.ts`(가변 로스터·레지스트리·`baseVersion` 캐시 무효화)로 구현.
> **시즌 중 이동**(방출→FA 풀·포지션 구멍 긴급 영입)은 본 문서(오프시즌 FA)와 별개 — `TRANSACTION_SYSTEM`.

## 1. 단계 ① — 선수 생애 토대 (선행)

FA가 돌아가려면 선수가 빠지고(은퇴) 채워져야(유망주) 한다.

### 1.1 경력 누적
- `career.seasons` 를 롤오버마다 +1. 시드 생성 시 베테랑은 `career.seasons ≈ age - 19`(데뷔 추정).
- FA 자격 타이머의 기준.

### 1.2 은퇴 (retirement)
- 롤오버 시점 확률적 은퇴. 나이·능력(OVR)·출전 기회에 의존.
  ```
  retireChance(age, ovr):
    age < 30 : ~0
    30~32    : 낮음 (능력 낮을수록↑)
    33~35    : 중간
    36+      : 높음 → 38~40 사실상 은퇴
    저능력(벤치) 노장은 더 빨리
  ```
- 은퇴 선수는 리그에서 제거 → **로스터 공백** 발생(통산 기록은 보존 — 6장).

### 1.3 유망주 풀 (rookie pool)
- 매 시즌 신규 유망주 N명 생성(결정론 시드 + 시즌번호). 나이 18~20, 포텐 높고 현재 낮음.
- ③ 드래프트로 팀에 배정(그 전까지는 자동 충원으로 로스터 유지).

### 1.4 로스터는 이제 가변 (영속)
- 은퇴/영입/드래프트로 팀 구성이 바뀐다 → **팀 로스터를 세이브에 영속**.
- 선수도 신규 생성되므로 **전역 선수 레지스트리**(시드 + 생성분)로 통합. id로 항상 조회 가능.

## 2. 단계 ② — FA 코어

### 2.1 자격 공시
- **최초 FA: 6시즌, 이후 계약 만료마다 재자격**(현 구현 — 재계약 2년이므로 사실상 ~2시즌 주기). 시즌 종료 시 자격 선수 공시 → FA 시장 등장.
- 잔여 계약이 남았으면 비자격(계약 만료/자격 충족 시만).

### 2.2 등급 A/B/C (직전 연봉 순위 → 보상 크기)

| 등급 | 기준(연봉) | 타 구단 영입 시 보상 |
|---|---|---|
| **A** | 상위 | 직전연봉 **300%** (보상선수 X) **또는** **200% + 보상선수 1명** |
| **B** | 중위 | **200%** (보상선수 X) **또는** **100% + 보상선수 1명** |
| **C** | 하위 | **보상 없음** |

> 영입 구단이 두 옵션 중 선택(보상선수 줄지/돈 더 낼지).
>
> **구현 현황(2026-06-14)**: **두 옵션 모두 구현**.
> - **기본(보상선수 동반)**: A/B FA 영입 시 보상선수 1명(비보호)에 **더해 보상금(A 200%·B 100% × 직전연봉)을
>   운영 자금(FINANCE)에서 차감**(`resolveFAMarket.compCash` → `store.endSeason` faSpend, FA 센터 표시).
> - **'돈만'(선수단 보호)**: FA 센터에서 A/B 지명 시 토글 → 보상선수 면제, 대신 **가중 보상금(A 300%·B 200%)**
>   만 지불(`compensationMoneyOnly`). 보상선수 추첨(`pickCompensation`)을 건너뛴다(`moneyOnlyIds` 전파:
>   store → `buildDraftContext`/`resolvePreDraft`/`faMarketPreview` → `resolveFAMarket`). 부자 구단의 레버
>   — 비싼 대가로 유망주를 지킨다. 검증 `tools/simMoneyOnly.ts`(유출 면제·비율 정확, 위반 0).
> - 영입 입찰 가능 판정은 연봉+보상금(돈만이면 가중분) 합으로 본다(지갑 빌 만큼은 못 뽑음).

### 2.3 보상선수 + 보호선수 명단 ★
- 보상선수 옵션 선택 시: **영입 구단이 보호선수 명단(예: 6명) 제출**.
- **원소속 구단이 비보호 선수 중 1명 지명** → 데려감.
- 단장 결정: "누구를 보호하나"(영입 가치 vs 잃을 위험). AI는 비보호 중 최고 가치 선수를 집어감.

### 2.4 원소속 우선협상 / 프랜차이즈 ★
- **우선협상 기간**: 시장 오픈 전, 원팀만 먼저 재계약 시도.
- **프랜차이즈 스타**(국내선수 + 한 팀 ≥6시즌 근속 + 스타급 기량 `overallRaw≥74`, 2026-06-18):
  - 재계약 시 **샐러리캡 일부 예외**(충성 프리미엄),
  - **충성도 보너스**로 잔류 확률↑,
  - 타 구단 영입 시 **보상 가중**(한 등급↑ 취급 등 — 데려가기 더 비쌈).

### 2.5 협상·수락 로직 (선수가 고른다)
```
asking = marketValue × FA프리미엄(1.1~1.3)
offerScore = w.money·(연봉/asking) + w.win·우승권(0.7·전력+0.3·prestige)
           + w.play·출전기회(포지션 뎁스) + w.loyalty·충성(원소속·프랜차이즈) + w.home·연고/선호팀 + 0.05·rand + 면담bias
선수는 최고 offerScore 수락. 동점·근소차는 원소속 우선.
※ 계약연수(wYrs)·나이적합(wAge)은 설계안이었으나 **미구현** — 현재 코드(`engine/faMarket.ts:offerScore`)는 위 6항
   (money/win/play/loyalty/home/rand+talkBias). 도입 시 이 식·코드 동시 갱신.
```

### 2.5b 선수별 FA 성향 (이적 동기 차등) ★ — 2026-06 구현

현실의 FA는 선수마다 동기가 다르다. "돈 좇는 선수 / 우승 좇는 선수 / 남고 싶은 선수 /
출전 보장 원하는 선수 / 연고(드림팀) 가려는 선수." 이를 **선수별 가중치 프로필 `faPref`** 로 모델.

- **생성(결정론)**: `rollFAPref(rng, teamCount)` — 선수 **id 기반 별도 RNG**로 생성해 리그 시드
  메인 스트림을 건드리지 않음(기존 경기/스탯 결과 불변, FA 의사결정만 변화). `makePlayer`/`makeProspect`.
- **아키타입 5종 + 분포**: 머니 34% / 윈나우(우승권) 16% / 충성 26% / 출전 14% / 연고 10%.
  각자 한 동기를 크게(0.42~0.55) 두고 나머지는 옅게 + 개인 노이즈 → 정규화(합 1).
- **연고/선호팀**: `preferredTeamId` 1곳. `home` 가중이 큰 연고형이 강하게 끌림.
- **우승권 신호(prestige)**: 직전 시즌 정규순위 + 우승 여부(`data/offseason.ts teamPrestige`, 우승=1).
  offerScore의 `win` 항 = `0.7·전력 + 0.3·prestige`.

**밸런싱 기록 (200시즌 + 24유니버스×40시즌 검증):**
- 초기 시도(prestige 0.55, 윈나우 22%): **부익부 폭주** — parity std 10.8→16.9, 왕조 13→17연패.
  원인: 우승→prestige↑→우승권FA가 강팀에 몰림→계속 우승. 캡으로 강팀이 선수를 풀어도 우승권FA로
  싸게 리로드해 캡이 무력. **prestige 자기강화 항이 주범.**
- 조정(prestige 0.30, 윈나우 16%↓·충성 26%↑): parity std 4.55, 왕조 평균 8(최대 12),
  **순위 지속성 r = −0.08±0.40(서열 고착 없음)**, 우승경험 6.3/7, 반등 100%. → 채택.
- 교훈: 단일 유니버스 r은 표본 7개라 노이즈 큼(한 팀만 강하고 나머지 동률이면 r 폭등) →
  **밸런스 판단은 다중 유니버스 평균으로.** `npx tsx tools/simLeague.ts 40 24`.

### 2.6 FA 센터 UI (실제 하는 느낌)
시즌 종료 → **FA 센터**:
1. **내 FA 자격 선수**: 우선협상·재계약 / 떠날 위험 경고
2. **FA 시장 보드**: 등급별 목록 · asking · 경쟁 구단 · 예상 보상
3. **영입**: 오퍼(연봉·연수) → (보상선수 옵션 시) **보호명단 제출** → 선수 수락/거절 → **보상 처리**
4. **결과 요약**: 이적/잔류, 보상선수 이동, 캡 현황

## 3. 단계 ③ — 신인 드래프트
- 유망주 풀에서 **역순 지명**(하위 팀 우선) 또는 추첨(로터리).
- **스카우팅**: 포텐/일부 스탯만 공개(7장). 픽 가치 판단.
- 인간 단장 지명 + AI 지명. 미지명 유망주는 일부 잔류(다음 시즌 풀) 또는 소멸.

### 3.1 AI 픽 우선순위 — 3티어 (2026-06-25 개선)
> "OVR만 높다고 뽑지 않는다." AI는 자기 팀 사정에 맞춰 뽑되, 세대급 재능은 예외로 무조건 잡는다.
> `engine/draft.ts` `pickWithReason` — 순수 함수, 결정론. 픽마다 **사유(reason)** 를 함께 반환(라이브 콘솔 표시용).

순번이 오면 남은 유망주 중에서 **아래 순서로** 고른다:

1. **슈퍼 유망주 예외 (BPA, reason=`super`)** — 남은 후보에 **특급 유망주**(최대 포텐셜 ≥ `SUPER_POT=88`, 기존
   ★★★ 등급)가 있으면 **포지션 필요와 무관하게** 그중 최고를 잡는다(가치=`prospectValue`×감독성향×스카우트).
   세대급 재능은 자리가 차 있어도 놓치지 않는다(현실 BPA).
2. **포지션 필요 (need, reason=`need`)** — 특급이 없으면, **부족 포지션(gap>0)** 후보만 보고 그중 최고
   (`prospectValue`×`needWeight(gap)`×감독성향×스카우트). 자리가 남는 포지션은 이 단계에서 안 뽑는다(`aiFillFromPool`과 동일 정책).
3. **필요 없음 → OVR + 성격 (best, reason=`best`)** — 부족 포지션이 하나도 없으면(이상 구성 충족),
   **현재 실력(OVR)과 성격**으로 고른다(`overall`×`personalityFactor`×감독성향×스카우트).
   - **성격(personalityFactor)** = 멘탈(집중·기복·VQ 평균) + 특성(긍정 가산/부정 감산). 범위 ~0.85~1.15(소폭).
     리더·클러치·노력형 같은 좋은 성격은 동급에서 우대, 새가슴·유리몸은 기피. *placeholder — 측정으로 튜닝.*

> `prospectValue`(포텐 비중 0.6)는 1·2티어(육성 베팅)에, `overall`(현재 실력)은 3티어(즉시 전력+성격)에 쓴다.
> 인간 단장의 위시리스트(reason=`wish`)는 모든 티어보다 우선(순번 내에서 가능한 첫 위시).
> 측정 가드 `tools/_ev_draftpick.ts`(슈퍼 무조건 선발·필요 포지션 우선·무필요 시 성격 영향·결정론·A/B).

### 3.2 라이브 드래프트 (sim-web 콘솔 + 앱)
> 단장 업무 하이라이트를 **보는 재미**로(관전형 1순위). 엔진은 순수 해석 그대로, **UI가 순번을 한 픽씩 재생**.

- **팀 캡**: 총연봉 ≤ CAP(예: 30억). 초과 영입/재계약 불가.
