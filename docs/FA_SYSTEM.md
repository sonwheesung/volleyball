## 1. 단계 ① — 선수 생애 토대 (선행)

FA가 돌아가려면 선수가 빠지고(은퇴) 채워져야(유망주) 한다.

### 1.1 경력 누적
- `career.seasons` 를 롤오버마다 +1. 시드 생성 시 베테랑은 `career.seasons ≈ age - 19`(데뷔 추정).
- FA 자격 타이머의 기준.

- **세이브 호환(오프시즌 재계산)**: 오프시즌 해석은 로드 시 base+seed로 재계산되므로(REALTIME_SIM), 앱 업데이트로 공식/상수가 바뀌면 **진행 중 오프시즌의 은퇴자 명단이 바뀔 수 있다**(엔진버전 게이트 밖). 수용(사용자 결정): 은퇴는 오프시즌 1회 확정 사건이고, 릴리스 사이 진행 중 오프시즌은 드문 경계 케이스 — 별도 게이트 없이 재계산 결과를 채택(관전형 재현 일관성 우선). 시즌 진행 중 확정된 과거 은퇴 기록(`retirements`)은 영속이라 불변.

### 2.3 보상선수 + 보호선수 명단 ★

### 2.4 원소속 우선협상 / 프랜차이즈 ★
- **우선협상 기간**: 시장 오픈 전, 원팀만 먼저 재계약 시도.
- **프랜차이즈 스타**(국내선수 + 한 팀 ≥6시즌 근속 + 스타급 기량 `overallRaw≥74`, 2026-06-18):
  - 재계약 시 **샐러리캡 일부 예외**(충성 프리미엄),
  - **충성도 보너스**로 잔류 확률↑,
  - 타 구단 영입 시 **보상 가중**(한 등급↑ 취급 등 — 데려가기 더 비쌈). **[아직 미구현]**(2026-07-08 코드 대조): FA 등급은 `assignFAGrades`가 **직전연봉 순위만**으로 매기고(§2.2), `resolveFAMarket`의 보상금(`compCost`)도 그 등급으로 계산 — 보상 경로에 `isFranchise` 분기 0건(`isFranchise`는 캡 재계약 예외·offerScore 충성 항에만 쓰임). 프랜차이즈의 "한 등급↑"은 미반영.

> 기존 `offerScore` **argmax(최고 점수 팀에 무조건 입단)** 을 **점수→확률→정렬·롤·fallback**으로 교체한다.
> "왜 이 팀에 갔나"가 투명한 가산/감점 점수가 되고, 인간관계망(RELATIONSHIP_SYSTEM)이 그 점수의 한 항(±)으로 들어간다.
> **순차 결정론 골격(`resolveFAMarket`)은 유지** — argmax만 확률 해석으로 바꾼다(대공사 아님).

### 2.7.1 오퍼 점수 (player, team) ∈ [0,100]
선수별 가중치(`faPref.w` — 2.5b, 합 ~1)로 각 요인 점수를 합산해 100점화. **요인(가산/감점)**:
| 요인 | 방향 | 출처 |
|---|---|---|
| 우승권(전력+prestige) | + | winT (2.5) |
| 연봉(제시/요구) | + | moneyT |
| 장기근속·원소속·프랜차이즈 | + | loyT |
| 주전 보장(포지션 구멍) | + | playT |
| **좋은 팀메이트 / 싫은 선수** | **±** | relT = `teamAffinity`(RELATIONSHIP §2) — 친구 +, 라이벌/앙숙 − |
| 면담(내 팀) | + | talkBias |
```
score = 100 × ( w.money·moneyT + w.win·winT + w.loyal·loyT + w.play·playT + w.home·homeT + w.rel·relT ) + talkBias·k
clamp [0,100]   // relT가 −면 감점(싫은 선수 있는 팀 점수↓). 선수 성향(w)이 "의리파/우승파"를 가른다
```

### 2.7.2 점수 → 확률 (완만한 S곡선 — 사용자 결정)
`P(수락) = smoothstep(SIT_FLOOR, CERTAIN, score)` — `t=clamp((score−FLOOR)/(CEIL−FLOOR),0,1); P=t²(3−2t)`.
- `score ≤ FLOOR`(예 30) → P≈0(거의 거절), `score ≥ CEIL`(예 80) → P≈1(거의 확정), 중간은 확률적(업셋 드라마).
- 높은 점수는 거의 확정·중간만 확률 → 강팀이 대체로 이기되 가끔 업셋(parity 안전).

### 2.7.3 여러 오퍼 해결 (선수 1명 기준)
1. **오퍼 팀 모음**: 그 포지션을 원하고(gap>0 또는 내 지명) **그 순간** 슬롯·캡·자금 가능한 팀만.
2. 각 팀 score→P 계산(점수는 **그 시점 로스터** 기준 — 친구 연쇄 반영).
3. **오퍼 0개 → 미입단**(FA 잔류 — 팀이 안 원해서. 양방향).
4. **최고 점수 < SIT_FLOOR → 시즌 아웃**(잔류, 사용자 결정 — "그럴 바엔 안 뛴다", 드물게).
5. 아니면 **P 내림차순 정렬 → 위에서부터 시드 롤 → 첫 성공 팀 입단**.
6. **전부 실패 → 가장 높았던 P 팀 입단**(fallback).
7. **오퍼 1팀 → fallback상 결국 그 팀**(전부 실패해도 최고=그 팀) → 자동 입단(단 4의 SIT_FLOOR는 적용).

### 2.7.4 엣지 케이스 / 동시성 (순차 결정론으로 해소)
실제 병렬 아님(순수 함수). 위험 = **논리적 동시성**(한 번에 여러 선수·팀, 공유 자원=슬롯·캡·"한 선수 한 팀"). 해법 = **고정 순서 + 매 선수 가용성 갱신**:
| 엣지 | 처리 |
|---|---|
| 같은 선수 ↔ 여러 팀 | 그 선수 1회 해결 → 한 팀만. 나머지 팀은 다음 선수로 |
| 같은 팀 ↔ 여러 선수 | **OVR 내림차순 순차** — 계약 시 그 팀 캡·슬롯 즉시 차감 → 다음 선수는 꽉 찬 팀 제외(이중계약·캡초과 0) |
| fallback 최고 팀이 그새 꽉 참 | 해결 시점 **계약 가능 팀만 재필터** 후 정렬·롤 |
| 친구 연쇄(A 입단→B의 그 팀 점수↑) | 점수를 **그 시점 로스터**로 계산 → "친구 따라간다" 자연 발생. 순서 고정(OVR)이라 결정론 |
| 내 팀 미리보기=결과 | 같은 시드·순서 → FA 센터 예상==실제(기존 보장 유지) |
| 보상선수 | 행선지 확정 후 적용(순서 무관) |
| 로스터 하한 | 시즌 아웃·이탈 누적으로 부족 시 드래프트·신인 충원(ROSTER_MIN 게이트) — SIT_FLOOR 낮게 둬 드물게 |

### 2.7.5 밸런스 관문

> 구현은 RELATIONSHIP_SYSTEM Phase 2(이 §2.7이 그 phase의 상세). `engine/faMarket.offerScore`→`offerScoreOf(0~100)`+`acceptProb`,

### 2.7.6 FA 센터 전달(UX) — 지연 구조·실패 사유·보상 개념 (✅ 2026-07-09 구현)

> 설계(경쟁·확률·지연·보상)는 KOVO/§2.2~2.7대로 정상인데 **화면이 그걸 안 알려줘** "이해가 안 됨"(확정 UX 결함).
> 엔진·`resolveFAMarket` 로직은 **불변** — 표시·카피·실패 사유 전달만 보강한다(`app/fa.tsx` + `data/offseason.ts` 관측 추가).

**(1) 지연 구조 1줄 안내(항상 표시)** — FA는 즉시 계약이 아니라 **지명(faSignings) → 시즌 시작 때 경쟁 입찰 해석
(resolveFAMarket) → 선수가 offerScore로 수락**하는 지연·확률 구조다. 상단 배너에 "지금은 '지명' 단계 — 결과는 시즌
시작 때 다른 구단과의 경쟁으로 확정"을 못박아 "영입 눌렀는데 실패"라는 오해를 없앤다. 헤더 카피도 `영입 성공 / 지명`으로,
"질 수도·잔류할 수도·캡/자금 안에서만"을 명시. 탭하면 상세 모달(지연·경쟁·잔류·자금 3경로 설명).

**(2) 실패 사유 세분화** — `resolveFAMarket`이 내가 지명한 FA의 **내 팀 입찰 게이트 결과**(myGate)를 관측해
`FAMarketResult.faFail: Record<id, FAFailCode>`로 노출(`FAPreview.faFail`로 전달). 화면은 '경합/불발' 뭉뚱그림을 제거하고
사유별로 표기:

| 코드 | 게이트(resolveFAMarket) | 화면 문구 |
|---|---|---|
| `ROSTER` | `ROSTER_TOTAL − 로스터 ≤ 0`(입찰 전 컷) | 선수 자리 없음 — 정원 초과 |
| `CAP` | `!canAfford(payroll, offer)`(국내 캡) | 샐러리캡 초과 — 입찰 못 함 |
| `CASH` | `offer + compCost > cashLeft`(운영 자금) | 운영 자금 부족 — 입찰 못 함 |
| `LOST` | 내 입찰 성사(BID) 후 다른 팀이 롤 승 | {팀}에 뺏김 (경쟁 입찰 패배) |
| `SIT_OUT` | 최고 점수 < `SIT_OUT`(0.14) → 선수 잔류 | {선수} 선수가 잔류를 택함 |

**우선순위 = 게이트 먼저**: 내 입찰이 아예 안 들어간 경우(ROSTER/CAP/CASH)는 AI가 그 선수를 뽑아 `lostTo`가 찍혀도
'뺏김'이 아니라 **그 사유**로 보여준다 — "자금 없어 입찰조차 못 함 ≠ 경쟁 패배"(옛 화면이 자금부족을 '경합'으로
오표시하던 핵심 결함 수정). 내 입찰이 성사(BID)된 경우만 `lostTo`면 LOST, 아니면 SIT_OUT.

**(3) 보상선수·'돈만' 인라인 설명** — "보상선수란?"/"'돈만'이 뭔가요?" 안내를 모달로 제공: **A·B 등급 FA를 영입하면
원소속팀이 우리 비보호 1명을 데려간다(§2.2~2.3)**, 보호명단(6명) 밖 최고 OVR부터 유출, **'돈만'은 보상선수 없이 돈을 더
내고(A 300%·B 200%) 선수단을 지키는 옵션**, **C 등급은 보상이 없어 토글이 안 뜬다**(왜 A·B만 토글이 뜨는지).

**모달은 전부 커스텀 다크 글래스(`Popup`) — 네이티브 `Alert` 금지(UI-21)**.

> 검증: `tools/_dv_fafail.ts`(신규 상설 가드) — (A) 합성 직접호출로 CASH/CAP/ROSTER 게이트를 강제해 코드 대조 +
> A/B 민감도(자금 뒤집으면 코드 사라짐) + 자금0인데 AI가 데려간 케이스가 CASH 우선(오표시 수정 확인), (B) 자연 장기런에서
> WON(코드 없음·내 팀)·LOST(lostTo=실제 소속)·SIT_OUT(어느 팀에도 없음)을 **관측 가능한 최종 로스터**로 교차검증.
> `_dv_uictx`(등급 pre-FA 무회귀)·`simFaDup`·`simMoneyOnly` 그린 유지, tsc 신규 0.

**① 주전보장 대가(핵심) — 공약 파기 재활용.** `starterGuarantee:true`로 **내가 서명한** FA 계약에만 `contract.starterGuarantee` flag를 남긴다
  (신규 영속 = 기존 Contract 객체에 optional 1비트. `faOffers`는 오프시즌 후 store가 비우므로 계약에 남겨 이후 시즌까지 재파생). 그 선수를
  이후 시즌에 **벤치**(sitCause `ownerBenched`/`outclassed`)하면:
  - `expectsPlayOf`가 보장 선수의 **주전 기대치=1**로 올려(OVR·경력 무관) 벤치 시 `minutes` 불만이 **확실히 발화** — 무명에게 주전 약속하고 앉히는 공짜 회피 차단.
  - `data/owner.ts buildOwnerFx`의 공약파기 항이 **면담 카드 OR `contract.starterGuarantee`** 를 '주전 보장 약속'으로 봐(`starterPromised` 재활용) → `topic==='minutes'`면 **재계약 거부 +`PROMISE_BREACH_REFUSE`(0.5)**.
  - 나머지 대가는 **기존 스트림이 벤치 자체로 자동 부과**: OWNER 불만(discontent) + 팬심 하락(`benchAngerPenalty`) + 폼 하락(FORM 결장 누적). 보장 레버는 여기에 **파기 가산**을 얹는 것.
  - 이행(주전으로 실제 기용)하면 `minutes` 불만 자체가 없어 **대가 0**(약속 지키면 공짜). 가드 `tools/_ev_fastarter.ts`: 벤치+보장(0.95) > 벤치만(0.78) > 미벤치+보장(0.00), expectsPlay(보장)=1.

**② 다년 대가 — 기존 기계장치 소비(과설계 금지).** 오퍼 `years`가 Phase 1에서 이미 `contract.years=remaining`으로 흐른다. 다년의 대가는 **추가 코드 없이** 두 기존 항이 부과:

## 2.9 대체 FA 안전망 — 필러 파이프라인 (📋 설계 확정·미구현 2026-07-09)

> 배경: §1.5 가변 로스터 + §1.6 floor 하향 + §1.7 attrition 강화로 **포지션 구멍이 자연 발생**하는데,
> organic FA 풀에 그 포지션 대체급이 없으면 팀이 그 자리를 못 메운다(경기 성립 불가·불변식 붕괴). 이를 막는 **2층 안전망**.

### 파이프라인 4층 (역할 분담)
`(C) 신인 공급(드래프트 §3)` → `(A) 시장 바닥 공급` → `fillRosters 소비(§1.6)` → `(B) 최후 방어`
필러는 (A)·(B) 두 층이다. (A)는 시장을 조용히 두껍게, (B)는 그래도 뚫리면 마지막에 강제로 막는다.

### (A) 시장 바닥 (organic 부재 시에만 1명 톱업)
- **발동 조건**: organic FA 풀에 **그 포지션 대체급이 하나도 없을 때만** 해당 포지션에 **1명** 주입(항상 스폰 아님 —
  시장에 대체급이 있으면 발동 안 함).
- **스펙**: OVR **raw 58~63(중심 60)** · 나이 **27~31** · **포지션당 최대 1명**. 평범한 즉시전력 백업.
- **비영속 시드**: `fa-filler:{시즌}:{포지션}` — **저장 안 함**, 매 오프시즌 결정론 재파생(리플레이 일치).
- **통계 격리(중요)**: 미서명 필러는 **medianOvr·리그 통계 산정에서 제외**한다 — 미서명 배관 물량이 리그 OVR
  앵커(§1.2 은퇴·연봉·AI잔류의 medOvr)를 오염시키면 안 됨. **뉴스 미노출**(시장 배관이라 서사 아님).
- **서명 시 승격**: 어떤 팀이 이 필러를 서명하면 **신규 고유 id로 승격**(필러 시드 id와 분리 → id 충돌 방지)하고
  **그때부터 실선수**(커리어 누적·은퇴·HOF 파이프라인 정상 진입). 미서명이면 다음 오프시즌 시드 갱신으로 사라짐.

### (B) 최후 강제 주입 (오프시즌 해석의 고정 마지막 순수 스텝)
- **위치**: 오프시즌 해석 순서 `은퇴 → FA → 드래프트 → fillRosters` **뒤의 고정 마지막 순수 스텝**.
- **발동 조건**: 그래도 로스터 < floor(§1.6) **또는 특정 포지션 전멸**인 팀에 강제 충원. **AI 팀 포함 리그 전역**
  (내 팀만이 아니라 전 구단 — 어느 팀도 경기 불가 상태로 시즌 진입 금지).
- **시드**: `fa-filler:{시즌}:{teamId}:{포지션}`(팀 종속 — (A)와 달리 팀에 귀속).
- **인시즌 금지(하드)**: (B)는 **오프시즌에만**. **인시즌 강제 영입은 금지** — base 스냅샷과 어긋나 결정론이 붕괴한다.
  인시즌 부상 구멍은 **floor의 −3 버퍼(주전+1부상흡수, §1.6)** 가 흡수한다.
- **불변식(방어 가드)**: 정상 리그에선 **발동하지 않아야 한다**(`buildLineup` throw-guard와 같은 결 — 뚫리면 버그 신호).
  (B)가 자주 발동하면 floor/attrition/클래스 규모 튜닝 오류 → §E 검증으로 감시.

### 가짜 드라마 방지 원칙

  4라운드 지명권을 준다(원치 않으면 패스).

   세대급 재능은 자리가 차 있어도 놓치지 않는다(현실 BPA).
   > **컷 정의 주의**: maxPot(스탯 14개 중 최대) 단독은 **포화**(클래스 71%가 ≥88 — 누구나 한 스탯은 99 근처)라
   > "특급"을 못 가린다. **드래프트가치(`prospectValue`=현재OVR×0.4+maxPot×0.6)** 로 정의 → 클래스 **상위 ~10%**
   > 만 특급(측정 `_tmp_pv`: pv≥80=18%·≥82=3%, median 75.6). 라이브 콘솔로 "전부 특급 BPA"를 잡아 교정(2026-06-25).
2. **포지션 필요 (need, reason=`need`)** — 특급이 없으면, **부족 포지션(gap>0)** 후보만 보고 그중 최고
   (`prospectValue`×`needWeight(gap)`×감독성향×스카우트). 자리가 남는 포지션은 이 단계에서 안 뽑는다(`aiFillFromPool`과 동일 정책).
3. **필요 없음 → OVR + 성격 (best, reason=`best`)** — 부족 포지션이 하나도 없으면(이상 구성 충족),
   **현재 실력(OVR)과 성격**으로 고른다(`overall`×`personalityFactor`×감독성향×스카우트).
   - **성격(personalityFactor)** = 멘탈(집중·기복·VQ 평균) + 특성(긍정 가산/부정 감산). 범위 ~0.85~1.15(소폭).
     리더·클러치·노력형 같은 좋은 성격은 동급에서 우대, 새가슴·유리몸은 기피. *placeholder — 측정으로 튜닝.*

> `prospectValue`(포텐 비중 0.6)는 1·2티어(육성 베팅)에, `overall`(현재 실력)은 3티어(즉시 전력+성격)에 쓴다.
> 인간 단장의 위시리스트(reason=`wish`)는 모든 티어보다 우선(순번 내에서 가능한 첫 위시).
> 측정 가드 `tools/_ev_draftpick.ts`(슈퍼 무조건 선발·필요 포지션 우선·무필요 시 성격 영향·결정론·A/B).

### 3.3 아마추어 성적표 — 스카우팅 2.0 (📋 설계 확정 2026-07-03 · GPT+Claude 합의 완료 · **구현 착수 신호 대기** · 코드 미착수)
> **사용자 결정(설계 방향)**: 현실 스카우팅처럼 유망주마다 **고교/대학(아마추어) 성적표**를 보여주되, **성적이 진짜 실력·천장을 정확히 말해주지 않게**(노이즈) 해서 발굴 도박·반짝 함정·숨은 보석을 만든다. 독립 리뷰(general-purpose, N=20,000 실측) + GPT 검토 + Claude 보강으로 **최종 합의(A/95점)**. **이 절이 곧 빌드 스펙 — 사용자 "시작" 신호에 착수.**
> **최종 목표(한 줄)**: 드래프트를 **"정답 맞추기"가 아니라 "정보를 해석하는 과정"** 으로. 성적=즉시전력 신호, 스카우터=미래를 조금 더. **끝까지 100% 정답은 없다** — 그 불확실성이 관전 재미·서사.

**핵심 판타지(왜)**: ① **반짝(flash)** — 아마추어 성적은 좋은데 포텐 낮은 유망주(뽑으면 bust). ② **숨은 보석(gem)** — "유망주 소리도 안 듣던 선수를 순위 높아 아무나 뽑았는데 포텐이 엄청 높은". 성적만 보면 둘을 못 가린다 → 천장은 **시즌이 지나며** 드러나는 도박(관전형·누적 서사 보상). 실측: `corr(현재OVR, maxPot)=0.51`, 중앙값 4분면에서 flash·gem 각 ~16%(극단 각 ~2%).

> **★ 사용자 정제(2026-07-03) — 리뷰의 "corr 0.51 유지"와 갈림**: 원하는 건 **"포텐은 현재(=성적)에 대체로 비례하되 *가끔* 이상치(대기만성·반짝)"** 다. 즉 **대부분은 예측 가능**(좋은 성적=대체로 좋은 포텐 → 스카우팅 판단이 대체로 보상됨)하고, **극단은 드물게(예: 5~10%)** 나와야 기억에 남는 서사가 된다. 현행 0.51(flash/gem 각 ~16%=꽤 잦음)은 "가끔"보다 흔함 → **목표: `corr(현재,포텐)`을 더 높이고(예 ~0.7, 대체로 비례) + 소수 이상치를 heavy-tail로 명시 주입**(대기만성=현재 낮은데 포텐 top / 반짝=현재 높은데 포텐 낮음). **이건 `makeProspect`의 talent→포텐 링크 조정 + 이상치 주입이라 리뷰의 "makeProspect 손대지 마라"와 상충** → 검증(GPT+사용자) 때 **목표 corr·이상치 비율**을 확정하고, 아래 검증 밴드(역산불가 corr 밴드)는 그 결정에 맞춰 재설정. **성적 자체는 여전히 현재+노이즈(포텐 항 0)** — 바뀌는 건 "현재↔포텐 분포"뿐이다.

**설계 원칙(리뷰 확정)**:
1. **성적 = 현재 윗단 스탯(포지션별 생산) × 학교강도 인플레(id 시드) × 노이즈.** **포텐 항 0 — 순수 현재 기반.** 포텐을 성적에 섞으면 gem이 좋은 성적을 내 정체가 드러나 도박이 붕괴(채택 금지). 현실 아마추어 박스도 "한 것(생산)"을 재지 "천장"을 안 잰다.
2. **약체 리그 인플레**: 아마추어는 약한 상대끼리 → **절대수치를 프로 규범 위로** 민다(공격성공 프로 40~55%→아마 55~65%·에이스↑). realism 향미 + **프로 규범에 앵커한 순진한 플레이어를 낚는 함정**. 학교/리그 강도 라벨("2부 대학 리그")로 서사 정당화. **은폐 주력은 노이즈, 인플레는 향미/함정**(대체로 순위 안 바꿈).
3. **노이즈(제일 중요)**: **곱셈 ±15%, `id × 스탯카테고리`별 독립**(단일 배수면 카테고리 비율 보존돼 삼각측량 가능 → 금지), **스카우터·팀·reveal 무관(seed=id만)** — 플레이어·AI 동일 조건, 스카우터로 성적 노이즈 못 지움. 목표: `corr(성적,현재)≈0.5`(역산 불가·현재분산 ~25%만 설명)·`corr(성적,maxPot)≈0.25`(천장 거의 안 샘)·특급 평균 최상위(성적top5%→실제 상위 적중 ~0.4 ≫ 무작위 0.10).
4. **집계 포텐 별(★) 표시 제거**("별 1개만 뜨면 좀 그렇다"). 남는 표시 = **아마추어 성적표 + (스카우터로 선명해지는) 현재 OVR 범위 + (스카우터가 공개한) 개별 스탯 잠재력 일부**(#5). **합성 "아마추어 OVR" 숫자 표시 금지**(거의 역산됨) — 박스만 주고 판단은 플레이어.
5. **스카우터가 잠재력을 부분 공개 (사용자 정제 2026-07-03) — 개별 스탯 잠재력 일부**: 스카우터 레벨↑ → 유망주의 **개별 스탯 잠재력(`potential[]`)을 몇 개** 공개(정확치, 스카우터 보상). **최고 스카우터라도 전부 못 봄 — 상한 ~50%**(늘 절반은 숨음) → 잔여 미스터리로 대기만성·반짝이 살아남는다(도박 유지). 리뷰의 "모호한 성장여지 프로즈/티어"보다 **구체적·정보량 큰 이 안**을 채택(스카우터 ROI 복원 + 은닉 상한으로 도박 보존 동시 달성).
   - **검증 확정 항목**: 공개 개수 산식(`reveal%`→공개 스탯 수, 최고≈14개 중 ~7개=50%)·**어느 스탯 우선 공개**(포지션 핵심 스탯 vs `id` 결정론 랜덤 vs 최고포텐 — 최고포텐 우선은 스포일러라 주의)·공개 정밀도(정확 vs 범위).
6. **AI 공정성(최대 갈림길) — 대칭 채택**: 플레이어·AI **동일 정보식단.** AI `pickWithReason`이 전지적 `maxPot`(0.6항) 대신 **자기 팀 스카우터의 부분 포텐 view(50% 캡, 공개된 스탯 잠재력만) + 아마추어 성적 + reveal 현재OVR + 포지션 need**로 평가(전지적 pot 직접참조 제거). 스카우터 좋은 팀이 더 많은 스탯 천장을 봄(현실적 비대칭 — 플레이어도 스카우터 투자로 대칭 확보). 특급 양방(고현재·고포텐)은 현재도 높아 성적 압도 → 여전히 상위 지명. **은닉된 스탯의 gem/반짝은 아무도 확신 못 함**(의도된 도박·서사).

- **역할 불변 원칙(GPT 우려 "성적만 보는 게임" 방어)**: **성적=현재(윈나우) · 스카우터=미래(빌딩).** 드래프트 유망주는 현재가 낮고 비슷 → 성적만 보면 즉전감만·반짝 함정 → 어느 스타일도 지배적이지 않게 유지(둘 다 유효).

**파일(착수 시)**: `engine/draft.ts`(pickWithReason pot 제거·SUPER_PV 재정의)·`engine/ratings.ts`(deriveRatings 입력)·`data/amateurRecord.ts`(신, 순수 성적 파생)·`engine/overall.ts`(fog 헬퍼 공유)·`app/draft.tsx`·`app/draft-live.tsx`(★ 제거·성적표)·`data/seed.ts`(makeProspect — **손대지 말 것**). 검증 `tools/_dv_amateur.ts`(신)·`tools/simLeague.ts`(밸런스 회귀).
