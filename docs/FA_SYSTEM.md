**실제 상수(문서 예시값과 차이):**
- 샐러리캡 `LEAGUE_CAP=350000`(**35억**, 문서 예시 30억), 개인상한 `MAX_SALARY=80000`(8억),

- **프랜차이즈 판정(2026-06-18 재정의)**: `isFranchise = 국내선수 AND clubTenure≥6 AND overallRaw≥74`.
  근속만 보면 시드 `clubTenure=나이−19` 탓에 25세↑ 전원이 프랜차이즈가 돼(측정 59%) 무의미했다.
  **장기근속 + 스타급 기량 + 외국인 제외**로 좁혀 "구단의 간판"만 남긴다(측정 ~7%). 캡 예외도 진짜
  간판에게만 적용돼 의미가 산다. 표시(계약관리·선수상세)·캡 예외 모두 이 한 함수로 일관(`engine/cap.ts`).
- 자격: `FIRST_FA_SEASONS=6` — `career.seasons≥6 AND contract.remaining≤1`. (문서 "이후 3시즌마다"는
  현재 계약 만료 기준으로 단순화.)
- 등급: 연봉 순위 **상위 35% A / 다음 35% B / 나머지 C**.
- 요구연봉 프리미엄 `PREMIUM = A 1.25 / B 1.15 / C 1.1`(2.2표의 보상 배수와는 별개).
- 보상금 `compensationMoney = A 2.0× / B 1.0× / C 0`(보상선수 동반), `compensationMoneyOnly = A 3.0× / B 2.0×`('돈만'·선수 보호). 보상선수는 **A·B만** 필요.
- `offerScore = w.money·연봉 + w.win·우승권 + w.loyalty·충성 + w.play·출전 + w.home·연고 + 0.05·rand`.
  **선수별 가중치 `w`** 가 동기를 차등화(2.5b). `우승권 = 0.7·전력 + 0.3·prestige`(최근 성적).
- AI 잔류 판단 `aiRetainProb`(2026-06-25, 구 `aiKeepsFA` 이진 절벽 대체): OVR·나이 **연속 확률**(0~1)을
  결정론 시드로 굴림 — 절벽(32세·OVR70 칼컷) 제거, **엘리트는 노쇠에도 소프트 플로어**(32세 에이스 안 버림),
  가끔 노장 잔류·영건 이탈로 리그 생동. 측정 보정(`_ev_airetain`): 순잔류 ~58%(구와 동률)·나이/OVR 매끄러운 그라데이션·
  33세 에이스 0.82·연속 73%. 로스터 이상치 `ROSTER_IDEAL{S3,OH5,OP2,MB4,L2}=16`.
- **재계약에도 샐러리캡 적용**: 만료 FA 잔류는 가치순으로 팀 캡(35억) 한도 내에서만. 초과분은 잔류 못 하고
  FA 시장으로(`data/offseason.ts buildOffseason`). 재계약 연봉은 개인상한 클램프(`renewedContract`→`clampSalary`).
  ~~단, 현재 연봉 수준에선 캡이 거의 안 걸려 왕조 억제 효과는 미미.~~ **정정(2026-06-25 측정)**: 실측상
  캡은 **상위권 팀에 실제로 작동**한다 — 팀 국내 페이롤이 캡(35억) 대비 **최대 99%·중앙 74%·≥90% 7%·≥80% 26%**
  (15시즌×7팀 n=105, `_tmp_cap`). 즉 **컨텐더(돈 쓰는 팀)만 캡에 밀착해 누구 지키고 포기할지 결정 압박**을 받고,
  약팀은 여유 — 현실 샐러리캡 그대로다. 100% 초과 시도 0%(개인상한·재계약 캡으로 사전 차단). 왕조 억제도 작동
  (simLeague 40시즌 우승 6팀 분산·고착 없음). → **밸런스 변경 불필요**(리뷰 "캡 안 걸림"은 본 stale 주석 기반 오판).

## 1. 단계 ① — 선수 생애 토대 (선행)

FA가 돌아가려면 선수가 빠지고(은퇴) 채워져야(유망주) 한다.

### 1.1 경력 누적
- `career.seasons` 를 롤오버마다 +1. 시드 생성 시 베테랑은 `career.seasons ≈ age - 19`(데뷔 추정).
- FA 자격 타이머의 기준.

### 1.2 은퇴 (retirement)
- 롤오버 시점 확률적 은퇴. 나이·능력(OVR)·출전 기회에 의존.
  ```
  retireChance(age, ovr):
    age < 30 : ~0
    30~32    : 낮음 (능력 낮을수록↑)
    33~35    : 중간
    36+      : 높음 → 38~40 사실상 은퇴
    저능력(벤치) 노장은 더 빨리
  ```
- 은퇴 선수는 리그에서 제거 → **로스터 공백** 발생(통산 기록은 보존 — 6장).

### 1.3 유망주 풀 (rookie pool)
- 매 시즌 신규 유망주 N명 생성(결정론 시드 + 시즌번호). 나이 18~20, 포텐 높고 현재 낮음.
- ③ 드래프트로 팀에 배정(그 전까지는 자동 충원으로 로스터 유지).

### 1.4 로스터는 이제 가변 (영속)
- 은퇴/영입/드래프트로 팀 구성이 바뀐다 → **팀 로스터를 세이브에 영속**.
- 선수도 신규 생성되므로 **전역 선수 레지스트리**(시드 + 생성분)로 통합. id로 항상 조회 가능.

## 2. 단계 ② — FA 코어

### 2.1 자격 공시
- **최초 FA: 6시즌, 이후 계약 만료마다 재자격**(현 구현 — 재계약 2년이므로 사실상 ~2시즌 주기). 시즌 종료 시 자격 선수 공시 → FA 시장 등장.
- 잔여 계약이 남았으면 비자격(계약 만료/자격 충족 시만).

### 2.2 등급 A/B/C (직전 연봉 순위 → 보상 크기)

| 등급 | 기준(연봉) | 타 구단 영입 시 보상 |
|---|---|---|
| **A** | 상위 | 직전연봉 **300%** (보상선수 X) **또는** **200% + 보상선수 1명** |
| **B** | 중위 | **200%** (보상선수 X) **또는** **100% + 보상선수 1명** |
| **C** | 하위 | **보상 없음** |

> 영입 구단이 두 옵션 중 선택(보상선수 줄지/돈 더 낼지).
>
> **구현 현황(2026-06-14)**: **두 옵션 모두 구현**.
> - **기본(보상선수 동반)**: A/B FA 영입 시 보상선수 1명(비보호)에 **더해 보상금(A 200%·B 100% × 직전연봉)을
>   운영 자금(FINANCE)에서 차감**(`resolveFAMarket.compCash` → `store.endSeason` faSpend, FA 센터 표시).
> - **'돈만'(선수단 보호)**: FA 센터에서 A/B 지명 시 토글 → 보상선수 면제, 대신 **가중 보상금(A 300%·B 200%)**
>   만 지불(`compensationMoneyOnly`). 보상선수 추첨(`pickCompensation`)을 건너뛴다(`moneyOnlyIds` 전파:
>   store → `buildDraftContext`/`resolvePreDraft`/`faMarketPreview` → `resolveFAMarket`). 부자 구단의 레버
>   — 비싼 대가로 유망주를 지킨다. 검증 `tools/simMoneyOnly.ts`(유출 면제·비율 정확, 위반 0).
> - 영입 입찰 가능 판정은 연봉+보상금(돈만이면 가중분) 합으로 본다(지갑 빌 만큼은 못 뽑음).

### 2.3 보상선수 + 보호선수 명단 ★
- 보상선수 옵션 선택 시: **영입 구단이 보호선수 명단(예: 6명) 제출**.
- **원소속 구단이 비보호 선수 중 1명 지명** → 데려감.
- 단장 결정: "누구를 보호하나"(영입 가치 vs 잃을 위험). AI는 비보호 중 최고 가치 선수를 집어감.

### 2.4 원소속 우선협상 / 프랜차이즈 ★
- **우선협상 기간**: 시장 오픈 전, 원팀만 먼저 재계약 시도.
- **프랜차이즈 스타**(국내선수 + 한 팀 ≥6시즌 근속 + 스타급 기량 `overallRaw≥74`, 2026-06-18):
  - 재계약 시 **샐러리캡 일부 예외**(충성 프리미엄),
  - **충성도 보너스**로 잔류 확률↑,
  - 타 구단 영입 시 **보상 가중**(한 등급↑ 취급 등 — 데려가기 더 비쌈).

### 2.5 협상·수락 로직 (선수가 고른다)
```
asking = marketValue × FA프리미엄(1.1~1.3)
offerScore = w.money·(연봉/asking) + w.win·우승권(0.7·전력+0.3·prestige)
           + w.play·출전기회(포지션 뎁스) + w.loyalty·충성(원소속·프랜차이즈) + w.home·연고/선호팀 + 0.05·rand + 면담bias
선수는 최고 offerScore 수락. 동점·근소차는 원소속 우선.
※ FA **시장 입찰** offerScore는 6항(money/win/play/loyalty/home/rand+talkBias) — 계약연수(wYrs) 항은 아직 미구현(AI 오퍼가 연수를 변별 안 함).
```

### 2.5c 재계약 협상 3택 ★ — 2026-06-25 구현
> "재계약 연봉이 시장가 일괄이라 협상의 폭이 없다"(리뷰) → **내 만료 선수 재계약 시 3택**. `engine/salary.resignOptions`.

| 택 | 연봉 | 연수 | 효과 |
|---|---|---|---|
| **표준** | 시장가 | 3년 | 기본 |
| **후하게** | +15%(개인상한 클램프) | 나이적합(어림 5·중간 4·**노장 2**) | 시장가↑ → 불만('money') 차단·길게 묶기. 캡 부담. |
| **짧게** | −15% | 2년(노장 1) | 싸게 — 곧 재협상(저평가 불만 위험·빨리 FA) |

- **나이 적합(wAge)**: 후하게도 노장(32+)엔 2년만 — "34세에게 5년" 방지(나이 항을 재계약 택에 구현).
- 효과는 기존 시스템으로 자연 발생: 연수=FA 시점, 후하게(≥시장가)=불만 차단(+면담 'raise' 공약 이행 고리, 추후),
  짧게=조기 FA + 저평가 불만. 캡/개인상한은 `maxSalaryFor` 클램프 + 화면 `canAfford` 게이트.
- UI: 앱 `app/contracts.tsx`(행 누르면 3택 액션시트) + 콘솔 연봉 탭(선수별 3택 표시). 가드 `tools/_ev_resign.ts`.

### 2.5b 선수별 FA 성향 (이적 동기 차등) ★ — 2026-06 구현

현실의 FA는 선수마다 동기가 다르다. "돈 좇는 선수 / 우승 좇는 선수 / 남고 싶은 선수 /
출전 보장 원하는 선수 / 연고(드림팀) 가려는 선수." 이를 **선수별 가중치 프로필 `faPref`** 로 모델.

- **생성(결정론)**: `rollFAPref(rng, teamCount)` — 선수 **id 기반 별도 RNG**로 생성해 리그 시드
  메인 스트림을 건드리지 않음(기존 경기/스탯 결과 불변, FA 의사결정만 변화). `makePlayer`/`makeProspect`.
- **아키타입 5종 + 분포**: 머니 34% / 윈나우(우승권) 16% / 충성 26% / 출전 14% / 연고 10%.
  각자 한 동기를 크게(0.42~0.55) 두고 나머지는 옅게 + 개인 노이즈 → 정규화(합 1).
- **연고/선호팀**: `preferredTeamId` 1곳. `home` 가중이 큰 연고형이 강하게 끌림.
- **우승권 신호(prestige)**: 직전 시즌 정규순위 + 우승 여부(`data/offseason.ts teamPrestige`, 우승=1).
  offerScore의 `win` 항 = `0.7·전력 + 0.3·prestige`.

**밸런싱 기록 (200시즌 + 24유니버스×40시즌 검증):**
- 초기 시도(prestige 0.55, 윈나우 22%): **부익부 폭주** — parity std 10.8→16.9, 왕조 13→17연패.
  원인: 우승→prestige↑→우승권FA가 강팀에 몰림→계속 우승. 캡으로 강팀이 선수를 풀어도 우승권FA로
  싸게 리로드해 캡이 무력. **prestige 자기강화 항이 주범.**
- 조정(prestige 0.30, 윈나우 16%↓·충성 26%↑): parity std 4.55, 왕조 평균 8(최대 12),
  **순위 지속성 r = −0.08±0.40(서열 고착 없음)**, 우승경험 6.3/7, 반등 100%. → 채택.

## 2.7 FA 수락 = 점수(100) → 확률 모델 (✅ 구현 2026-06-26, 사용자 결정)

> **구현됨**: `engine/faMarket`(offerScore relT 항·`acceptProb` 완만 S곡선 `SIT_FLOOR 0.22`/`CERTAIN 0.60`·`SIT_OUT 0.14`),
> `data/offseason.resolveFAMarket`(argmax→정렬·롤·fallback·SIT, 로컬 affinity 친구연쇄, `relationBonds()` 컨텍스트).
> 측정: **장기 A/B(30시즌×8, 관계 격리)** — 관계 OFF std 3.42 vs ON 4.06(친구 연쇄가 컨텐더 집중)을 발견해 **rel 가중 ~0.03·REL_SCALE_FA 6**으로
> 재튜닝 → ON std **3.12**(OFF보다 낮음·왕조 max 8·r −0.18·1위점유 31%·반등 100%) = 관계가 집중 안 시키는 은은한 타이브레이커.
> simKovo 불변·무결성 0(simAudit/simFaDup)·4시나리오(`_dv_fa_relations`)·205테스트.

> 기존 `offerScore` **argmax(최고 점수 팀에 무조건 입단)** 을 **점수→확률→정렬·롤·fallback**으로 교체한다.
> "왜 이 팀에 갔나"가 투명한 가산/감점 점수가 되고, 인간관계망(RELATIONSHIP_SYSTEM)이 그 점수의 한 항(±)으로 들어간다.
> **순차 결정론 골격(`resolveFAMarket`)은 유지** — argmax만 확률 해석으로 바꾼다(대공사 아님).

### 2.7.1 오퍼 점수 (player, team) ∈ [0,100]
선수별 가중치(`faPref.w` — 2.5b, 합 ~1)로 각 요인 점수를 합산해 100점화. **요인(가산/감점)**:
| 요인 | 방향 | 출처 |
|---|---|---|
| 우승권(전력+prestige) | + | winT (2.5) |
| 연봉(제시/요구) | + | moneyT |
| 장기근속·원소속·프랜차이즈 | + | loyT |
| 주전 보장(포지션 구멍) | + | playT |
| **좋은 팀메이트 / 싫은 선수** | **±** | relT = `teamAffinity`(RELATIONSHIP §2) — 친구 +, 라이벌/앙숙 − |
| 면담(내 팀) | + | talkBias |
```
score = 100 × ( w.money·moneyT + w.win·winT + w.loyal·loyT + w.play·playT + w.home·homeT + w.rel·relT ) + talkBias·k
clamp [0,100]   // relT가 −면 감점(싫은 선수 있는 팀 점수↓). 선수 성향(w)이 "의리파/우승파"를 가른다
```

### 2.7.2 점수 → 확률 (완만한 S곡선 — 사용자 결정)
`P(수락) = smoothstep(SIT_FLOOR, CERTAIN, score)` — `t=clamp((score−FLOOR)/(CEIL−FLOOR),0,1); P=t²(3−2t)`.
- `score ≤ FLOOR`(예 30) → P≈0(거의 거절), `score ≥ CEIL`(예 80) → P≈1(거의 확정), 중간은 확률적(업셋 드라마).
- 높은 점수는 거의 확정·중간만 확률 → 강팀이 대체로 이기되 가끔 업셋(parity 안전).

### 2.7.3 여러 오퍼 해결 (선수 1명 기준)
1. **오퍼 팀 모음**: 그 포지션을 원하고(gap>0 또는 내 지명) **그 순간** 슬롯·캡·자금 가능한 팀만.
2. 각 팀 score→P 계산(점수는 **그 시점 로스터** 기준 — 친구 연쇄 반영).
3. **오퍼 0개 → 미입단**(FA 잔류 — 팀이 안 원해서. 양방향).
4. **최고 점수 < SIT_FLOOR → 시즌 아웃**(잔류, 사용자 결정 — "그럴 바엔 안 뛴다", 드물게).
5. 아니면 **P 내림차순 정렬 → 위에서부터 시드 롤 → 첫 성공 팀 입단**.
6. **전부 실패 → 가장 높았던 P 팀 입단**(fallback).
7. **오퍼 1팀 → fallback상 결국 그 팀**(전부 실패해도 최고=그 팀) → 자동 입단(단 4의 SIT_FLOOR는 적용).

### 2.7.4 엣지 케이스 / 동시성 (순차 결정론으로 해소)
실제 병렬 아님(순수 함수). 위험 = **논리적 동시성**(한 번에 여러 선수·팀, 공유 자원=슬롯·캡·"한 선수 한 팀"). 해법 = **고정 순서 + 매 선수 가용성 갱신**:
| 엣지 | 처리 |
|---|---|
| 같은 선수 ↔ 여러 팀 | 그 선수 1회 해결 → 한 팀만. 나머지 팀은 다음 선수로 |
| 같은 팀 ↔ 여러 선수 | **OVR 내림차순 순차** — 계약 시 그 팀 캡·슬롯 즉시 차감 → 다음 선수는 꽉 찬 팀 제외(이중계약·캡초과 0) |
| fallback 최고 팀이 그새 꽉 참 | 해결 시점 **계약 가능 팀만 재필터** 후 정렬·롤 |
| 친구 연쇄(A 입단→B의 그 팀 점수↑) | 점수를 **그 시점 로스터**로 계산 → "친구 따라간다" 자연 발생. 순서 고정(OVR)이라 결정론 |
| 내 팀 미리보기=결과 | 같은 시드·순서 → FA 센터 예상==실제(기존 보장 유지) |
| 보상선수 | 행선지 확정 후 적용(순서 무관) |
| 로스터 하한 | 시즌 아웃·이탈 누적으로 부족 시 드래프트·신인 충원(ROSTER_MIN 게이트) — SIT_FLOOR 낮게 둬 드물게 |
| 외인/아시아 | 관계·이 모델 비대상(별 트랙) |
| 결정론 | 모든 롤 `seed(playerId:season[:team])` → 같은 세이브 같은 결과(리플레이) |

### 2.7.5 밸런스 관문
확률·시즌아웃은 **변량(업셋)·로스터 구멍**을 만들 수 있어 **parity 측정 필수**(`simLeague` 다중 유니버스 — 도입 전과 동등). 깨지면 S곡선 기울기·SIT_FLOOR·w.rel 하향. KOVO는 경기 무파급(불변). 미리보기=결과·결정론 회귀.

> 구현은 RELATIONSHIP_SYSTEM Phase 2(이 §2.7이 그 phase의 상세). `engine/faMarket.offerScore`→`offerScoreOf(0~100)`+`acceptProb`,
> `data/offseason.resolveFAMarket` 입찰 루프를 argmax→정렬·롤·fallback·SIT로 교체. 계수 전부 placeholder(측정 튜닝).

### 3.1 AI 픽 우선순위 — 3티어 (2026-06-25 개선)
> "OVR만 높다고 뽑지 않는다." AI는 자기 팀 사정에 맞춰 뽑되, 세대급 재능은 예외로 무조건 잡는다.
> `engine/draft.ts` `pickWithReason` — 순수 함수, 결정론. 픽마다 **사유(reason)** 를 함께 반환(라이브 콘솔 표시용).

순번이 오면 남은 유망주 중에서 **아래 순서로** 고른다:

1. **슈퍼 유망주 예외 (BPA, reason=`super`)** — 남은 후보에 **특급 유망주**(`prospectValue ≥ SUPER_PV=81`)가
   있으면 **포지션 필요와 무관하게** 그중 최고를 잡는다(가치=`prospectValue`×감독성향×스카우트).
   세대급 재능은 자리가 차 있어도 놓치지 않는다(현실 BPA).
   > **컷 정의 주의**: maxPot(스탯 14개 중 최대) 단독은 **포화**(클래스 71%가 ≥88 — 누구나 한 스탯은 99 근처)라
   > "특급"을 못 가린다. **드래프트가치(`prospectValue`=현재OVR×0.4+maxPot×0.6)** 로 정의 → 클래스 **상위 ~10%**
   > 만 특급(측정 `_tmp_pv`: pv≥80=18%·≥82=3%, median 75.6). 라이브 콘솔로 "전부 특급 BPA"를 잡아 교정(2026-06-25).
2. **포지션 필요 (need, reason=`need`)** — 특급이 없으면, **부족 포지션(gap>0)** 후보만 보고 그중 최고
   (`prospectValue`×`needWeight(gap)`×감독성향×스카우트). 자리가 남는 포지션은 이 단계에서 안 뽑는다(`aiFillFromPool`과 동일 정책).
3. **필요 없음 → OVR + 성격 (best, reason=`best`)** — 부족 포지션이 하나도 없으면(이상 구성 충족),
   **현재 실력(OVR)과 성격**으로 고른다(`overall`×`personalityFactor`×감독성향×스카우트).
   - **성격(personalityFactor)** = 멘탈(집중·기복·VQ 평균) + 특성(긍정 가산/부정 감산). 범위 ~0.85~1.15(소폭).
     리더·클러치·노력형 같은 좋은 성격은 동급에서 우대, 새가슴·유리몸은 기피. *placeholder — 측정으로 튜닝.*

> `prospectValue`(포텐 비중 0.6)는 1·2티어(육성 베팅)에, `overall`(현재 실력)은 3티어(즉시 전력+성격)에 쓴다.
> 인간 단장의 위시리스트(reason=`wish`)는 모든 티어보다 우선(순번 내에서 가능한 첫 위시).
> 측정 가드 `tools/_ev_draftpick.ts`(슈퍼 무조건 선발·필요 포지션 우선·무필요 시 성격 영향·결정론·A/B).

### 3.2 라이브 드래프트 (sim-web 콘솔 + 앱)
> 단장 업무 하이라이트를 **보는 재미**로(관전형 1순위). 엔진은 순수 해석 그대로, **UI가 순번을 한 픽씩 재생**.
- **순위별 팀 설정(콘솔)**: 28팀을 각 순위(1위~꼴찌)에 배치 → 로터리 가중(하위 우선)·지명 순번이 그 설정에서 산출.

  공개. 각 픽에 **누가·누구를·왜**(super/need/best/wish) 표시. 결정론(같은 설정=같은 결과).
- 앱(`app/draft-live.tsx`, ✅ 구현 2026-06-25): 미리보기(draft.tsx)와 동일 입력 → 동일 결과. 내 차례 ★강조 +

## 8. 밸런싱 체크리스트
