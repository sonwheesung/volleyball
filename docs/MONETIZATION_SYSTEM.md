# 수익화 시스템 (MONETIZATION_SYSTEM) — 광고 · IAP · 엔타이틀먼트

> 무료 출시 + 광고(ad-supported) + 서포터 팩(광고 제거 포함) + DLC(월드컵) 모델의 정본.
> **설계 확정 2026-06-30**(독립 리뷰 1.5단계 거침). 구현은 추후(EAS 개발 빌드 필요 — 네이티브 모듈).
> 핵심 원칙: **광고는 "뜬다(ad-supported)"만 허용, "봐야 넘어간다(ad-gated)"는 금지.** 관전형 1순위·오프라인·안티과금을 깨지 않는다.

## 1. 모델 한 줄

**무료로 받아 풀게임을 하고, 광고가 뜬다. ₩5,000 "광고 제거"를 사면 광고가 사라진다. 월드컵은 별도 DLC.**
신생 인디 + 니치 장르라 **도달(무료)** 을 최대화하고, 광고 제거·DLC가 같은 IAP 인프라(`lib/iap`)를 공유한다.

## 2. 설계 원칙 (이 문서가 지키는 것)

1. **관전형 1순위 불가침** — 광고는 **자동진행·경기 관전의 흐름을 막지 않는다.** 시즌 진행은 광고 응답에 의존하지 않는다.
2. **오프라인 보존** — 네트워크 없어도(지하철·비행기) 플레이·시즌 진행이 멈추지 않는다. 광고는 못 뜨면 그냥 스킵.
3. **안티과금** — 광고제거=편의, DLC=콘텐츠, 서포터=코스메틱. **전부 밸런스 무영향.** 보상형 광고로 전력·스카우팅·자금 이점을 주지 않는다(advantage 금지).
4. **무저장 결정론 격리** — 엔타이틀먼트는 **표시/가용성만** 게이트한다. 시드·리플레이엔 절대 안 들어간다. 엔진은 store 엔타이틀먼트를 직참조하지 않고 명시 파라미터로만 받는다(remove_ads·supporter는 엔진이 안 읽음, worldCup만 WORLDCUP_SYSTEM의 epoch+forward-only로 격리).
5. **유저 관대** — 오프라인 싱글이라 환불 회수가 구조적으로 불가능. 안티치트에 힘 빼지 않는다(false-negative로 정당 구매자 잠그는 게 더 나쁨).

## 3. 광고 (ad-supported)

**배치 = 플레이어가 능동적으로 누르는 "시즌 시작하기" 버튼**(오프시즌 → 새 시즌 킥오프, 2026-06-30 확정).
자동진행·경기 관전 흐름은 **절대 안 건드린다** — 이건 passive 인터럽트가 아니라 player가 의도적으로 새 해를 여는 자연스러운 분기점이라 관전형 1순위와 충돌하지 않는다(독립 리뷰가 반대한 "자동진행 가로채기"와 다름, §9).

- **첫 시즌 무광고** — 첫인상은 깨끗하게(온보딩 마찰 0).
- **이후 "시즌 시작하기" 버튼 탭 시 동영상 광고**(전면/리워드) → 광고 후 새 시즌 시작.
- **오프라인이면 광고 없이 즉시 시작(하드 블록 금지)** — 광고 로드 실패로 시즌이 안 열리면 오프라인 기둥 위반. 못 뜨면 바로 진행. (이때 "비행기모드 광고 회피" 누수가 생기나 싱글플레이라 무시 — 유저 관대, §2.5.)
- **서포터 팩(광고제거) 구매자 = 버튼 눌러도 광고 없이 즉시 시작.**
- **경기 보드·랠리 관전 중 / 자동 시즌진행 중에는 광고 절대 금지**(경기 진행과 무관 — 사용자 명시).
- **(선택) 시간 빈도캡** — 몰아보기(짧은 시즌 연타)가 문제되면 직전 광고 후 N분 캡을 손잡이로 추가. 의도적 버튼이라 필수 아님 — 실측 후 결정.
- **(선택) 정적 메뉴 배너 / 앱오픈 광고** — 멈춘 화면 하단·세션당 1회. 경기보드 금지. 추후 검토.
- 🚫 **금지**: ① 경기 관전·자동진행을 가로채는 광고 ② 보상형 광고로 게임 이점(전력·스카우팅·자금) 해금(안티과금 위반).

### 3.1 구현 (2026-06-30)
- **단일 연결점 `lib/ads.ts showSeasonStartAd()`** — 앱에서 광고를 띄우는 유일한 함수. 항상 resolve(reject 없음) → 호출부 `endSeason()`이 광고 결과와 무관하게 실행(스킵·실패·오프라인이어도 진행 하드블록 없음). `removeAds` 소유면 즉시 통과(`setRemoveAds`).
- **게이트 위치** — 새 시즌 commit인 `endSeason()` 호출 직전(`app/draft.tsx`·`app/draft-live.tsx` `onFinish`). 버튼 라벨 "시즌 시작하기". **버튼 없이는 `endSeason` 호출 경로가 없어** "안 누르면 시즌 시작 안 함"이 구조적으로 성립.
- **"첫 시즌 제외"** — 첫 시즌(season 0)은 팀 선택에서 시작(`endSeason` 없음) → 광고 자연 제외. `endSeason`은 시즌 전환(0→1, 1→2…)에서만 → 그때마다 광고.
- **현재 = 스텁**: AdMob은 네이티브라 Expo Go 불가 → `showSeasonStartAd`는 no-op(흐름은 완성, 광고만 안 뜸). EAS 빌드(P0) 후 `lib/ads.ts`의 "=== P2 ===" 블록을 `react-native-google-mobile-ads`로 교체하면 켜짐(호출부 불변).

## 4. IAP / SKU · 상점

전부 **비소모(non-consumable)**. 가격은 잠정. **마이페이지에 "상점" 섹션** 하나로 모은다(2026-06-30 사용자 확정).

| SKU | 표시명 | 가격(잠정) | 내용 | 안티과금 |
|---|---|---|---|---|
| `remove_ads` | **광고 제거** | ₩5,000 | "시즌 시작하기" 버튼 광고 등 전 광고 제거(순수 편의) | ✅ 편의, 밸런스 0 |
| `dlc_worldcup` | **월드컵 시즌 구매** | (별도) | 월드컵 4년 국가대표 차출 콘텐츠(WORLDCUP_SYSTEM). 미구매자 비트동일 | ✅ 콘텐츠, 밸런스 0 |

- **상점 위치**: 마이페이지(`app/(tabs)/mypage.tsx`) 안 "상점" 섹션 — 위 2개를 카드로 나열. 구매 배선은 P2(EAS 후) — 그 전엔 "출시 시 결제 연결" 안내(기존 `supporter.tsx`와 같은 정직 플레이스홀더).
- ~~서포터 팩(₩5,000, 광고제거+코스메틱 통합)·"응원" 프레이밍~~ → **폐기(2026-06-30 사용자 결정)**. 독립 리뷰는 "서포터 응원 프레이밍으로 통합"을 권했으나, 사용자가 **명확한 "광고 제거" 단품 + 상점 2칸**을 선택. 따뜻한 카피보다 직관적 상점 UX 우선(사용자 권한 — 기획 결정). 기존 ₩2,000 코스메틱 서포터 SKU는 상점에서 빠짐(크레딧 화면은 잔존 가능).

## 5. 엔타이틀먼트 (소유 상태)

- **진실의 원천 = 앱스토어**(StoreKit/Play Billing). 비소모성은 플랫폼이 "구매 복원"·소유권을 무료 보장.
- **로컬 캐시** — 소유 SKU를 기존 save의 **설정군(resetSave에서 살아남는 필드)** 에 둔다(`worldCupOwned`·`supporter` 패턴 1:1). `freshSave`에 넣지 않음 → 기변·세이브 초기화에도 소유 보존. SAVE_SYSTEM version/migrate로 추가.
- **구매 복원 버튼**(필수, 스토어 정책) — 재설치·기변 시 스토어 재조회(온라인 1회). 오프라인 재설치면 일단 미소유로 graceful, **진행 데이터는 절대 삭제 금지**(온라인 되면 복원).
- **엔진 격리** — `removeAds`는 광고 표시 토글(`lib/ads setRemoveAds`)일 뿐 엔진 미참조. `worldCup`만 sim 영향이나 WORLDCUP_SYSTEM의 epoch+forward-only+비트동일 A/B로 격리. (구현: `lib/iap getEntitlements`)

## 6. 검증 (RevenueCat 하이브리드)

- **RevenueCat** — 영수증 검증·구매 복원·환불 웹훅·iOS/안드 교차. 무료 티어(매출 ~$2.5k/월). 보일러플레이트 절감용.
- **불변 조건**: **출시·시즌 진행은 RevenueCat 응답에 의존하지 않는다.** RC는 구매·갱신 시에만 통신, 진행경로 밖. 오프라인은 로컬 캐시로.
- (리뷰 소견: 비소모성만이라 RC는 다소 오버킬 — 네이티브 복원이 진실의 원천이면 RC 없이도 가능. 무료·편의라 채택 유지. 미래 구독·클라우드 세이브 때 RC 값이 커짐.)

### 6.1 RevenueCat 쓰면 Supabase 따로 저장 필요? → **아니오**(2026-06-30 확정)
- **RevenueCat이 결제 백엔드 그 자체** — 영수증 검증·엔타이틀먼트 저장·구매 복원·**구매/환불 로그·분석 대시보드**·웹훅을 서버측에서 전부 담당. **결제를 Supabase에 따로 저장할 필요 없다**(이중 저장 = 진실의 원천 2개 → 어긋남 위험).
- **광고 로그 = AdMob 대시보드.** 자체 결제 서버·DB·로그 백엔드 = **불필요.**
- 로컬에 영구 저장할 것 = 소유 엔타이틀먼트 캐시뿐인데, **그것도 RevenueCat SDK가 자체 로컬 캐시로 오프라인까지 처리**(`getCustomerInfo`).
- **Supabase는 결제와 무관** — 나중 게임 데이터 클라우드(세이브 백업·글로벌 명예의전당·랭킹, CLAUDE 8)에만.

### 6.2 예외처리·로깅 (확실하게 — 2026-06-30 구현)
- **모든 IAP/광고 함수는 throw하지 않는다.** 결과를 typed로 반환(`PurchaseResult`: ok / cancelled·network·unavailable·error)하거나 항상 resolve(광고). 예외는 전부 잡아 graceful — **결제·광고 실패가 게임을 멈추거나 크래시시키지 않는다.**
  - 광고(`lib/ads`): 로드 실패·오프라인·모듈 없음 → catch 후 즉시 진행(하드블록 금지) + `logError`.
  - IAP(`lib/iap`): 유저 취소=오류 아님(조용히), 네트워크/미가용/기타 구분해 UI가 친절히 안내. `initIap`·`loadEntitlements` 실패해도 미소유로 graceful.
- **로깅**: 단일 경량 로거 `lib/log`(`logEvent`/`logError`, dev 콘솔). **결제 진실 로그=RevenueCat, 광고=AdMob 대시보드**(서버측·유실 없음) — 자체 로그 저장소 안 만든다(local-first).

## 7. 빌드 전제

- IAP·AdMob·RevenueCat 전부 **네이티브 모듈 → Expo Go 불가.** **EAS 개발 빌드** 필요(CLAUDE 8 "네이티브 모듈 추가 시 사전 확인").
- SDK 후보: `react-native-google-mobile-ads`(AdMob), `react-native-purchases`(RevenueCat) 또는 `react-native-iap`(네이티브 단독).

## 8. Phase 로드맵

- **P0** EAS 개발 빌드 셋업 + 네이티브 모듈 사전확인(빌드 통과 확인).
- **P1** 광고 — ✅ **흐름·게이트 구현 완료**("시즌 시작하기" 버튼 → `showSeasonStartAd()` → `endSeason()`, 첫 시즌 제외·오프라인 즉시진행). 📋 **AdMob SDK 연결은 EAS 후**(`lib/ads.ts` 스텁 교체). **서버 무관.**
- **P2** IAP `supporter_pack`(광고 제거 포함) + 로컬 엔타이틀먼트 캐시(save 설정군) + 구매 복원.
- **P3** RevenueCat 연결(검증·복원·환불). 진행경로 불간섭 유지.
- **P4** `dlc_worldcup` — 같은 IAP 인프라 재사용(WORLDCUP_SYSTEM 연동).
- **P5(추후)** Supabase 클라우드 세이브·글로벌 명전(CLAUDE 8).

## 9. 독립 리뷰 기록 (1.5단계, 2026-06-30)

- **기각된 안 — "시즌 진행 광고 하드 게이트"(광고 안 보면 진행 불가)**: 리뷰가 강하게 반대.
  - 이유: ① **관전형 1순위 위반**(가장 손이 안 가야 할 자동진행 지점에 강제 인터럽트, CLAUDE 11) ② **오프라인 치명타**(광고 로드 실패=게임 멈춤) ③ **비행기모드 익스플로잇**(무광고=무료통과 폴백 → 정직한 유저만 광고 보는 벌금) ④ **몰아보기 과세**(20시즌=광고 20번). "advantage 아니니 OK"는 **잘못된 잣대** — 게이트는 밸런스가 아니라 더 윗줄 기둥(관전형·오프라인)을 깬다.
  - → **배치로 해소(§3)**: 광고를 자동진행이 아니라 **플레이어가 능동적으로 누르는 "시즌 시작하기" 버튼**에 붙인다(2026-06-30 사용자 확정). 의도적 분기점이라 관전 흐름·무푸시를 안 깬다. + 오프라인이면 광고 없이 즉시 시작(하드 블록 제거)·경기 진행 무관. 리뷰가 반대한 건 "passive 자동진행 가로채기"였고, 의도적 버튼은 거기 해당 안 됨.
- **채택된 교정**: ① 엔타이틀먼트 표시 전용·결정론 격리 ② 환불 유저 관대 ③ 광고는 의도적 버튼(§3).
- **리뷰 권고 중 기각**: "서포터 팩 통합 + 응원 프레이밍"(①②) → 사용자가 **"광고 제거" 단품 + 마이페이지 상점 2칸**(광고제거·월드컵)으로 결정(2026-06-30). 사유: 직관적 상점 UX 우선(기획 권한). 맹종 안 함.
- **유지(리뷰가 "써도 무방")**: RevenueCat 하이브리드 — 진행경로 불간섭 조건 하에.

## 11. 다이아(consumable) + 특별 훈련 (2026-06-30, 사용자 결정 — 안티과금 기둥 반전)

> ⚠ **기둥 반전 명시**: 본 문서 §2.3·§4는 "전부 비소모·보상형 광고로 이점 금지·밸런스 무영향"을 못박았고
> CLAUDE 기둥3은 안티과금이다. **사용자가 이를 명시적으로 뒤집어** 소비성 재화(다이아) + 다이아로 **선수 강화**
> (포텐 상향) + **광고→다이아**를 도입한다(2026-06-30, 여러 차례 P2W 영향 고지 후 결정). 취소선 보존:
> ~~§2.3 보상형 광고 advantage 금지~~ · ~~§4 전부 비소모~~ → 다이아는 **소비성 IAP** + advantage 재화.
> 시뮬 근거: 포텐 상향은 장기 인플레 버전(`tools/simSpecialTrain`: 천장 돌파 시 피크 OVR Δ+3.19·99 수렴).
> 99 상한 + **선수당 시즌 1회** + 300/회로 속도를 묶는다.

### 11.1 다이아 용도·획득
- **용도(소비)**: **오직 특별 훈련**(300 💎/회). DLC·광고제거는 다이아 아님 — **즉시 결제(비소모 IAP)** 유지(§4).
- **획득**: ① 다이아 구매(소비성 IAP — 100개 ₩1,000·500개 ₩4,800 등 묶음, 스토어 상품) ② **업적 보상**(난이도별
  10~1000, `engine/achievements.ts ACH_REWARD`/`achReward`) ③ **보상형 광고**(30 💎/광고, 일일 5회 캡=150/일).
- **광고 정책 예외**: §2.3은 "보상형 광고 advantage 금지"였으나 본 결정으로 **광고→다이아→특별훈련(강화) 허용**.
  단 광고는 여전히 **선택**(안 봐도 진행 무관, 오프라인 OK) — ad-gated 아님(§9 유지).

### 11.2 특별 훈련 (화면 `app/special-training.tsx`)
- **흐름**: 선수 선택(내 팀) → 능력치 선택(15 훈련 스탯) → 300 💎 차감 → 그 스탯 **포텐셜 +1**(min 99).
- **제약(핵심)**: **선수 1명당 시즌 1회**. `specialTrainedThisSeason`(playerId 집합) 영속, `endSeason`에서 초기화.
  이미 99인 스탯 불가, 잔액<300 불가. 화면에 규칙 설명 명시(§11.2 카피).
- **결정론·세이브**: 포텐 상향은 선수 base.potential에 영구 반영(스냅샷 영속) → 리플레이 보존. 다이아 잔액·
  claimed 업적·이번시즌 훈련목록을 **영속 필드로 추가**(SAVE_SYSTEM version+migrate). 엔진(경기/생산)은 포텐을
  안 읽으므로 경기 결정론 무관 — 성장(training)만 새 천장까지 더 큰다.
- **카피**: "능력치 하나의 성장 한계(포텐셜)를 +1 / 선수당 시즌 1회 / 최대 99 / 능력치마다 포텐 다름 / 영구·환불불가".

### 11.3 업적 → 다이아 (1회 지급)
- 업적은 무저장 재계산(ACHIEVEMENT_SYSTEM)이라, **claimed 집합**(영속)으로 새로 달성된 미수령 업적에 `achReward`
  지급 1회. `endSeason`(또는 업적 화면 진입) 시 reconcile. 구세이브는 기달성분을 claimed로 시드(소급 폭탄 방지)할지
  결정 필요(§11.5).

### 11.4 백엔드 (소비성이라 장부 필요 — §6.1 정정)
- ~~§6.1 "Supabase 결제 저장 불필요"~~ → **정정**: 비소모는 RevenueCat만으로 됐지만 **소비성 다이아는 잔액·획득/
  소비 원장이 필요**. **RevenueCat=IAP 검증/영수증, Supabase=다이아 지갑(balance)·거래 로그(획득:구매/업적/광고,
