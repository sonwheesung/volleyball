## 1. 모델 한 줄

**무료로 받아 풀게임을 하고, 광고가 뜬다. ₩5,000 "광고 제거"를 사면 광고가 사라진다. 월드컵은 별도 DLC.**
신생 인디 + 니치 장르라 **도달(무료)** 을 최대화하고, 광고 제거·DLC가 같은 IAP 인프라(`lib/iap`)를 공유한다.

## 2. 설계 원칙 (이 문서가 지키는 것)

1. **관전형 1순위 불가침** — 광고는 **자동진행·경기 관전의 흐름을 막지 않는다.** 시즌 진행은 광고 응답에 의존하지 않는다.
2. ~~**오프라인 보존** — 네트워크 없어도(지하철·비행기) 플레이·시즌 진행이 멈추지 않는다.~~ → **정정(2026-07-01, 온라인 전환)**:
   "오프라인 기둥" 폐기. **온라인 우선** — 계정(소셜 로그인)·다이아 지갑·결제는 온라인 필요. **단 관전·시즌 시뮬은 캐시로 계속 동작**
   (관전형 보호 — 독립 리뷰 권고). **다이아 사용(전지훈련)·적립(광고/업적)·결제는 무조건 온라인**(서버 확인 후 반영). 정본 `docs/BACKEND_SYSTEM.md`.
3. **안티과금** — 광고제거=편의, DLC=콘텐츠, 서포터=코스메틱. **전부 밸런스 무영향.** 보상형 광고로 전력·스카우팅·자금 이점을 주지 않는다(advantage 금지).
4. **무저장 결정론 격리** — 엔타이틀먼트는 **표시/가용성만** 게이트한다. 시드·리플레이엔 절대 안 들어간다. 엔진은 store 엔타이틀먼트를 직참조하지 않고 명시 파라미터로만 받는다(remove_ads·supporter는 엔진이 안 읽음, worldCup만 WORLDCUP_SYSTEM의 epoch+forward-only로 격리).
5. **유저 관대** — 오프라인 싱글이라 환불 회수가 구조적으로 불가능. 안티치트에 힘 빼지 않는다(false-negative로 정당 구매자 잠그는 게 더 나쁨).

## 3. 광고 (ad-supported)

**배치 = 플레이어가 능동적으로 누르는 "시즌 시작하기" 버튼**(오프시즌 → 새 시즌 킥오프, 2026-06-30 확정).
자동진행·경기 관전 흐름은 **절대 안 건드린다** — 이건 passive 인터럽트가 아니라 player가 의도적으로 새 해를 여는 자연스러운 분기점이라 관전형 1순위와 충돌하지 않는다(독립 리뷰가 반대한 "자동진행 가로채기"와 다름, §9).

- **첫 시즌 무광고** — 첫인상은 깨끗하게(온보딩 마찰 0).
- **이후 "시즌 시작하기" 버튼 탭 시 동영상 광고**(전면/리워드) → 광고 후 새 시즌 시작.
- **오프라인이면 광고 없이 즉시 시작(하드 블록 금지)** — 광고 로드 실패로 시즌이 안 열리면 오프라인 기둥 위반. 못 뜨면 바로 진행. (이때 "비행기모드 광고 회피" 누수가 생기나 싱글플레이라 무시 — 유저 관대, §2.5.)
- **서포터 팩(광고제거) 구매자 = 버튼 눌러도 광고 없이 즉시 시작.**
- **경기 보드·랠리 관전 중 / 자동 시즌진행 중에는 광고 절대 금지**(경기 진행과 무관 — 사용자 명시).
- **(선택) 시간 빈도캡** — 몰아보기(짧은 시즌 연타)가 문제되면 직전 광고 후 N분 캡을 손잡이로 추가. 의도적 버튼이라 필수 아님 — 실측 후 결정.
- **(선택) 정적 메뉴 배너 / 앱오픈 광고** — 멈춘 화면 하단·세션당 1회. 경기보드 금지. 추후 검토.
- 🚫 **금지**: ① 경기 관전·자동진행을 가로채는 광고 ② 보상형 광고로 게임 이점(전력·스카우팅·자금) 해금(안티과금 위반).

### 3.1 구현 (2026-06-30)
- **단일 연결점 `lib/ads.ts showSeasonStartAd()`** — 앱에서 광고를 띄우는 유일한 함수. 항상 resolve(reject 없음) → 호출부 `endSeason()`이 광고 결과와 무관하게 실행(스킵·실패·오프라인이어도 진행 하드블록 없음). `removeAds` 소유면 즉시 통과(`setRemoveAds`).
- **게이트 위치** — 새 시즌 commit인 `endSeason()` 호출 직전(`app/draft.tsx`·`app/draft-live.tsx` `onFinish`). 버튼 라벨 "시즌 시작하기". **버튼 없이는 `endSeason` 호출 경로가 없어** "안 누르면 시즌 시작 안 함"이 구조적으로 성립.
- **"첫 시즌 제외"** — 첫 시즌(season 0)은 팀 선택에서 시작(`endSeason` 없음) → 광고 자연 제외. `endSeason`은 시즌 전환(0→1, 1→2…)에서만 → 그때마다 광고.
- **현재 = 스텁**: AdMob은 네이티브라 Expo Go 불가 → `showSeasonStartAd`는 no-op(흐름은 완성, 광고만 안 뜸). EAS 빌드(P0) 후 `lib/ads.ts`의 "=== P2 ===" 블록을 `react-native-google-mobile-ads`로 교체하면 켜짐(호출부 불변).

## 5. 엔타이틀먼트 (소유 상태)

- **진실의 원천 = 앱스토어**(StoreKit/Play Billing). 비소모성은 플랫폼이 "구매 복원"·소유권을 무료 보장.
- **로컬 캐시** — 소유 SKU를 기존 save의 **설정군(resetSave에서 살아남는 필드)** 에 둔다(`worldCupOwned`·`supporter` 패턴 1:1). `freshSave`에 넣지 않음 → 기변·세이브 초기화에도 소유 보존. SAVE_SYSTEM version/migrate로 추가.
- **구매 복원 버튼**(필수, 스토어 정책) — 재설치·기변 시 스토어 재조회(온라인 1회). 오프라인 재설치면 일단 미소유로 graceful, **진행 데이터는 절대 삭제 금지**(온라인 되면 복원).
- **엔진 격리** — `removeAds`는 광고 표시 토글(`lib/ads setRemoveAds`)일 뿐 엔진 미참조. `worldCup`만 sim 영향이나 WORLDCUP_SYSTEM의 epoch+forward-only+비트동일 A/B로 격리. (구현: `lib/iap getEntitlements`)

> 아래 서술은 역사 보존. 정본 `docs/BACKEND_SYSTEM.md`.
- **RevenueCat이 결제 백엔드 그 자체** — 영수증 검증·엔타이틀먼트 저장·구매 복원·**구매/환불 로그·분석 대시보드**·웹훅을 서버측에서 전부 담당. **결제를 Supabase에 따로 저장할 필요 없다**(이중 저장 = 진실의 원천 2개 → 어긋남 위험).
- **광고 로그 = AdMob 대시보드.** 자체 결제 서버·DB·로그 백엔드 = **불필요.**
- 로컬에 영구 저장할 것 = 소유 엔타이틀먼트 캐시뿐인데, **그것도 RevenueCat SDK가 자체 로컬 캐시로 오프라인까지 처리**(`getCustomerInfo`).
- **Supabase는 결제와 무관** — 나중 게임 데이터 클라우드(세이브 백업·글로벌 명예의전당·랭킹, CLAUDE 8)에만.

### 6.2 예외처리·로깅 (확실하게 — 2026-06-30 구현)
- **모든 IAP/광고 함수는 throw하지 않는다.** 결과를 typed로 반환(`PurchaseResult`: ok / cancelled·network·unavailable·error)하거나 항상 resolve(광고). 예외는 전부 잡아 graceful — **결제·광고 실패가 게임을 멈추거나 크래시시키지 않는다.**
  - 광고(`lib/ads`): 로드 실패·오프라인·모듈 없음 → catch 후 즉시 진행(하드블록 금지) + `logError`.
  - IAP(`lib/iap`): 유저 취소=오류 아님(조용히), 네트워크/미가용/기타 구분해 UI가 친절히 안내. `initIap`·`loadEntitlements` 실패해도 미소유로 graceful.
- **로깅**: 단일 경량 로거 `lib/log`(`logEvent`/`logError`, dev 콘솔). **결제 진실 로그=RevenueCat, 광고=AdMob 대시보드**(서버측·유실 없음) — 자체 로그 저장소 안 만든다(local-first).

### 11.2 전지훈련 (화면 `app/training-camp.tsx`) — 오프시즌 해외 캠프
- **서사**: 오프시즌, 선수를 해외 캠프로 보내 집중 훈련(현실 KOVO 전지훈련). 이름·타이밍·1회 제약이 서사로 정당화됨.
- **흐름**: (오프시즌/시즌시작 전) 선수 1명 선택 → **올릴 능력치 여러 개** 선택 → 300×개수 💎 차감 → 각 능력치 **현재 +1 & 포텐 +1**(min 99).
- **제약(핵심)**: **선수 1명당 오프시즌 1회**(한 캠프에서 다부위). `campTrainedThisOffseason`(playerId 집합) 영속, 새 시즌 시작 시 초기화.
  이미 99 스탯 불가, 잔액<300×개수 불가. 화면에 규칙 명시.
- **오프시즌↔개막전 게이트(2026-07-04 사용자 요청)**: 일정 화면(`app/(tabs)/schedule.tsx`)에서 **오프시즌엔 전지훈련 카드만, 다음 경기(개막전)는 숨긴다** — "전지훈련이 보이면 다음 경기가 아니어야"(둘 동시 노출 금지). 오프시즌 = `currentDay===0 && campDoneSeason!==season`. 전지훈련 카드의 **"전지훈련 마치고 개막전으로 →"**(confirm) → `finishCamp()`가 `campDoneSeason=season` 세팅 → 그때 다음 경기 노출(개막전은 여전히 "경기 시작"이 진행, `currentDay` 불변). `campDoneSeason`은 **시즌번호라 새 시즌 자동 리셋**(=season이면 완료). 영속 필드(SAVE §1). 마친 뒤엔 그 시즌 전지훈련 불가(캠프는 day0 한정이라 정합). 검증: 에뮬 — 오프시즌 전지훈련만 표시 → 마치기 → 개막전(10/18) 노출·전지훈련 사라짐.
- **타이밍=오프시즌(day0)이 핵심**: 시즌 생산은 `data/production.allProdRows`가 baseVersion 키로 **시즌 전체를 한 번에 캐시**
  → 시즌 중 현재 스탯을 바꾸면 ①전 시즌 재시뮬(느림) ②치른 경기 소급 변경(결정론 붕괴). **경기 0개인 시즌시작(day0)에만 적용**해
  재시뮬·소급 0 보장(검증된 제약). 다음 시즌부터 반영.
- **결정론·세이브(기록입력 모델)**: 부위 상향을 base에 적용하되 **`campLog`(시즌·선수·스탯 기록)을 영속**한다 — 시드 폴백
  재생에서 재적용 가능(silent 굽기 금지, 사용자 지적). 정상 로드=영속 base 사용(재적용 안 함)·시드 폴백=campLog 재적용(상호배타).
  경기 엔진은 현재 스탯만 읽고 포텐 무읽음 → 경기 결정론은 현재스탯 변화에만 의존(그래서 day0 적용).
- **카피**: "오프시즌 해외 캠프 / 능력치 여러 개 현재+1·포텐+1 / 선수당 오프시즌 1회 / 최대 99 / 다음 시즌부터 / 영구·환불불가".
