## 1. 모델 한 줄

**무료로 받아 풀게임을 하고, 광고가 뜬다. ₩5,000 "광고 제거"를 사면 광고가 사라진다. 월드컵은 별도 DLC.**
신생 인디 + 니치 장르라 **도달(무료)** 을 최대화하고, 광고 제거·DLC가 같은 IAP 인프라(`lib/iap`)를 공유한다.

## 2. 설계 원칙 (이 문서가 지키는 것)

1. **관전형 1순위 불가침** — 광고는 **자동진행·경기 관전의 흐름을 막지 않는다.** 시즌 진행은 광고 응답에 의존하지 않는다.
2. ~~**오프라인 보존** — 네트워크 없어도(지하철·비행기) 플레이·시즌 진행이 멈추지 않는다.~~ → **정정(2026-07-01, 온라인 전환)**:
   "오프라인 기둥" 폐기. **온라인 우선** — 계정(소셜 로그인)·다이아 지갑·결제는 온라인 필요. **단 관전·시즌 시뮬은 캐시로 계속 동작**
   (관전형 보호 — 독립 리뷰 권고). **다이아 사용(전지훈련)·적립(광고/업적)·결제는 무조건 온라인**(서버 확인 후 반영). 정본 `docs/BACKEND_SYSTEM.md`.
3. **안티과금** — 광고제거=편의, DLC=콘텐츠, 서포터=코스메틱. **전부 밸런스 무영향.** 보상형 광고로 전력·스카우팅·자금 이점을 주지 않는다(advantage 금지).
4. **무저장 결정론 격리** — 엔타이틀먼트는 **표시/가용성만** 게이트한다. 시드·리플레이엔 절대 안 들어간다. 엔진은 store 엔타이틀먼트를 직참조하지 않고 명시 파라미터로만 받는다(remove_ads·supporter는 엔진이 안 읽음, worldCup만 WORLDCUP_SYSTEM의 epoch+forward-only로 격리).
5. **유저 관대** — 오프라인 싱글이라 환불 회수가 구조적으로 불가능. 안티치트에 힘 빼지 않는다(false-negative로 정당 구매자 잠그는 게 더 나쁨).

## 3. 광고 (ad-supported)

**배치 = 플레이어가 능동적으로 누르는 "시즌 시작하기" 버튼**(오프시즌 → 새 시즌 킥오프, 2026-06-30 확정).
자동진행·경기 관전 흐름은 **절대 안 건드린다** — 이건 passive 인터럽트가 아니라 player가 의도적으로 새 해를 여는 자연스러운 분기점이라 관전형 1순위와 충돌하지 않는다(독립 리뷰가 반대한 "자동진행 가로채기"와 다름, §9).

- **첫 시즌 무광고** — 첫인상은 깨끗하게(온보딩 마찰 0).
- **이후 "시즌 시작하기" 버튼 탭 시 동영상 광고**(전면/리워드) → 광고 후 새 시즌 시작.
- **오프라인이면 광고 없이 즉시 시작(하드 블록 금지)** — 광고 로드 실패로 시즌이 안 열리면 오프라인 기둥 위반. 못 뜨면 바로 진행. (이때 "비행기모드 광고 회피" 누수가 생기나 싱글플레이라 무시 — 유저 관대, §2.5.)
- **서포터 팩(광고제거) 구매자 = 버튼 눌러도 광고 없이 즉시 시작.**
- **경기 보드·랠리 관전 중 / 자동 시즌진행 중에는 광고 절대 금지**(경기 진행과 무관 — 사용자 명시).
- **(선택) 시간 빈도캡** — 몰아보기(짧은 시즌 연타)가 문제되면 직전 광고 후 N분 캡을 손잡이로 추가. 의도적 버튼이라 필수 아님 — 실측 후 결정.
- **(선택) 정적 메뉴 배너 / 앱오픈 광고** — 멈춘 화면 하단·세션당 1회. 경기보드 금지. 추후 검토.
- 🚫 **금지**: ① 경기 관전·자동진행을 가로채는 광고 ② 보상형 광고로 게임 이점(전력·스카우팅·자금) 해금(안티과금 위반).

### 3.1 구현 (2026-06-30)
- **단일 연결점 `lib/ads.ts showSeasonStartAd()`** — 앱에서 광고를 띄우는 유일한 함수. 항상 resolve(reject 없음) → 호출부 `endSeason()`이 광고 결과와 무관하게 실행(스킵·실패·오프라인이어도 진행 하드블록 없음). `removeAds` 소유면 즉시 통과(`setRemoveAds`).
- **게이트 위치** — 새 시즌 commit인 `endSeason()` 호출 직전(`app/draft.tsx`·`app/draft-live.tsx` `onFinish`). 버튼 라벨 "시즌 시작하기". **버튼 없이는 `endSeason` 호출 경로가 없어** "안 누르면 시즌 시작 안 함"이 구조적으로 성립.
- **"첫 시즌 제외"** — 첫 시즌(season 0)은 팀 선택에서 시작(`endSeason` 없음) → 광고 자연 제외. `endSeason`은 시즌 전환(0→1, 1→2…)에서만 → 그때마다 광고.
- **현재 = 스텁**: AdMob은 네이티브라 Expo Go 불가 → `showSeasonStartAd`는 no-op(흐름은 완성, 광고만 안 뜸). EAS 빌드(P0) 후 `lib/ads.ts`의 "=== P2 ===" 블록을 `react-native-google-mobile-ads`로 교체하면 켜짐(호출부 불변).

### 3.2 AdMob 실연동 (2026-07-05, 독립 리뷰 반영 — general-purpose)
> 리뷰 결론: 계획은 근본적으로 건전(광고=진행에 인접, 게이트 아님). **과설계 1곳=SSV / 과소 1곳=시즌광고 빈도캡.** 아래로 확정.

- **SSV 보류(클라 지급이 출시 경로)**: 보상형 다이아는 ~~AdMob SSV 서버검증~~ → **`onUserEarnedReward`(SDK 콜백) → 기존 `earnDiamonds('ad', adKey)`(멱등 슬롯키)** 로 지급.
  근거(리뷰): ① 싱글플레이·공유경제 없음 → 치팅=자기 세이브만 손해(§2.5) ② **업적 다이아는 이미 클라 신뢰(BACKEND H3)** — 광고만 SSV로 엄격은 앞뒤 안 맞고, 어택커는 더 쉬운 업적 위조로 감 ③ 3중 상한(50/회·30분 쿨다운·하루8=최대 400/일)로 blast radius 제한 ④ SSV는 공개 무인증 콜백+회전키 ECDSA 검증+custom_data 바인딩+콘솔 URL 설정+이중지급 화해까지라 위협 대비 과함. **SSV 설계문(BACKEND §13/H4)은 보존 — 부활 트리거 = 다이아가 교차플레이어 가치(거래·랭킹·토너먼트)를 갖게 될 때.**
- **시즌 시작 = 전면(Interstitial)**(보상 없음 → Rewarded 아님). **로드 하드 타임아웃(3~4s)** — 못 뜨면 즉시 resolve하고 시즌 진행(오프라인·노필 대비, "always resolve"만으론 `load()` 무한대기 위험). 프리로드 후 탭 시 즉시 show(오클릭 방지 — 버튼 확정 후 표시).
- **★ 시즌광고 빈도캡(리뷰 최대 갭)**: 관전형은 시즌을 연타로 넘겨 광고 폭탄 위험(policy '너무 잦음' + 관전형 위반). **직전 전면광고 후 N분 캡(또는 세션당/‑k시즌당 1회)**. `engine/diamonds.ts`의 광고 쿨다운과 별개 — `lib/ads` 메타(시드 무관).
- **보상형 free-faucet 차단(출시 블로커)**: 현 `watchAdForDiamonds`는 **광고 없이 +50 지급** → 반드시 **실제 `onUserEarnedReward` 후에만 지급**. 광고 로드 실패/취소 시 미지급 + **쿨다운 미소모** + "잠시 후 다시" 안내. 오프라인이면 버튼 비활성("온라인에서만").
- **UMP 동의(매 실행)**: `requestConsentInfoUpdate` → `loadAndShowConsentFormIfRequired` 매 부팅. 비EEA는 자동 무표시(다운사이드 0, 누락 시 EEA 정책 위반). ATT(iOS)는 iOS 출시 때만.
- **remove_ads 상호작용(확정)**: 광고 제거 = **시즌 전면광고 끔**(프리로드 전 게이트). **보상형 버튼은 유지**(자발적 opt‑in 혜택이라 제거 대상 아님).
- **개발=테스트 ID 불변식**: `__DEV__`이면 구글 `TestIds`(실 광고단위로 개발 클릭 시 계정 정지). 실 ID는 운영 빌드만. `lib/ads.ts`에 하드 불변식으로.
- **광고 단위(AdMob 콘솔)**: 앱ID `ca-app-pub-2731473780180274~2622190257`(공개 — app.json 플러그인). 보상형 `rewarded_diamonds`·전면 `interstitial_season`(각 ID는 공개 상수). 정산(수익 수령)은 세금정보 필요 → 나중(테스트 광고는 즉시 동작).

- **같은 IAP 인프라 재사용** — `dlc_worldcup`·`remove_ads`와 동일하게 `lib/iap` SKU + RevenueCat 엔타이틀먼트 게이트웨이를 쓴다. 결제·복원·검증 배선을 새로 만들지 않는다.
- **안티과금 유지(§3 기둥)** — DLC = **콘텐츠/경험**이지 전력이 아니다. 엔타이틀먼트는 **표시/가용성만** 게이트하고 **밸런스에 무영향**, 시드·리플레이엔 절대 안 들어간다(월드컵 DLC의 epoch+forward-only+비트동일 격리 원칙과 동일). 상세 설계 때도 이 격리를 반드시 지킨다.
- **SKU id는 아직 미정의** — 후보 플레이스홀더 `dlc_olympics`·`dlc_allstar`(예시일 뿐). **현재 `lib/iap`의 `Sku`는 `remove_ads | dlc_worldcup`뿐**이며 위 두 id는 **아직 정의되어 있지 않다**(실제 착수 시 SKU·엔타이틀먼트 추가).

> **수치 정정(사용자 결정 2026-07-08)**: ~~현재 +2 · 포텐 +7~~ → **현재 +3 · 포텐 +3(대칭)**. 사용자 지시:
> "포텐 +7이 오버밸런스, 현재·포텐을 동일 +2~3으로" → 상단(+3) 채택. **트레이드오프(정직)**: 포텐 +7의 성장-후 실현 폭은
