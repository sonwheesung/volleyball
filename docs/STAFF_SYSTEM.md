# 스태프 계약 시스템 (STAFF_SYSTEM)

> 단장이 **감독·전문 코치·스카우터**를 스태프 예산 안에서 영입/방출한다. (2026-06-09 신설)
> 결정론 유지: 모든 효과는 순수 함수, 풀은 시드 생성, 계약 상태만 세이브.

## 1. 역할과 효과 — 감독=방향, 코치=상한/수명, 스카우터=정보

| 역할 | 효과 | 통합 지점 |
|---|---|---|
| **감독(head)** | 성향(공/수/밸 → 경기)·~~카리스마(타임아웃 기세폭)~~ **→ 능력 3축(경기 운영 `matchOps`·육성 철학 `dvPhilosophy`·리더십 `leadership`)으로 분해(§9 스태프 3.0 Phase A, ✅ 구현 2026-07-20 — Phase A는 `matchOps`만 엔진에 물림[구 charisma 등가], 2축은 생성·표시·영속만·엔진 훅 Phase D **✅ 구현(2026-07-20 §9.6-D — dvPhilosophy=U23 라인업/경기경험, leadership=FORM/불만/면담/건의)**)**·**훈련선호(어떤 스탯이 크는지=방향)** | `coachInfoOf`·`teamFocus` — 영입 시 시드 감독 대체 |
| **전문 코치(assistant)** | **분야별 다른 효과**(아래). 같은 분야 최고 1명만. **팀당 슬롯 `COACH_SLOTS=3`** | `engine/staff.ts staffEffects` → `evolvePlayer`(훈련·노쇠) |
| **스카우터(scout)** | **내 팀 外 능력 공개도↑**(표시 안개 — 타 구단 선수·드래프트 유망주) + **실제 지명 정확도↑**(공개도 낮으면 가치 오판, AI/단장) | `teamScoutReveal` → `draft.tsx`·선수상세·`RosterList`(표시)·`aiDraftPick`(픽 노이즈) |

**전문 코치 분야별 효과** (현실 코치진 — 더 빨리만이 아니라 더 높이/더 오래):

| 분야 | 효과 | 대상 |
|---|---|---|
| 공격/수비/세터(**기량**) | 해당 스탯 **숨은 포텐셜 상한 +α**(역량100=+5) + 성장 속도↑ | 공격→스파이크·서브 / 수비→리시브·디그·블록 / 세터→세팅·VQ |
| **체력** | **노쇠 하락 둔화**(역량100=−45%, 전성기 연장) + 피지컬 속도↑ | jump·agility·staminaMax·staminaRegen (노쇠 지연은 `DECAY_STATS` 전체) |
| **멘탈** | 집중·기복 **포텐 상한↑** + 멘탈 속도↑ | focus·consistency |

> 포텐 상한 상향(`potBonus`)이 핵심 — 선수를 *진짜 더 잘하게* 만들어 경쟁에 반영(속도 가속만으론 약했음).

> **노쇠 지연(ageSlow) 실효 상한 — 삼자 일치(2026-07-15 발견 모드 2차)**: `applyAgingDay`(`engine/aging.ts:39`)가 `Math.min(0.6, ageSlow)`로 **캡 0.6**을 건다. 기량 코치 flat 최대 `ageSlowOf(100)=0.45`, **노쇠억제형(antiaging)은 ×1.4 → 이론상 0.63(rating 100)**이나 코치 rating 상한이 95(`playerToCoach` clamp)라 **실효 최대 ≈0.5985** → 0.6 캡은 **rating≥96에서만 절단**(현 rating 상한선 밖이라 사실상 미발동). `engine/staff.ts`의 `ageSlow` 필드 주석·아래 §8.1 antiaging(×1.4) 행과 정합.

> **등급 단조성 실측·봉인(2026-07-07, 검증=Fable 5)**: 코치無 < 역량60 < 역량95 OVR 단조(Δ≈+1.2/3시즌·공격코치 1명), `potRaise(60)=3·potRaise(95)=5`(산식 `round(rating/100×5)` 정확) — `engine/staff.test.ts` "코치 역량 등급 단조성" 테스트(동일 시드 유망주 3갈래 evolvePlayer, 단조 관계 어서션).

## 6. 감독 생애주기 (STAFF_LIFECYCLE, 2026-06-13 설계)

정적 감독 풀(시드 1회 생성·영원)을 **순환하는 생태계**로. `engine/staffLifecycle.ts`(순수·결정론),
오프시즌에 `data/offseason`이 호출. 선수 은퇴(applyRetirements)와 같은 계층.

### 6.1 노쇠·은퇴
- 감독은 매 오프시즌 `age+1`. 선수보다 늦게 은퇴(현장직은 60대까지) — `coachRetireChance(age)`:
  55세↓ 0 · 56~60 0.06 · 61~65 0.18 · 66~70 0.4 · ~~71+ 0.7 (75+ 0.95)~~ → **71~75 0.7 · 76+ 0.95**(정정 2026-07-15 발견 모드 2차 — `coachRetireChance`는 `age<=75`까지 0.7, else 0.95). 시드 결정론.
- 은퇴 감독은 풀에서 제거. 소속 감독이 은퇴하면 그 팀은 다음 시즌 새 감독 필요(AI 자동 선임).

### 6.2 신규 공급 — 은퇴 선수 → 지도자
- 선수가 은퇴할 때 **VQ(배구 IQ) 높으면**(≥72) 일정 확률로 **전문 코치(assistant)**로 풀 합류
  (`playerToCoach`). 코치 속성은 선수 커리어에서 파생:
  - **분야(specialty)**: 포지션 매핑(`POS_SPECIALTY`) — S→setter · MB→defense · L→defense · OH/OP→attack. ~~(체력/멘탈은 VQ·전성기 길이로 가끔)~~ → **정정(2026-07-15 발견 모드 2차)**: `POS_SPECIALTY`는 setter/defense/attack만 매핑한다 — **은퇴 선수는 stamina/mental 코치를 절대 만들지 않는다**. 체력/멘탈 코치 공급은 오직 **외부 명장**(`makeExternalCoach`, 5분야 균등 시드 유입 §8.1 phase③)뿐.
  - **역량(rating)**: `clamp(round(skillAvg×0.85 + elite + (isLegend?8:0) + rng[-4,4]), 45, 95)`, skillAvg=(VQ+위치선정+반응)/3, elite=상위 은퇴자 보너스(§8.1 phase③). **카리스마**(승격 시 `coachToHead`): `clamp(round(rating×0.6 + starRep×0.3 + rng[-5,5]), 45, 95)`. ~~VQ + 리더 특성 + 명성~~ → **정정(2026-07-15 발견 모드 2차)**: 실제 입력은 코치 rating(≈VQ 주도)과 starRep(명성=레전드 80/일반 40)뿐. **리더 특성(traits)은 미입력** — "리더 특성" 반영은 기지 미결(백로그 next-work-todo "미결1 리더특성"과 연결).
  - **style**: 포지션·플레이 성향(공격형 포지션→attack 편향 등).
- **명성 가산**: 레전드(영구결번)·통산 상위 선수는 카리스마·승격 속도 보너스("스타 출신 감독").

### 6.3 승격 — 코치 → 감독
- 전문 코치는 **명성**(retired 선수 시절 + 코치 경력)이 쌓이면 **감독(head) 풀**로 승격(`promotesToHead`):
  - 트리거: ① 코치로 성공(소속 팀 호성적 누적 = coachRep) **또는** ② 본래 명성 높음(레전드 출신).
    → ~~**미구현·후속(2026-07-15 발견 모드 2차)**: `headWorthiness(a.rating, 50, starRep)` — **coachRep은 상수 50 고정**(호성적 누적 추적 없음).~~ → **✅ 구현·상수 청산(2026-07-20, 스태프 3.0 Phase D §9.6-D)**: `headWorthiness(a.rating, assistantCoachRep(asstCareerLog, a.id, teamCount), starRep)` — **coachRep = 실제 코치 경력 파생**. `assistantCoachRep`(engine/reputation): 무경력=`COACH_REP_BASE`(50, 구 상수 등가 → 무경력 코치 승격률 회귀 보존) + **재직 1시즌당 +3** + 상위 절반 팀 시즌당 +2, 상한 90. 영속 경량 경력 로그 `coachAsstCareerLog: {season, coachId, teamId, teamRank}`(additive, 오프시즌 배정 코치 1행 append, 캡 1000). 즉 오래·상위권에서 지도한 코치일수록 승격 자질↑.
  - 승격 시 head Coach로 재생성(`coachToHead`): matchOps는 산출(§6.2)·style은 분야 기반(attack/defense/balanced)·연봉 책정. **초기 명성(renown)에도 코치 경력 반영(§9.6-D)**: `renown = clamp(renownBase(레전드72·일반38) + (coachRep − 50)×0.4)` — 장수 명코치는 승격 시 초기 명성 프리미엄(coachRep 90 → +16). ~~trainingFocus 계승~~ → **정정(2026-07-15 발견 모드 2차)**: `coachToHead` 호출부(`data/staffLifecycle.ts`)가 **전원 `DEFAULT_FOCUS`(`{primary:[4,6], secondary:[1,10,12]}`)를 넘긴다** — 계승·다양화 없음. **OPEN(측정 필요, 추정 단정 금지)**: 모든 승격 감독이 동일 focus면 장기 팀 훈련 분화가 수렴할 우려 — `sim-league` 장기 focus 분포로 실측 후 판단.
- 즉 경로: **은퇴 선수(고VQ) → 전문 코치 → (성과·명성) → 감독**. 레전드는 코치 단계를 빨리 통과.

### 6.4 경질 (성적 부진)
- **시즌 후 경질(AI)**: 최하위(또는 하위 + 장기 부진)면 그 팀 감독 해촉 → 풀로(또는 은퇴) → 새 감독 선임.
  플레이어 팀은 강제 경질 없음(구단주가 결정 — 기존 hireHeadCoach 교체).
- **시즌 중 경질(2026-06-13 요청)** — **정정(2026-07-15 발견 모드 2차): 플레이어 전용, AI 자동 시즌 중 경질 없음.**
  `firedMidSeason(wins, losses)`(12경기+ & 승률<0.2)는 **플레이어 팀 스태프 화면의 부진 경고 배지(`slumping`)로만** 쓰인다(`app/staff.tsx`).
  실제 경질은 플레이어가 **수동 "경질" 버튼**(`store.fireCoach`→`data/league.fireCoach`)을 눌러야 발생 — AI 팀은 시즌 중 자동 경질하지 않는다(시즌 후 경질만, `advanceCoaches`).
  경질 시 **전문 코치 중 1명이 감독 대행(acting)**으로 임시 지휘(성향·카리스마는 대행 코치 기준, 카리스마 ×0.7) →
  다음 영입 시점(또는 시즌 종료)에 정식 감독 선임. 플레이어는 대행 상태에서 직접 감독을 구한다.
  - 구현: 대행은 `headCoachOverride`에 임시 감독(대행 코치를 head로 캐스팅) 주입, `acting:true` 플래그.
- **정식 감독 재직 중 시장 영입 차단(2026-07-11 테스터 정정)**: 예전엔 감독이 있어도 시장에서 새 감독을
  영입하면 **즉시 무경질 교체**됐다(실수로 감독이 바뀌는 사고). → 이제 정식 감독(`head && !acting`)이 있으면
  감독 시장의 "영입" 버튼을 **비활성화 + 안내**("현재 감독을 먼저 경질해야…"), tryHireCoach도 방어적으로
  알림 후 차단. 즉 **교체 = 경질 → 영입 2단계**로 명시화. 대행/공석(`acting || !head`)은 정식 감독을 세우는
  절차라 영입 그대로 허용. 엔진 무관(UI 게이트만) — `hireHeadCoach` 자체는 불변.

### 6.5 결정론·세이브
- 풀 변화(은퇴·신규·승격)는 시드 기반 순수 변환. 세이브엔 계약 상태 + 풀 델타(생성된 코치)만.
- 선수→코치 변환 id는 `staff:{playerId}` 시드로 결정론. 리플레이 일관.

### 6.6 계약·재계약(감독 FA, 2026-06-13 구현)
- `Coach/AssistantCoach.contractYears`: 영입/재계약 시 부여(3년·시드 2~4년), 매 오프시즌 −1.
- 만료(0) 시: **AI**는 성적·연령으로 재계약(`aiResigns`) 또는 FA로 놓아줌 → 다른 팀이 영입(감독 쟁탈).
  **플레이어**는 스태프 화면에서 `재계약(3년)` 또는 새 감독 영입(기존 감독 FA로). 유예 유지(놓치지 않게).
- 경질 인내심: 꼴찌 1년 45% 확률·하위권 2년 연속 확정(`firedEndSeason`+`bottomStreak`) — 매 시즌 경질 과churn 방지.

**구현 현황(2026-06-13)**: ① 엔진 코어 ✅ ② 오프시즌 통합·세이브 영속 ✅ ③ 계약·재계약·FA ✅
④ 장기 검증 ✅(`tools/simStaffLife.ts` — 100시즌 풀 최소 7·연령 안정·감독 79% 선수출신·FA 활성)
⑤ 시즌 중 대행 ✅(`fireCoach` — 경질 시 최고 역량 전문 코치가 대행, 카리스마 ×0.7, 새 감독 영입 시 대행 제거.
   경질 감독은 그 팀 영구 배제·FA로. 스태프 화면: 부진 경고·경질 버튼·대행 표시·영입 유도).

> — 감독 S **0.2**·A 0.3, 코치 S **0.2**·A 0.3, **시즌당 신규 S/A 유입 0.00**. 원인(코드 확인): 신규 코치는 은퇴선수 유래(`rating=skillAvg×0.8`)·승격

> **함의**: (a) 원계획 **1축 "공급 조임"은 오히려 악화**(공급은 이미 붕괴 중). (b) 진짜 문제는 **장기 스태프 무의미화**(전원 C=단장 결정 소멸·육성 정체) + 초반 경미한 grab-the-best(자연 소멸).
**★ 확정 설계 — 두 축(경쟁 아님, 서로 다른 문제 해결)**. 실측이 밝힌 두 문제를 각각의 축이 담당한다:

| 축 | 담당 문제 | 역할 |
|---|---|---|
| **① 성향(개성)** | 초반 스칼라 지배전략("최고 코치가 늘 정답") | **결정을 재미있게** — 같은 역량도 효과 벡터가 달라 "우리 팀에 맞는 코치"를 찾게 |
| **② 성장 + 재생성** | 장기 C급 수렴 붕괴(실측 — 상위코치 멸종) | **시장을 건강하게** — 선수 출신이 실제 명장이 되는 경로 + 상위 유입으로 효과가 시즌 지나도 유지 |

> **핵심(놓치면 안 됨)**: 성향만 넣으면 "**개성 있는 C급**"만 남아 결정은 재밌어도 효과가 계속 약하다. 성장만 넣으면 "**더 성장한 코치가 다시 정답**"이 돼 지배전략이 재발한다. **둘은 경쟁이 아니라 서로 다른 문제를 푼다 — 둘 다 필수.** 특히 **성장·재생성은 선택 기능이 아니라 실측된 시장 붕괴(C급 수렴)를 해결하는 필수 축**이다(2026-07-03 GPT 명시 동의).

**① 성향 설계** — 같은 분야 코치라도 **효과 벡터가 다르게**(역량 스칼라 비교 불가):

## 9. 스태프 3.0 — 감독 능력 3축 + 명성(시장 가치)

> 📋 **설계 확정(2026-07-20, 사용자 기획 + 독립 리뷰 교정 반영 · 미구현)**.
> 스태프 2.0(§8.1)이 **코치**(육성·시장)를 정비했다면, 스태프 3.0은 **감독**을 정비한다 —
> 단일 스칼라 `charisma`(타임아웃 기세폭 하나뿐)를 실존 시스템에 각각 물린 **능력 3축**으로 쪼개고,
> 그 위에 **명성(시장 가치)** 을 얹어 감독 영입·재계약·서사를 만든다.
> 스태프 2.0의 두 원칙(**스칼라 지배전략 금지**·**죽은 스탯 금지**)을 감독 축에서도 그대로 집행한다.

### 9.0 철학

- **감독 = 결정(Decision)**: 누구를 기용할지·어떤 운영·어떤 철학. **감독은 선수를 직접 성장시키지 않는다.**
- **코치 = 실행(Execution)**: 훈련·성장·잠재력·상한. 이는 §1의 "감독=방향, 코치=상한/수명" 원칙의 재천명이다.
- 한 줄 표어: **"감독은 기회를 만들고, 코치는 그 기회를 성장으로 바꾼다."**
- **죽은 스탯 절대 금지**: 신설 감독 능력치·명성의 모든 축은 이미 존재하는 시스템(MATCH·FORM·OWNER·FINANCE·NEWS·드래프트)에만 연결한다. "표시만 되고 아무 데도 안 물리는 숫자"는 넣지 않는다(스태프 2.0 ①성향 orthogonality 게이트와 동일 기준).

### 9.1 감독 능력 3축 (0~100, 선수와 동일 체계) — 기존 `charisma` 대체

기존 단일 `charisma`(타임아웃 수렴폭 하나에만 물림)를 **세 개의 독립 축**으로 분해한다.
각 축은 서로 다른 실존 시스템에 물린다(죽은 스탯 금지).

| 축 | 이름 | 물리는 시스템 | 효과 |
|---|---|---|---|
| ① | **경기 운영** matchOps | MATCH_SYSTEM §7.4·§8 | 경기 중 의사결정 — 타임아웃(기존 `charisma`의 기세 수렴폭을 **그대로 이관**)·교체·작전 변경의 품질. **엔진 동작 등가에서 출발**(마이그레이션 시 `charisma`→경기 운영, `pull = matchOps/100 × 0.6` 그대로). 이후 교체/작전 품질 훅은 phase D에서 얹음 |
| ② | **육성 철학** dvPhilosophy | 기용 성향(드래프트·라인업) | **훈련 능력이 아니라 기용 성향.** U23 출전 기회·유망주 선발 우선순위·경기 경험 획득 소폭 보정. **훈련 효율에는 불개입**(코치 영역 침범 금지 — 독립 리뷰 교정). 감독은 어린 선수에게 *코트를 준다*, 성장 자체는 코치가 만든다 |
| ③ | **리더십** leadership | FORM_SYSTEM·OWNER_SYSTEM | 선수단 관리 — 벤치 선수 경기감각(FORM) 하락 완화·선수 불만 축적 감소·면담 성공률 보정. 실존 시스템에만 연결 |

- **감독 OVR = 3축 종합.** **명성은 OVR에 불포함**(실력과 명성은 분리). 예: "OVR 91·명성 48 = 실력파 무명" / "OVR 82·명성 94 = 거장(왕년의 명장)". 실력과 시장가치의 괴리 자체가 영입 판단의 재미다.
- **감독 유형(승부형 / 육성형 / 조직관리형)**: 생성 시 **능력치 범위의 방향만** 결정한다(고정 프리셋 아님 — 같은 유형도 전부 다른 수치). 승부형=경기 운영 편향·육성형=육성 철학 편향·조직관리형=리더십 편향.
- **유형 라벨은 저장하지 않는다** — 능력 프로필에서 **파생 표시** 권장. (기존 전술 성향[공/수/밸]과 유형 라벨을 둘 다 저장하면 이중화·모순 위험 → 표시는 3축 프로필에서 계산. 정합 체크 §9.5-1 참조.)

### 9.2 명성 (0~100, 시장 가치 — **능력치가 아니다**)

명성은 감독이 얼마나 "쳐주는" 시장가치인가다. **경기 엔진에는 전혀 개입하지 않는다.**

- **영향**(전부 시장·서사 레이어): 연봉 프리미엄 · 재계약 요구 강도 · AI 영입 경쟁 강도 · **FA 선수 유인**(명장 팀 오퍼 평가 소폭 가산) · 관중 소폭 가산 · 뉴스 등장 빈도 · 명장 열전(§9.4). ※ 스폰서 가치 연동은 **출시 후**(§9.5-4).
  > **역스칼라 지배 방지(독립 리뷰 최중요 교정)**: 명성이 "연봉만 비싸고 실익 없는 호구세"가 되면 유저는 항상 무명 감독만 쓰는 게 최적이 된다 → 명성은 **연봉 부담과 균형을 이루는 실질 유인**(FA 선수 유인·관중·서사)을 가져야 한다. "명장=호구"도 "무명=항상 정답"도 안 되게 phase D 밸런스 게이트가 확정.
- **저장 모델(핵심): 명성 숫자를 저장하지 않는다.** 대신 **감독 경력 로그**(시즌·팀·언론 예상 순위·실제 순위·플옵 성과·우승·경질 여부)를 영속하고, **명성 = 경력 로그의 순수 함수 파생.**
  - 이점: 산식 재튜닝 시 **무마이그레이션**(로그만 있으면 명성은 언제든 재계산) · 연표/명전/뉴스가 **공짜**(같은 로그를 읽음) · **중복 가산 원천 차단**(누적 변수에 매번 더하는 방식은 리플레이·이중가산 버그의 온상 → 명전 무저장 재계산 원칙[6장 ACHIEVEMENT]과 동일 척추).
- **평가 원칙: 절대 순위가 아니라 기대 대비.**
  - 예상 7위 → 실제 4위 = **상승** / 예상 2위 → 실제 5위 = **하락** / 예상 7위 → 실제 7위 = **변화 거의 없음**(리빌딩 팀에 불이익 없음).
  - **하락 트리거에 절대 순위(최하위 등)를 쓰지 않는다** — 기대 대비 부진·시즌 중 경질만 하락. (약체 팀을 예상대로 이끈 감독을 "꼴찌라서" 깎으면 리빌딩 지휘가 불가능해진다.)
  - **보조 축(소폭)**: 유망주 육성(U23 성장·주전 안착·신인상 배출) · 특별 업적(우승·정규 1위·기록).
- **티어**: 0~19 **무명** / 20~39 **주목** / 40~59 **인정받는 감독** / 60~79 **명장** / 80~100 **거장**(★1~5 표시).
  - "레전드"는 **선수 축 고유 용어**(영구결번·HOF)라 감독 최고 등급은 **거장**으로(용어 충돌 정정). "신예"는 나이 함의가 있어 하위 티어는 **주목**으로.
- **선수 인기(popularity)와의 구분**(정의 한 줄): **인기 = 팬이 좋아하는 정도(팬 파생)** / **명성 = 시장이 쳐주는 값(성과 파생)**. 둘은 다른 축이다.
- **초기 명성**: 레전드 선수 출신 감독은 이름값으로 높게 시작(기존 `starRep` 이진값을 흡수 — 수치는 튜닝).
- **전문 코치 경량 경력 추적**: 코치 재직 시즌·소속팀 성적을 기록해 감독 승격 시 **초기 명성으로 환산** → §6.3의 `headWorthiness` `coachRep` 상수 **50 고정 구멍**을 실제로 메운다(§6.3 미구현 항목과 연결 — 코치 경력이 승격·초기 명성에 실제 반영).

### 9.3 언론 예상 순위 (프리시즌 공개 콘텐츠 — 사용자 확정)

- 시즌 시작 전, **전력 기반**(선수 OVR·포지션 밸런스·외국인·연령·경험·벤치 뎁스)으로 전 팀 예상 순위를 **결정론 산출**.
- **뉴스 기사로 공개**("언론 예상 순위" — 개막 프리뷰, NEWS_SYSTEM). 이는 명성 평가·뉴스·스토리·감독 평가의 **공통 기준선**이다.
  > **왜 공개가 필수인가**: 예상 순위를 숨은 수치로 두면, 명성이 "보이지 않는 근거"를 인용하는 **가짜 드라마**가 된다("근거 없이 명장이 됐다"). 공개하면 유저가 "예상 7위였는데 4위로 끌어올렸다"를 직접 검증할 수 있다 → 서사가 정직해진다(가짜 드라마 금지 기둥).
- **폐기: 구단 목표를 유저가 매 시즌 선택하는 안** — 착취 경로(쉬운 목표만 선택)·AI 목표 부재 문제·매 시즌 강제 입력의 관전형 위배 → **예상 순위에서 자동 파생**한다.

### 9.4 시장·계약 UX (사용자 기획 채택)

- **연봉 = 능력(OVR) 기반 base + 명성 프리미엄(상한 캡).** **대체 금지** — 명성이 능력 연봉을 갈아치우는 게 아니라 **위에 얹는다**(실효과 스탯을 공짜로 만들지 않기 위함 — 스태프 2.0 "실효과 스탯 공짜 방지"와 동일 원칙).
- **재계약 화면**: 감독 카드(명성 등급·수치·3축·계약 잔여·현재/요구 연봉) + 시즌 주요 업적 + **요구 연봉 상승 이유 표시**(경력 로그에서 파생: "명성 상승·우승·타 구단 관심"). 버튼 **[재계약]** / **[놓아주기]**.
  - **놓아주기**는 확인창(**관심 구단 목록 표시** — FA로 이동해 경쟁팀에 갈 수 있다는 경고).
  - **카운터오퍼는 1회**(성공/결렬 판정으로 끝 — 흥정 루프 금지, 관전형).
- **AI 영입 경쟁**: 계약 만료 감독에게 명성 기반으로 **복수 구단이 관심**(결정론 시드). 
  - **감독 선호는 새 랜덤 속성이 아니다** — **명성 티어 + 유형에서 파생**(육성형 명장 → 젊은 로스터 선호, 승부형 명장 → 컨텐더 선호).
  - **공석 폴백 필수**: 최하위 팀도 반드시 누군가는 수락(데드락 방지 — 검증 게이트 ④).
- **감독 코멘트**: 재계약 화면 한 줄 심경 — **반드시 실제 데이터**(성적·계약·팀 연령)에서 파생한 문구만. **없는 인과를 지어내지 않는다**(가짜 드라마 가드 — "우승했지만 노쇠한 팀이 걱정" 같은 문구는 실제 성적+평균연령이 뒷받침될 때만).

### 9.5 정합 체크 (메인 세션 지적 — 기존 두 축·현행 효과와의 충돌 방지)

1. **전술 성향(공/수/밸)·훈련선호(핵심2+보조3)는 존치한다.** 3축 신설은 이 둘을 **대체하지 않는다**:
   - 전술 성향(`CoachStyle`) = 경기 스타일 분포(MATCH §8: 서브 공격성·속공 비중·타임아웃 타이밍) — **유지**.
   - 훈련선호(`trainingFocus` 핵심2+보조3) = 팀 성장 방향(TRAINING) — **유지**.
   - 신설 3축 = 운영 품질/기용/관리. **셋은 별개 축**이며, 3축 도입이 위 둘의 삭제로 오독되면 안 된다.
2. **체력 코치 "부상 감소·컨디션 관리"(사용자 원문)는 현행 미존재 효과.** 현행 체력 코치 효과(**노쇠 하락 둔화 + 피지컬 속도↑**, §1)를 정본으로 유지하고, **부상 감소 훅은 "후속 검토(INJURY_SYSTEM 연동)"로 격리** — 죽은 스펙 문구를 본문에 박제하지 않는다.
3. **"승부형: 출장한 선수는 효율적으로 성장"(사용자 원문)은 감독의 성장 효율 개입으로 오독 가능** → **"승부형은 U23 기회가 적고 주전 위주로 기용한다(주전은 경기 경험을 그대로 축적)"** 로 서술 교정. 감독은 기회를 배분할 뿐 성장 계수에 손대지 않는다(§9.0 철학·②육성 철학 훈련 불개입과 정합).
4. **스폰서 가치 연동은 출시 후.** 명성 영향 목록(§9.2)에서 스폰서/모기업 재정 연동은 FINANCE_SYSTEM 추가 부담이라 **출시 후** 각주(관중 소폭 가산까지가 출시 범위).
5. **`charisma` 제거 = 세이브 마이그레이션 1회.** `charisma` → **경기 운영**으로 값 이관(엔진 등가) + SAVE_SYSTEM **버전 범프**(현행 `SAVE_VERSION=3` → 4, §3.1 ③경로: 필드 모양/의미 변경).
   - **3축 생성으로 개정 필요한 생성식 열거**(phase A 작업 목록): 시드 감독 풀 생성(`data/seed.ts`) · `playerToCoach`(§6.2 코치 생성) · `coachToHead`(§6.3 승격 시 `charisma` 산출) · `DEFAULT_COACH`(`engine/match.ts`) · `CoachInfo` 타입 — 전부 `charisma` 단일 필드에서 3축 생성으로 개정.
