# STATS_PROTOCOL — 통계 측정·기입 원칙 (2026-06-12, 0장 추가 2026-06-17)

> 모든 시뮬레이션 통계(승률·분포·궤적·빈도)에 적용. 어기면 저표본 노이즈로 밸런스를 결정하거나,
> 로직이 바뀐 뒤 문서 수치가 조용히 거짓이 되는 사고가 난다.

## 0. 추정 절대 금지 (가장 중요 — 다른 원칙보다 우선)

**엔진의 수치·분포·동작·구조에 대한 어떤 단정도 추론으로 말하지 않는다. 측정해서 말한다.**

- 대상은 통계만이 아니다 — "이 분기는 0건일 것", "디그가 리시브보다 낮을 것", "이 수정이
  교차를 줄일 것" 같은 **동작·구조·수정 효과**도 전부 측정 대상이다. "정상이라면 ~할 것"
  류의 직관은 자주 틀린다(2026-06-17 한 세션에서 3연속 빗나감: 디그 품질 역전·커버 후보 0명
  가설·liveX 효과). 코드를 읽어 "당연히 그럴 것" 같아도 돌려서 확인한다.
- 통계가 없으면 → 기존 `tools/sim*.ts`로 재거나, 없으면 텔레메트리 싱크(결과 불변)를 더해
  새 도구를 만들어 돌린다. 표본 N≥10,000(1장). 측정 전엔 "추정컨대/아마/~일 것" 단정 금지.
- **측정 도구도 추정하지 말고 검증한다 (허위 오라클 금지).** 새로 만든 측정기는 결론의
  근거가 되기 전에 신뢰성을 확인한다: ① 알려진 정답 케이스로 sanity-check, ② 가능하면
  수정 전/후 A/B로 도구가 변화를 실제로 감지하는지 본다. 2026-06-17 커버 교차 조사에서
  잘못된 프록시(x 순서 역전)가 25~34% 허위 경보를 냈고, 올바른 점-선분 관통 지표로는
  0.8%였다 — 도구를 검증하지 않고 결론 냈으면 멀쩡한 연출을 "버그"로 고칠 뻔했다.
- 측정이 직관과 반대로 나오면 **측정을 믿는다**(단, 먼저 위 도구 검증을 통과한 측정에 한함).
- 검증 안 된 수정은 넣지 않는다 — A/B로 개선이 입증돼야 커밋한다(중립/악화면 되돌린다).

## 1. 표본 최소 10,000

- 승률·확률류: **경기 10,000+** (예: 스탯 A/B 실험은 진영 교대 포함 1만 경기).
- 동작·이벤트류: **이벤트 10,000+** (예: 포지션별 체력 궤적의 셀당 동작 수).
- 도달 불가능한 단위(예: 유니버스 단위 parity)는 가능한 최대 표본 + 산하 표본 수를 함께 적는다
  (예: "유니버스 8개 × 25시즌 ≈ 21만 랠리").
- 1만 미만은 **탐색용(방향 잡기)으로만** — 문서·커밋 메시지에 결론으로 적지 않는다.

## 2. 기입 형식 — 표본 수 + 로직 버전 필수

```
수치 (N=10,000경기 · 엔진 d2a5eb5 · 2026-06-12)
```

- N: 무엇을 몇 번 (경기/랠리/이벤트/시즌).
- 엔진: 측정 시점의 커밋 해시(짧은 형식) — 어떤 로직을 시뮬레이션했는지의 단일 식별자.
- 이 메타데이터가 있으면, 내(Claude)가 로직 변경 시 문서 갱신을 놓쳐도 독자가 stale함을 안다.

## 측정 도구 규약

- 모든 측정 도구는 표본 수를 CLI 인자로 받는다 (`npx tsx tools/simX.ts [N]`).
- 도구 기본값은 빠른 스모크용으로 작아도 되지만, **문서에 적는 실행은 N≥10,000**.
- 오래 걸리는 실행은 백그라운드(`run_in_background`)로.
