# BOARD_RULES — 관전 연출 검증 기준 (구단주의 눈)

> 사용자가 폰으로 경기를 **보다가** 직접 잡아낸 어색한 장면들을 검증 가능한 기준으로 정식화한 문서.
> 관전형 게임에서 경기 연출은 1순위 투자처(CLAUDE.md 기둥 1)이므로, 이 기준들은 전부
> `tools/auditBoard.ts`의 **상설 룰**로 자동 검사된다. 새 주의사항이 나오면:
> **① 이 문서에 행 추가 → ② 연출 수정 → ③ 감사 룰 추가(회귀 방지) → ④ 풀배터리 0건** 순서로 처리한다.
>
> 검증 실행: `/verify-board` 스킬 또는 `npx tsx tools/auditBoard.ts [경기수]`
>
> **수비/리시브 포지셔닝의 "왜"** (역할→위치 자동화 모델)는 [COURT_POSITIONING](./COURT_POSITIONING.md)에
> 별도 기준으로 둔다 — 본 문서가 *어색한 장면*(증상→수정)이라면, 그쪽은 *왜 그 자리인가*(역할 원리)다.
> 포지션 룰(32·43·45·46·49 등)이 오락가락한 근본 원인이 "magic 숫자"라 그 모델을 단일 기준으로 박는다.

## 대원칙

1. **보드는 엔진의 사실(`PointHow`)을 그대로 그린다** — 연출이 결과를 지어내지 않는다(분포 = 엔진 분포).
2. **모순 금지가 리얼리즘보다 우선** — "공에 닿았는데 못 잡음", "죽은 공에 대형 재배치" 같은
   장면은 한 컷만 나와도 관전 몰입이 깨진다.
3. 렌더(MatchCourt)와 감사기는 **같은 순수 모듈**(courtPath/courtDirector/courtLayout)을 쓴다 —
   화면에 보이는 위치 = 검사하는 위치.

#### 2b 설계(2026-06-24) — 보드 디그 마커를 엔진 디그 귀속(touches)에 그래프트

> (rules 52/55/56과 같은 "화면=기록" 클래스의 디그 칸). recvId/setId/byId 그래프트 패턴의 연장.

**그래프트(점진·저위험, 루프 전면 재작성 회피)**: 엔진 디그 터치 순서 `digTouches = touches.filter(act==='dig')`를
큐로 들고, 보드가 **att를 뒤집는 실제 디그**(클린 디그·팁 디그세이브·블록 전환 디그 — `att=def`로 전환되는 것)를
그릴 때마다 큐에서 하나 꺼내 **그 선수 슬롯을 디거로** 쓴다. 블록커버(att 유지=재공격)·프리볼(엔진에 없는 보드
연출)은 큐를 **소비 안 함**(비박스 비주얼 — 보드 고유 선수로 그대로, "살림/찬스볼"로 식별).
- **워프 회피(핵심)**: 엔진이 고른 디거가 공에서 멀면 순간이동(룰 A/N)이 난다 → 디그 좌표를 **그 디거 위치 쪽으로**
  잡는다(스파이크 낙구를 디거 존으로 유도 — 공격은 원래 수비수 쪽으로 가므로 자연스러움). nearestDig가 고르던
  좌표 대신 "엔진 디거의 실제 위치"로 공을 보낸다.
- **hop 구조 불일치**: 보드가 프리볼/커버로 att-flip을 추가하면 보드 디그 수 > 엔진 디그 수일 수 있다 → 큐 소진 시
  잔여 보드 디그는 nearestDig 폴백(그 디그는 박스에 없는 보드 연출 디그라 무방). 큐가 남으면(보드가 더 일찍 종결)
  남은 엔진 디그는 안 그려짐(그 박스 디그는 여전히 집계됨 — 통계 불변). **per-event 100%는 구조 재생이 전제**라,
