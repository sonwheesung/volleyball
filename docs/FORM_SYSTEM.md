# 경기감각 (Match Form) — 설계 문서

> **구현 현황(2026-06-11)**: ✅ `engine/form.ts` + dynamics(출전 이력 기록 → 날짜별 감각 적용,
> `formFactorOnDay`/`availableTeamPlayers`). UI 컨디션 점은 미구현(다음 단계).
> 검증: 100시즌 parity 유지(std 6.2·왕조 7·r −0.01·우승경험 7/7), KOVO 20지표 유지,
> 무결성 0위반. 결장 누적 선수 평균 계수 0.933(설계 −7% 바닥과 일치).

> 2026-06-11 설계. "훈련은 기술을 만들고, 경기는 감각을 만든다."
> 훈련만 하고 실전을 안 뛰면 못해지는 선수 — 현실 배구의 상식을 엔진 밖 시즌 계층에 넣는다.

---

## 0. 왜 넣는가

- 지금은 벤치 선수와 주전의 차이가 **성장 속도**뿐이다(뛴 선수만 XP). 결장 자체의 대가가 없다.
- 경기감각이 생기면: 부상 복귀 선수가 바로 펄펄 날지 않고, 벤치에 묻힌 베테랑이 녹슬고,
  감독 건의로 빼놓은 선수(OWNER_SYSTEM)를 다시 쓸 때 **복귀 비용**이 생긴다.
- 로테이션 운영(가비지타임에 유망주 출전)이 "성장 + 감각 유지"라는 이중 보상을 갖게 된다.

## 1. 정의 — 파생 스탯, 저장하지 않는다

- 경기감각(form) = **최근 출전 이력의 함수**. 선수 객체에 저장하지 않고 시즌 계층(dynamics)이
  날짜별로 파생한다(리플레이 결정론 보존 — 부상과 동일 패턴).
- `form(p, day) = 1 − 0.07 × (1 − 최근5경기 출전 비율)` (placeholder)
  - 최근 5경기 모두 출전 → 1.00 (페널티 없음 — **주전은 아무 변화 없음, 밸런스 보존**)
  - 5경기 전부 결장 → 0.93 (체감 −7%: OVR 75가 70처럼 뜀)
  - 시즌 개막(이력 없음) → 1.00에서 시작(오프시즌 동안 리셋 — 다 같이 새 출발)
- 효과 적용: 시즌 계층이 그날 출전 명단을 만들 때 능력치(윗단 입력)에 form을 곱한 사본을
  엔진에 전달. **엔진은 모른다** — 받은 선수를 굴릴 뿐(결정론·SOLID 유지).

## 2. "출전"의 기준 — 이미 있는 것을 재활용

- dynamics forward-pass가 경기마다 라인업(코트 7인)을 이미 만든다(부상 굴림용).
  그 명단 기록이 곧 출전 이력 — 새 데이터 소스 없음, 순환 없음(day 순서 진행이라
  오늘의 감각은 어제까지의 출전으로만 계산).
- 부상 결장도 결장이다 — 복귀 직후 감각이 낮은 게 **부상 시스템의 공짜 디테일**이 된다.

## 3. 회복 — 뛰면 돌아온다

- 별도 회복 수식 없음: 최근 5경기 창이 굴러가며 출전이 쌓이면 자동 회복.
  복귀 후 5경기를 채우면 1.00 — "감을 찾는 데 2주" 정도의 체감.

## 4. 표시

- 선수 상세·스쿼드에 컨디션 점(●): 좋음(1.0, 초록) / 살짝 무뎌짐(0.97~, 노랑) / 녹슮(~0.93, 빨강).
- 경기 보드 중계와 연결(추후): "오랜만의 실전, 감각이 아직인 모습".

## 5. 밸런스 가드 (시뮬 검증 목표)

- 주전 로테이션이 정상 가동되는 팀은 **수치 변화 0** — KOVO 정렬·parity에 영향 없음을
  풀배터리(simKovo·simStats·simLeague 다중 유니버스)로 확인하고 들어간다.
- 영향 받는 경우만: 부상 복귀 직후, 벤치 지시 해제 직후, 시즌 중 영입 선수의 첫 1~2주.
- 상한 −7% 고정(중첩 없음). 감각이 부상·특성과 곱으로 겹쳐도 바닥이 뚫리지 않게 clamp.

## 6. 코드 맵 (구현 시)

- `engine/form.ts` — 순수: 출전 이력 → form 계수. 테스트 우선.
- `data/dynamics.ts` — forward-pass에서 경기별 onCourt 기록 → 날짜별 form 산출,
  `availableTeamPlayers`가 form 적용된 선수 사본 반환(부상 필터와 같은 자리).
- UI — 스쿼드/선수 상세 컨디션 점.
