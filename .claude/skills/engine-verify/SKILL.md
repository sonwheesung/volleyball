# engine-verify — 100+ 독립 세션 엔진 전수 검증 + 자기보강

> **왜**: [`independent-verify`](../independent-verify/SKILL.md)는 **1개** 신선 세션이 한 주장을 검증한다.
> 그러나 단일 세션은 **자기 한 사각에 갇힌다** — 그 세션이 안 본 area·안 쓴 렌즈의 버그는 통째로 샌다.
> 100+개를 **area×lens×seed로 분산 + 의도적 중복**으로 띄우면 세 가지가 동시에 된다:
> 1. **넓은 표면 동시 수색** — 엔진 44모듈·데이터 36셀렉터를 한 번에.
> 2. **검출 신뢰도 + 사각 측정** — 같은 버그를 *여러* 세션이 잡으면 신뢰↑. *한* 세션만 잡았으면
>    "**A는 찾고 B는 왜 놓쳤나**"(B의 배정 area/lens/seed 갭)가 곧 사각(TEST_METHODOLOGY §4)이다.
> 3. **자기보강** — 그 사각을 만든 누락 렌즈/area를 `lenses.md`(진화 체크리스트)에 환류 → 다음 run이 커버.
>
> 검증자 역할·규칙·보고형식의 정본은 [`.claude/.agents/independent-verifier.md`](../../.agents/independent-verifier.md)
> 와 [`docs/TEST_METHODOLOGY.md`](../../../docs/TEST_METHODOLOGY.md) §1.B·§3·§4. 여기서 다시 쓰지 말고 인용/전달한다.

## 사전조건 (반드시 확인)

- **오케스트레이션 = `Workflow` 도구.** 이 스킬을 호출한 것 자체가 다중 에이전트 오케스트레이션 **opt-in**이다
  (Workflow 도구 규칙: "스킬 지시로 Workflow 호출"=허용). 100+ 에이전트는 **토큰·시간 비용이 크다** —
  사용자가 이 스킬을 명시 호출했을 때만 돈다. 규모/예산은 `budget`로 스케일(없으면 기본 N=100).
- **검증자는 소스 수정·git 금지** — 측정 도구만 작성, 결론은 보고로. 고치는 건 메인 세션.

## 실행 순서

### 1. 범위·렌즈 로드 (lenses.md = 진화하는 정본)
- [`lenses.md`](./lenses.md)를 읽는다 — **area 목록**(엔진/데이터 표면)과 **lens 목록**(검증 각도:
  불변식·분포·결정론·문서↔코드 drift·경계/극단·동시성·장기누적·단위정합·교차계층 귀속 …) + 과거 run이 환류한 항목.
- `docs/README.md` "검증 루틴"·`docs/TEST_METHODOLOGY.md`·`docs/STATS_PROTOCOL.md`와 건드린 area의 `*_SYSTEM.md`를
  진입점으로 검증자에게 넘길 준비. **추정 금지** — area·렌즈는 lenses.md/코드에서 가져오지 지어내지 않는다.

### 2. 분산 설계 (중복 = 사각 측정의 핵심)
- **N ≥ 100 = areas(M) × lenses(L) × seeds**. 핵심: **각 area를 서로 다른 렌즈/seed의 세션 ≥3개가 겹쳐** 본다.
  겹침이 없으면 "A 찾고 B 놓침"을 **측정할 수 없다**(중복은 낭비가 아니라 *검출 매트릭스의 표본*이다).
- 각 세션에 `{area, lens, seed, 읽을 파일·문서}`를 **배정표**로 고정(결정론·재현). 배정표를 산출물로 보관.

### 3. Workflow 팬아웃 (100+ 독립 검증자)
- `Workflow`로 N개 `agent()` 스폰. 각 프롬프트 = **independent-verifier 규칙**(문서서 기대 도출·구현 베끼지 말 것·
  기존 가드는 돌려보되 *다른 각도* 새 측정·**모든 체크 A/B 자가검증**·소스/git 금지) + 배정(area·lens·seed) +
  진입점(파일·문서). **구조화 스키마로 반환**(파싱 불요):
  ```
  { area, lens, seed, findings: [{ severity, claim(사양용어), symptom, evidence, repro, confidence, toolPath }],
    coveredOk: [검사했고 정상인 불변식] }   // coveredOk = "이 세션이 무엇을 봤나" = 미스 분석의 분모
  ```
- 동시성은 Workflow가 캡(≈cores−2)하고 100+는 큐로 소화. 길면 `run_in_background`.
- 편향 누수 금지: 정답·내 구현·고친 파일을 알려주지 않는다(독립성).

### 4. 2차 적대 검수 (나 = 오케스트레이터 — 사용자 요구 "너가 또 2차 검수")
- 모든 후보 finding을 **재현·반증 시도**: 해당 sim/측정 도구를 직접 돌리고, **A/B 자가검증**(일부러 깬 입력에
  검출되나)으로 오라클 유효성 확인. 재현 안 되거나 A/B 둔감이면 **허위양성으로 기각**(STATS_PROTOCOL 0장).
- 살아남은 것만 "실버그". WAI(문서가 정상이라 명시)면 고치지 말고 기록만.

### 5. 디둡 + 검출 매트릭스 (사각 측정)
- finding을 **시그니처**(area+증상+위치)로 묶어 디둡. 각 실버그에 대해 **버그 × 세션** 매트릭스를 만든다:
  어느 세션이 잡았나(found) / 잡을 위치였는데 놓쳤나(그 area를 본 `coveredOk` 세션 중 miss).
- **"A 찾고 B 놓침" 사유 분석**: B의 배정(area/lens/seed)·읽은 문서·coveredOk를 비교해 **무엇이 B의 시야를 가렸나**
  를 한 줄로(예: "B는 분포 렌즈만 — 결정론 위반은 결정론 렌즈에서만 나옴"·"B는 *_SYSTEM 문서 미열람"). = TEST_METHODOLOGY §4 사각.
- 검출률 보고: 실버그별 found/총-담당 세션(낮을수록 잡기 어려운 = 사각 깊은 버그).

### 6. 자기보강 (스킬이 스스로 똑똑해짐)
- **사각 → `lenses.md` 환류**: 미스를 만든 **누락 렌즈/under-covered area**를 lenses.md "run 환류" 절에 append
  (다음 run이 그 렌즈·area를 기본 배정에 포함). 한 세션만 잡은 강력한 렌즈는 **표준 렌즈로 승격**.
- **실버그 정식 등록**: `docs/EDGE_CASES.md`(영입/오프시즌)·`BOARD_RULES`(연출)·`TEST_METHODOLOGY §4`(사각) 중 해당에
  `증상→원인→수정→잡는 도구` + **상설 가드**(검증자가 남긴 `_iv_*`/`_dv_*` 중 A/B 통과분을 `docs/README.md` 검증 루틴에 등록).
- 수정은 메인 세션(엔진/셀렉터를 고친다 — 도구를 느슨화하지 않는다). 새 시스템 케이스면 `analyze-cases`→`verify-cases` 연계.

## 끝맺음

- 보고: 돌린 **세션 수·area×lens 배정 요약**, **실버그**(재현 시드+심각도+file:line), **검출 매트릭스**(버그별 found/담당),
  **사각 분석**("A 찾고 B 놓친" 사유), **lenses.md 환류분**(추가/승격한 렌즈·area), 등록한 가드.
- 핵심 산출물 두 줄: ① 엔진이 실제로 견뎠나(실버그 0이면 "100+ 독립 세션 수렴") ② **이번에 스킬이 무엇을 새로 배웠나**(자기보강).

## 원칙
- **추정 금지·A/B 필수** — 돌려서 본 것·깬 입력에 무너지는 오라클만 신뢰. 2차 검수가 마지막 관문.
- **중복은 자산** — area 겹침이 없으면 사각을 못 잰다. 분산은 "넓게"가 아니라 "넓게+겹치게".
- **비용 인지** — 100+ 에이전트는 비싸다. budget로 스케일, 명시 호출 시에만.
- **자기보강이 목적** — 버그를 찾는 것만큼 *"왜 어떤 세션은 못 찾았나"를 lenses.md에 남기는 것*이 이 스킬의 산출물.
